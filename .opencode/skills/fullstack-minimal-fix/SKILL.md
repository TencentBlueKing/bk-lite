# fullstack-minimal-fix

## Goal
Fix a clearly scoped issue end-to-end with the smallest safe diff.

## Must do
- Treat the issue as one fullstack flow
- Change frontend, backend, docs, or tests together if needed
- Keep diff minimal
- Follow existing code patterns
- Run validation for every changed area
- Stop if scope expands beyond the original issue
- Prepare draft PR summary

## Must not do
- Do not split the issue artificially by module
- Do not refactor unrelated code
- Do not change forbidden paths
- Do not suppress type errors
