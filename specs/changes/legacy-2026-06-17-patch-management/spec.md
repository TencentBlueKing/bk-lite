## 1. 产品定位

BK-Lite 新增独立补丁管理 app，以“主机是否满足补丁基线”为核心，而不是让用户围绕扫描任务和安装任务工作。

核心闭环：

> 纳管主机 → 绑定固定清单基线 → 自动评估 → 产生不合规项 → 选择主机与补丁治理 → 重启与验证 → 恢复合规

扫描、安装、重启和验证仍由后台任务执行，但只作为风险治理的执行记录。日常入口是主机合规状态、待治理风险及下一步动作。

本设计重点参考阿里云云安全中心的资产风险、受影响主机、一键修复、结果验证和失败排查体验，但不照搬其 Agent、计费、快照或自动修复架构。

### MVP 覆盖

- Windows/Linux 服务器 OS 安全补丁及管理员导入的通用补丁。
- Microsoft Update Catalog、WSUS、Linux yum/dnf/apt repo 和手工上传。
- 固定必装补丁清单基线、不可变基线版本及主机绑定。
- 周期评估、立即评估、主机合规状态和风险项。
- 主机、补丁、基线三个风险视角及一键治理。
- 立即执行、一次性执行窗口、窗口内主机级串行排队。
- 安装、重启、验证、失败重试、导出和操作审计。
- PatchTarget 独立 Ansible 目标及 Node Manager 已纳管节点。

### MVP 不做

- Mac 和第三方应用补丁。
- 动态风险规则基线、推荐/可选/禁止等级。
- 单机补丁豁免和白名单。
- 审批、测试批准、自动快照和自动回滚。
- 周期自动修复策略。
- 用户可配置的分批策略。
- 独立报表中心和独立通知设置。
- 补丁卸载、回滚和专用 Patch Agent。
- 页面自定义 Linux 插件。

### 首页

首页展示当前治理态势和配置待办：

- 纳管主机数。
- 已评估合规率。
- 评估覆盖率。
- 不合规主机数。
- 未配置基线主机数。
- 待治理风险数。
- 修复异常数。

统计规则：

```text
纳管主机数 =
合规 + 不合规 + 待评估 + 评估失败 + 未配置

已评估合规率 =
合规 / (合规 + 不合规)

评估覆盖率 =
(合规 + 不合规) / 纳管主机数
```

未配置、待评估和评估失败不进入合规率。首页可显示其状态分布，但不将其误报为不合规。
当分母为 0 时，对应比率显示 `--`，不显示 0% 或 100%。

点击“未配置基线主机数”进入目标管理并自动筛选未配置主机；其他指标跳转到对应风险或主机筛选结果。

### 风险治理

“待治理风险”提供：

- 主机视角：某台主机缺少哪些基线要求。
- 补丁视角：某项补丁要求影响哪些主机及状态分布。
- 基线视角：某个基线版本下的主机合规和治理分布。

聚合对象不设置单一状态，而显示风险项状态分布。例如：

> KB503XXXX：影响 5 台｜22:00-01:00 计划 3 台｜23:00-02:00 计划 2 台

“执行记录”展示评估、安装、重启和验证任务的计划、进度、主机级结果、日志、取消和失败重试。

### 补丁库

补丁库是补丁资产和就绪状态中心，不是脚本编辑器。默认展示可用补丁，并提供“需处理”异常视图。

列表重点字段：

- KB/包名、标题和严重级别。
- 适用 OS、版本和架构。
- 来源。
- 就绪状态。
- 被基线引用数量。
- 最近同步时间。

### 基线管理

用于创建和维护固定清单基线：

- 基线名称和说明。
- 单一 OS 类型。
- 适用版本范围和架构。
- 全部为必装项的补丁要求清单。
- 当前版本、绑定主机数和合规分布。
- 历史版本查看。

### 目标管理

目标管理维护 PatchTarget，并可选择 Node Manager 已纳管节点执行治理。

列表展示：

- 主机、IP、OS、版本和架构。
- 当前基线模板。
- 当前基线版本。
- 合规状态。
- 最后评估时间。
- 连通性状态。

主机可以不绑定基线。支持单台或批量绑定、切换基线；只允许选择与主机 OS、版本和架构兼容的基线。

## 3. 核心对象

新增后端 app `server/apps/patch_mgmt`。

### PatchSource

补丁源配置：

- Windows：Microsoft Update Catalog、WSUS。
- Linux：yum/dnf/apt repo。
- 代理、启用状态、同步状态和连通性测试。

### WindowsOsVersion

Windows Server 版本识别配置：

- 显示名称。
- 一个或多个规范化版本别名。
- 可选构建号范围。
- 支持架构。
- 启用状态和是否内置。

内置 Windows Server 2008、2008 R2、2012、2012 R2、2016、2019、2022、2025。管理员可扩展未来版本或特殊版本标识。

版本配置只用于主机识别和基线适配，不开放检测、安装或重启脚本。重复别名和重叠构建号范围禁止保存。

### LinuxPatchPlugin

Linux 通过后端插件注册能力：

- 插件声明发行版匹配规则、版本范围和包管理器。
- 提供系统识别、repo 解析、评估、安装方案、执行、验证和重启检测接口。
- MVP 内置 Debian 系插件和 RHEL 系插件。
- 页面不提供插件代码管理入口。

未匹配插件为 `unsupported`；匹配后执行异常为 `failed`。

### Patch / PatchPackage

Patch 表示补丁资产和要求语义，PatchPackage 表示实际安装包。

Patch 维护：

- OS 类型、补丁类型、KB/包名、标题、严重级别和 CVE。
- 适用 OS、版本、架构和规则。
- Windows 替代关系。
- Linux 包版本关系。
- 来源、发布时间、最近同步时间和就绪状态。

PatchPackage 维护：

- 文件名、大小、hash 算法和值。
- 签名或来源校验状态。
- Object Storage object key。
- 下载状态、失败原因和重试信息。

相同 hash 的包去重复用。

### PatchBaseline / PatchBaselineVersion

PatchBaseline 是可复用基线模板：

- 名称、说明、OS 类型、适用版本范围、架构。
- 当前版本和组织归属。

PatchBaselineVersion 是不可修改的版本快照：

- 基线、版本号、补丁要求快照。
- 创建人、创建时间和变更说明。

每次修改基线都生成新版本。已绑定主机自动使用新版本，历史评估和任务继续引用原版本快照。MVP 允许查看历史版本，不提供回滚。

### BaselineRequirement

BaselineRequirement 表达“主机必须达到的状态”，而不是固定安装包：

- Windows：指定 KB 要求，指定 KB 或可信来源确认的有效替代 KB 均可满足。
- Linux：包名及最低版本，已安装版本大于或等于最低版本即满足。

基线保留管理员选择的原始要求，不因同步到更新替代补丁而修改历史语义。

### HostBaselineBinding

主机当前基线绑定：

- 目标来源、目标标识和基线。
- 当前生效基线版本。
- 绑定人和绑定时间。

每台主机同时最多绑定一个基线，也允许未绑定。

### ComplianceAssessment / HostComplianceSnapshot

ComplianceAssessment 表示一次评估运行；HostComplianceSnapshot 表示主机针对某个基线版本的评估快照。

主机合规状态：

- `compliant`：所有适用要求均满足。
- `non_compliant`：存在缺失要求。
- `pending`：新绑定、基线变更或旧结果过期，等待评估。
- `failed`：连接、权限、识别或评估执行失败。
- `unconfigured`：未绑定基线。

评估快照保存满足证据，例如已安装指定 KB、替代 KB或当前 Linux 包版本。

### PatchRiskItem

风险项按“主机 × 基线版本 × 基线要求”生成。

数据层分开保存：

合规状态：

- `missing`：当前不满足要求。
- `satisfied`：已通过指定补丁、替代补丁或更高版本满足。
- `invalidated`：因基线删除要求、切换基线等不再适用。

治理状态：

- `unplanned`：待修复。
- `scheduled`：已进入未执行任务。
- `remediating`：安装中。
- `pending_reboot`：等待重启验证。
- `failed`：安装、重启或验证失败。
- `fixed`：验证通过。

页面可组合显示“待修复、已计划、修复中、待重启、修复失败、已修复、已失效”，但不能把合规状态和治理进度混成一个数据库状态机。

同一风险项不能同时进入多个有效治理任务。不同主机上的同一补丁要求可分别进入不同任务和执行窗口。

### GovernanceTask / GovernanceTaskHost / RemediationPlanItem

GovernanceTask 是后台执行记录，类型包括：

- 评估。
- 安装。
- 重启。
- 验证。

安装治理任务保存：

- 选择的风险项快照。
- 执行方式和执行窗口。
- 重启策略。
- 超时时间。
- 创建人、范围确认时间和审计信息。

GovernanceTaskHost 保存主机级队列、阶段、结果和日志。

RemediationPlanItem 分离：

- 基线要求。
- 创建任务时计划安装的补丁。
- 执行时重新计算的实际安装补丁。
- 选择替代补丁或依赖包的原因。

## 4. 补丁就绪流水线

就绪状态：

- `processing`：处理中。
- `ready`：可加入基线并可一键治理。
- `action_required`：缺包、元数据不完整或需管理员补充必要信息。
- `unavailable`：完整性失败或平台当前没有执行能力。

只有 `ready` 补丁可以加入基线。

官方源补丁自动完成：

1. 同步元数据和适用范围。
2. 下载或关联安装包。
3. 校验 hash、签名和文件完整性。
4. 匹配 Windows 通用引擎或 Linux 插件。
5. 解析替代、版本和依赖关系。
6. 确认检测、安装、验证和重启判断能力。

手工上传只要求管理员选择包类型、适用系统、版本和架构等必要信息，不允许编辑任意执行脚本。

同步失败后允许手工上传并补齐元数据；“需处理”页面提供补传包、重新下载、重新解析和查看不支持原因。

## 5. 基线与评估

### 基线规则

- 一个基线只适用于一种 OS 类型及版本范围。
- 基线内全部要求均为必装项。
- 不支持单机豁免；确需不同要求时应绑定另一套基线。
- 未就绪补丁不能加入基线。

### 基线变更

保存新基线或修改补丁清单时，弹窗展示受影响主机数，并让管理员选择：

- **立即扫描**：立即创建评估任务。
- **按周期扫描**：等待全局扫描周期。

选择按周期扫描后：

- 受影响主机立即进入 `pending`。
- 旧评估结果标记过期。
- 旧结果不再计入当前合规率。

删除基线要求或切换主机基线后，旧风险标记 `invalidated`，不标记为已修复。

### 扫描设置

MVP 提供一个全局扫描周期，不允许每个基线单独设置。

触发评估：

- 全局周期到达。
- 主机首次绑定或切换基线后选择立即扫描。
- 基线变更后选择立即扫描。
- 页面点击立即扫描。
- 安装或重启完成后的自动验证。

## 6. 风险治理入口

支持三个治理入口：

- 主机维度：治理该主机选中的或全部缺失要求。
- 补丁维度：选择受影响主机治理指定补丁要求。
- 基线维度：选择该基线下的主机批量治理缺失要求。

一次任务可以包含多台主机和多个补丁。一个任务使用统一的执行方式、执行窗口和重启策略。

同一主机的多个补丁统一完成安装后再判断重启，避免重复重启。单个补丁失败不阻止与其无依赖关系的其他补丁继续执行，任务可汇总为部分成功。

## 7. 提交前范围确认

用户点击“一键安装”后，页面只确认范围，不连接主机执行实时检查。

展示：

- 主机数、补丁要求数和风险项数。
- 本次纳入及未纳入治理的主机。
- 已进入其他有效任务、已失效、补丁未就绪等平台已知阻断项。
- Windows 基线要求及预计采用的替代补丁。
- Linux 最低版本要求及预计安装版本。
- 执行方式。
- 选择窗口执行时的开始和结束时间。
- 重启策略。
- 可能涉及重启的主机数量。

用户需确认：

> 我已确认影响范围、必要备份和重启安排。

系统记录确认人、确认时间和任务快照。

## 8. 执行方式、窗口与队列

### 立即执行

“立即执行”表示提交后立即进入执行时预检，不进入排队。

- 遇到正在执行的 BK-Lite 主机治理任务：该主机标记“任务冲突未执行”。
- 遇到 Windows Update、yum、dnf、apt、rpm 或 dpkg 临时占用：该主机标记“系统更新占用，未执行”。
- 环境异常：标记“预检失败”。
- 其他无冲突主机继续执行。
- 未执行风险恢复为待修复，管理员可一键重试。

若已有窗口任务正在该主机队列中等待且已进入有效窗口，立即任务不插队，按冲突未执行处理。

### 执行窗口

用户设置一次性执行窗口，例如：

> 2026-06-25 22:00 至 2026-06-26 01:00

规则：

- 窗口开始前不执行。
- 窗口内同一主机严格串行。
- 不同主机独立执行。
- 同一主机按进入窗口队列的先后顺序 FIFO。
- 重叠窗口只提示可能排队，不阻止任务创建。
- 临时任务或系统更新占用时，在窗口内等待并周期性重试。
- 窗口结束后不启动新的安装。
- 已启动安装不强制中断，继续完成并标记“超出窗口完成”。
- 因排队或占用未能启动的主机标记“窗口结束未执行”，相关风险恢复待修复。

排队状态必须区分：

- 等待 BK-Lite 前序任务。
- 等待系统更新进程。
- 环境异常，无法执行。

### 重启策略

MVP 只提供：

- `no_auto_reboot`：不自动重启，安装后进入待重启。
- `reboot_in_window`：仅在执行窗口内剩余时间充足时自动重启。

立即执行固定为 `no_auto_reboot`，页面不展示窗口重启选项；只有执行窗口任务可以选择 `reboot_in_window`。

系统使用统一的重启预留时长：

```text
当前时间 + 重启预留时长 <= 窗口结束时间
```

满足时才启动重启；否则转为待重启。立即执行没有窗口，不自动重启，需后续安排执行窗口处理待重启风险。

待重启风险支持单台或批量创建重启治理任务，并且必须设置执行窗口。重启任务同样遵循主机级 FIFO、窗口结束不再启动和执行时预检规则；重启完成后自动验证基线要求。

## 9. 执行时预检与安装方案

预检是治理任务的首个执行阶段：

```text
等待执行/等待窗口
→ 预检中
→ 安装中
→ 重启中或待重启
→ 验证中
→ 完成
```

预检内容：

- 主机连通性和执行权限。
- 当前 OS、版本和架构。
- 风险项及基线版本仍有效。
- 当前仍不满足基线要求。
- 补丁包可用、hash 和签名有效。
- 磁盘空间。
- Windows Update、Windows Modules Installer 和包管理器状态。
- BK-Lite 主机任务占用。
- 外部更新进程和事务占用。
- 当前待重启状态。
- 实际安装补丁、替代补丁和依赖方案。

临时占用按执行方式处理；rpm 数据库损坏、dpkg 中断、依赖无法解析等需要人工处理的问题直接预检失败，不进入等待。

### 实际安装方案

- Windows：指定 KB 或可信源确认的有效替代 KB 均可满足要求。
- Linux：达到或超过最低版本即可满足要求。
- 前置补丁和依赖包由系统自动排序。
- 被替代补丁跳过。
- Linux 依赖优先由包管理器事务处理。
- 某补丁失败后，仅阻止依赖它的后续补丁。

执行前重新计算方案。普通的同源替代更新可以继续执行并记录差异；若方案扩大影响范围、改变包来源或改变重启要求，则暂停该主机并要求重新确认。

任务详情同时展示：

> 基线要求 → 当前满足证据 → 创建时安装方案 → 实际安装方案

## 10. 风险与任务状态展示

风险示例：

- 待修复。
- 已计划 · 22:00-01:00。
- 修复中。
- 待重启。
- 修复失败。
- 已修复。
- 已失效。

同一补丁影响 5 台，可以：

- 选择 3 台创建 22:00-01:00 任务。
- 选择 2 台创建 23:00-02:00 任务。

聚合显示各状态数量，状态归属于每条风险项。

任务主机阶段：

- 等待执行。
- 等待窗口。
- 等待前序任务。
- 等待系统空闲。
- 预检中。
- 预检失败。
- 安装中。
- 重启中。
- 待重启。
- 验证中。
- 成功。
- 失败。
- 窗口结束未执行。
- 任务冲突未执行。
- 已取消。

任务级状态根据主机和风险项汇总为等待、执行中、完成、部分成功、失败、部分取消或已取消。

## 11. 基线变更与计划任务

主机切换基线或基线发布新版本时：

- 旧基线中不再适用的风险标记失效。
- 计划任务中对应项保留快照，但执行时跳过。
- 仍属于新基线的风险保留原计划。
- 任务内所有风险均失效时，任务自动取消并注明“基线变更”。
- 变更弹窗显示受影响主机和计划任务数量。
- 管理员选择立即扫描或按周期扫描。

计划任务只包含创建时选中的风险项。基线后续新增补丁不会静默加入已有任务。

## 12. 取消与重试

- 尚未开始的任务：全部取消，风险恢复待修复。
- 部分主机已执行：只取消等待执行、等待窗口、排队或尚未下发的主机。
- 正在预检、安装、重启或验证的主机不中断，继续完成。
- 已安装但待重启时取消任务，不撤销安装，风险保持待重启。
- 任务可显示部分取消，并记录取消人、时间和原因。
- 修复失败、任务冲突未执行、系统占用未执行、窗口结束未执行均支持按失败项重新治理。

## 13. 日志、导出与错误处理

执行记录保存主机级：

- 当前阶段和进度。
- 实际开始、完成时间及是否超出窗口。
- exit code。
- stdout/stderr 摘要和流式日志。
- 失败阶段、错误码、错误原因和处理建议。
- 基线要求、安装方案及方案变化。

扫描、风险和治理结果支持按当前筛选条件导出 Excel。

错误应区分：

- 目标连接与权限。
- Windows 版本未识别。
- Linux 暂不支持。
- 补丁未就绪。
- 磁盘空间不足。
- 更新服务不可用。
- 临时更新进程占用。
- 包管理器事务异常。
- 依赖解析失败。
- 安装失败。
- 重启失败。
- 验证失败。

## 14. 权限、安全与审计

权限建议：

- 首页：View。
- 风险治理：View、Execute、Cancel、Retry、Export。
- 补丁库：View、Add、Edit、Delete、Download、Import。
- 基线管理：View、Add、Edit、Delete、Bind。
- 目标管理：View、Add、Edit、Delete、Test、BindBaseline。
- 设置：View、Edit、Test。

数据权限：

- 基线、目标、评估、风险和治理任务带组织归属。
- Node Manager 目标等跨模块数据均按当前用户授权组织过滤。
- 任务执行时服务端再次校验目标授权。

安全边界：

- 凭据加密存储。
- Windows 执行模板内置。
- Linux 执行行为由后端插件提供。
- 页面不接受临时命令、任意脚本或任意 shell 参数。
- 补丁包执行前校验就绪状态、hash 和签名/可信来源。
- 系统保留主机级执行锁和全局并发上限。

统一操作日志记录：

- 补丁源和 Windows 版本管理变更。
- 补丁导入、下载、处理和删除。
- 基线创建、修改、删除和发布版本。
- 主机绑定或切换基线。
- 立即/周期评估选择。
- 治理任务创建、范围确认、取消和重试。
- 重启和基线变更导致的任务项失效。

## 15. 测试与验收

### 后端重点

- 补丁就绪状态和异常处理。
- Windows 替代 KB、Linux 最低版本和满足证据。
- 基线版本快照及变更后风险失效。
- 主机可不绑定基线及合规率/覆盖率统计。
- 周期、立即评估和旧结果过期。
- 三视角风险查询和状态分布。
- 同一风险项有效任务唯一性。
- 同一补丁不同主机拆分执行窗口。
- 多主机、多补丁任务及依赖顺序。
- 立即执行冲突不排队。
- 窗口任务 FIFO、临时占用等待和窗口结束未执行。
- 执行时预检、环境异常和实际方案重算。
- 窗口内重启预留判断。
- 待重启风险创建重启窗口任务并在完成后验证。
- 取消、部分取消、重试和权限。

### 前端重点

- 首页指标跳转和未配置基线引导。
- 风险治理三个视角及状态分布。
- 补丁库可用/需处理体验。
- 基线固定清单、版本历史和变更扫描选择。
- 目标管理单台/批量绑定基线。
- 提交前范围确认。
- 立即执行和执行窗口表单。
- 执行记录阶段、排队原因和失败处理建议。

### 最小验收场景

1. 15 台未配置主机从首页进入目标管理并批量绑定兼容基线。
2. 绑定后选择立即扫描，主机从待评估进入合规或不合规。
3. 某补丁要求影响 5 台，安排 3 台在 22:00-01:00、2 台在 23:00-02:00。
4. 同一主机的窗口任务按 FIFO 串行；其他主机不受影响。
5. 立即执行遇到已有任务或外部更新进程时不排队，失败项可重试。
6. 一个任务处理多主机、多补丁，自动排序依赖并统一判断重启。
7. Windows 由替代 KB 满足，Linux 由更高版本满足。
8. 基线变更选择按周期扫描，旧结果过期且不进入合规率。
9. 基线切换导致部分计划风险失效，执行时跳过；全部失效则任务自动取消。
10. 窗口剩余时间不足时不自动重启，风险进入待重启。
11. 待重启风险创建重启窗口任务，重启后自动验证。
12. 修复完成后风险已修复且主机恢复合规。
13. 补丁未就绪、磁盘不足、包管理器异常和验证失败均有明确处理入口。
