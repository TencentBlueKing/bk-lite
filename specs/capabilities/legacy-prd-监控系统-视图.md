# 监控系统 · 视图

> Migrated from `spec/prd/监控系统/视图.md` as legacy capability evidence.

## 1. 背景与目标

视图是监控数据的可视化入口，让用户基于指标查询语句和监控对象维度查看实时与历史指标，并提供内置仪表板快速观测常见对象。

## 2. 范围定义

- 指标查询：即时查询与范围查询。
- 维度过滤：按监控对象与维度筛选指标实例。
- 单位换算：自动推荐或手动指定目标单位。
- 专业仪表板：按监控对象提供预置的首屏观测视图，并按分组（硬件设备、主机资源、网络、数据库、中间件）组织导航。

## 3. 关键能力

- 支持以指标查询语句（PromQL）做即时查询和范围查询（指定起止时间与步长）。
- 支持按监控对象、维度过滤查看指标实例。
- 支持指标单位的自动换算与手动指定目标单位。
- 提供覆盖主流监控对象的专业仪表板，当前内置：主机（主机资源），网站、Ping（网络），MySQL、Redis、MongoDB、MSSQL、PostgreSQL、Elasticsearch（数据库），Nginx、Docker、ActiveMQ、Apache、Consul、RabbitMQ、Tomcat、Zookeeper（中间件）。
- 仪表板提供统一的侧边栏导航：按分组折叠展开、支持按对象名/别名搜索、高亮当前对象、可整体折叠。
- 每个对象仪表板以「首屏 P0」组织：在不滚动的首屏内呈现该对象最关键的健康与性能信息，由若干可复用的可视化组件拼装（关键指标卡、趋势折线图、环形占比图、条形/迷你趋势图、状态信号面板、详情面板）。
- 仪表板提供两种展示模式：默认的「仪表板」聚合视图，以及展示该对象全量指标的「指标」视图，可在页面内切换。
- 仪表板支持选择具体实例、调整时间范围、按固定频率自动刷新与手动刷新，并展示采集探针的采集状态及其时间线。
- 关键指标卡可对比上一周期值并标注变化方向；运行时长类指标卡可识别区间内的重启并提示运行状态。
- 各对象在「视图」列表与告警等位置可一键跳转至其对应的专业仪表板。

## 4. 关键规则

- 维度由指标定义驱动，维度组合用于定位指标实例。
- 视图数据受用户组织权限范围约束；专业仪表板访问权限继承「视图」菜单的权限。
- 专业仪表板按对象唯一标识匹配，并支持对象别名（如 activemq/active_mq、postgres/postgresql、zookeeper/zk、host/os/主机资源、website/web/网站）映射到同一仪表板。
- 仪表板组件按数据可用性自适应：指标无数据时关键指标卡显示占位且自动隐藏不可度量的趋势缩略图，标题头的身份信息 chip（如版本号）仅在对应指标有数据时展示。
- 详情面板仅对真实可度量的数值型指标渲染趋势缩略图；状态、枚举、时长及无单位的标识类指标只展示文本值。

## 5. 关键技术架构选择

- 指标统一存储于 VictoriaMetrics，查询语言为 PromQL，视图通过即时/范围查询接口取数。
- 专业仪表板采用统一框架：以注册表集中声明各对象（分组、对象名、别名、权限路径与对应组件），通过共享的可视化组件库与配置驱动的「简单仪表板内核」拼装各对象首屏，新增对象只需提供配置与组件并注册即可。

## 6. 验收标准

- 用户可用 PromQL 完成即时与范围查询，并按对象、维度过滤。
- 单位可按自动或手动方式正确换算。
- 各内置对象的专业仪表板可正常展示对应对象的指标，首屏在不滚动时即呈现关键健康与性能信息。
- 仪表板侧边栏可按分组折叠展开、搜索定位并高亮当前对象，点击可切换不同对象仪表板。
- 仪表板可切换「仪表板」与「指标」两种模式，可选择实例、调整时间范围、自动/手动刷新，并展示采集状态。
- 对象别名可正确匹配到对应仪表板；指标无数据时组件以占位展示且不报错。
