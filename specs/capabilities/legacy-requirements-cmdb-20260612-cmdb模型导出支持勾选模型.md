# CMDB 模型导出支持勾选模型

> Migrated from `spec/requirements/CMDB/20260612.CMDB模型导出支持勾选模型.md` as legacy capability evidence.

日期：2026-06-12

## 背景

当前 CMDB「导出模型配置」是一键全量导出：点击「查看更多 → 导出模型配置」后，后端把全部分类、全部模型、每个模型的属性表与关联表、以及公共枚举库一次性写入 `model_config.xlsx` 下载。

实际使用中，用户往往只想导出其中一部分模型（用于迁移到另一套环境、备份某个子集、或分享给他人），全量导出文件冗余、不便筛选。

本期为导出增加「按模型勾选」能力：用户可在导出前选择要导出的模型，仅导出选中的部分。

## 需求描述

CMDB「导出模型配置」由一键全量导出，升级为「勾选导出」。用户点击导出后，先弹出模型选择弹窗，勾选要导出的模型，再执行导出；导出文件只包含勾选的模型及其配套数据。

### 1）模型选择交互

- 点击「导出模型配置」菜单项，由直接下载改为打开**模型选择弹窗**。
- 弹窗内按**模型分类（classification）折叠树**展示全部模型：分类为父节点、模型为叶子节点。
- 分类节点带复选框，可**整组勾选/取消**；部分子项选中时父节点呈**半选中间态**。
- 顶部提供**搜索框**（按模型名称/ID 过滤）与**全选/反选**。
- 弹窗打开时**默认全选**（沿用原「导全部」习惯，用户按需取消）。
- 底部为「导出」「取消」按钮。一个模型都未选时，「导出」禁用或提示「请至少选择一个模型」。

### 2）导出范围规则

勾选导出时，导出文件各部分按以下规则生成：

- **模型与属性**：只导出 `model_id` 在勾选集合内的模型，及其对应的 `attr-{model_id}` 属性表。
- **分类（classifications）**：只导出勾选模型实际所属的分类。
- **公共枚举库（public_enum_libraries）**：**全量**导出（公共资源，体量小，不做引用筛选）。
- **关联关系（asso-{model_id}）**：逐行过滤，**仅保留源模型与目标模型都在勾选集合内**的关联；另一端未勾选的关联行直接丢弃。某模型过滤后无任何关联，则不生成其 `asso-` 表。

### 3）向后兼容

- 当未传入任何勾选（`model_ids` 为空）时，导出行为与现状完全一致（全量导出），保证存量调用方与回归不受影响。

## 技术设计

### 后端

- `ModelManage.export_model_config(language, model_ids=None)` 增加可选参数 `model_ids`：
  - 为空 → 行为不变，全量导出。
  - 有值 → 按「导出范围规则」过滤 models / attr / classifications / asso；枚举库保持全量。
  - `model_ids` 含不存在的 ID 时跳过，不报错。
- 视图 `export_model_config` 由 `GET` 改为 `POST`，从 `request.data` 读取 `model_ids` 并透传给 service；权限 `model_management-View` 不变。
- 涉及文件：`server/apps/cmdb/services/model.py`、`server/apps/cmdb/views/model.py`。

### 前端

- 新建独立组件 `exportModelConfigModal.tsx`（与 `importModelConfigModal.tsx` 同级，位于 `management/list/`）。
  - 复用页面已有的「分类 + 模型」数据渲染折叠树，无需新增列表接口。
  - 实现折叠树勾选、半选中间态、搜索过滤、全选/反选、默认全选。
- 页面 `management/page.tsx`：将「导出模型配置」菜单项的 `onClick` 由直接下载改为打开弹窗。
- API 层 `web/src/app/cmdb/api/model.ts`：`exportModelConfig` 增加 `modelIds: string[]` 参数，改为 `POST` 携带 `{ model_ids }`，下载文件名仍为 `model_config.xlsx`。

## 适用范围

- CMDB 资产管理「导出模型配置」入口（超管 + Default 分组可见，权限范围不变）。

## 非目标（本期不包含）

- 导出模型实例数据（本功能仅导出模型配置/Schema，不含实例）。
- 勾选模型时自动带入其关联模型或被引用枚举库（即不做依赖自动补全）。
- 对公共枚举库做按引用筛选（本期保持全量）。
- 导入侧改动（导入逻辑不变）。
- 新增模型列表查询接口（复用前端已有数据）。

## 验收标准

- 点击「导出模型配置」弹出模型选择弹窗，按分类折叠树展示全部模型，支持整组勾选、半选中间态、搜索、全选/反选，默认全选。
- 勾选子集导出时：models 表与 attr 表只含勾选模型；classifications 表只含勾选模型所属分类；public_enum_libraries 表全量。
- 关联表仅保留源、目标模型均被勾选的关联行；跨选区关联被丢弃；模型无剩余关联时不生成其 asso 表。
- 一个模型都未选时无法导出，并给出提示。
- 未传 `model_ids`（空）时，导出文件内容与升级前的全量导出一致。
