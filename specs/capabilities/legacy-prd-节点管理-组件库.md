# 节点管理 · 组件库

> Migrated from `spec/prd/节点管理/组件库.md` as legacy capability evidence.

## 1. 背景与目标

组件库管理采集器（数据采集的执行单元）及其配置与安装包，是监控、日志、资产采集能力的来源。

## 2. 范围定义

- 采集器：内置与自定义采集器的管理。
- 采集器配置：采集器在节点上的实例化配置。
- 包管理：采集器安装包的版本管理。

## 3. 关键能力

### 3.1 采集器

- 提供系统内置采集器库，支持自定义采集器。
- 采集器分两类：执行任务型与服务型，按应用模块标签（监控/日志/资产）与系统标签分类。
- 采集器面向特定操作系统与 CPU 架构组合，记录执行路径、参数、默认配置模板等。

### 3.2 采集器配置

- 采集器配置是采集器的实例化，为节点创建具体配置（含模板、节点绑定、环境变量）。
- 支持系统预置配置。
- 支持子配置（细粒度配置单元），可设采集类型、配置类型与排序，并可覆盖父级环境变量。

### 3.3 包管理

- 支持采集器安装包的上传、版本管理与下载。
- 包按操作系统与 CPU 架构维度区分，支持版本识别与去重。

## 4. 关键规则

- 采集器通过名称匹配被具体应用模块消费（监控/日志/资产）。
- 采集器配置与节点通过绑定关系关联，可一对多。
- 子配置的排序决定有序执行。

## 5. 关键技术架构选择

- 采集器配置下发至节点指定目录，由控制器管理其生命周期，经 NATS 异步下发执行。
- 采集器标签决定数据流向的应用模块。

## 6. 验收标准

- 用户可维护内置与自定义采集器，并为节点创建采集器配置。
- 采集器安装包可上传、版本化并下载。
- 采集器配置可正确绑定节点并被对应应用模块消费。
