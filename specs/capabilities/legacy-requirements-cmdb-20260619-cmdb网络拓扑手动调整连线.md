# CMDB 网络拓扑手动调整连线

> Migrated from `spec/requirements/CMDB/20260619.CMDB网络拓扑手动调整连线.md` as legacy capability evidence.

## 1. 背景与问题

CMDB 实例详情的「网络拓扑」Tab（`web/src/app/cmdb/(pages)/assetData/detail/relationships/networkTopo.tsx`）目前是**只读可视化**：以当前设备为中心、按跳数 BFS 展开接口直连，支持分层/力导向/环形布局与导出图片。拓扑里的每条边本质是**接口到接口的 `connect` 关联**（端口级，`model_asst_id=interface_connect_interface`），只是渲染成设备到设备；这些边目前**只能由网络拓扑采集（LLDP/SNMP）自动发现**。

存在的缺口：

1. 采集可能漏发现部分连线（设备不支持 LLDP/SNMP、邻居无法解析等），或采集出错连线，用户**无法在画布上手动补建或纠正**。
2. 画布默认**只展示有连线的设备**，无法把一台采集没覆盖、但确实相连的设备拉进来连线。

本需求在现有网络拓扑 Tab 上新增**手动调整连线**能力：进入「编辑模式」后可拖拽连线、删除连线、把画布外设备加入画布再连线。连线粒度与采集完全同构（端口级 `connect`），数据可与采集结果混排。

经核查：建立「两个接口实例间 `connect` 关联」的接口层管线**已具备**（`createInstanceAssociation` / `deleteInstanceAssociation` 等，列表 Tab 的关联/取消关联已在用），本需求**后端零新增 view**，工作量集中在前端画布编辑交互。

## 2. 需求项

### 2.1 编辑模式（显式开关）

1. 网络拓扑 Tab 工具栏新增「编辑」开关。**默认只读**（现有行为：拖动=平移/移动节点、点对端=逐跳展开、布局切换、导出均不变）。
2. 进入编辑态：启用 X6 `connecting`（设备 hover 出连接桩、可拖拽生边）；「点对端=展开」临时让位为「点边选中」；显示「添加设备」按钮。
3. 退出编辑态：关闭 `connecting`、恢复只读的展开点击；**保留**本次添加的未连线游离孤点（见 2.4）。
4. 权限：进入编辑/连线/加端口需 `Add Associate`，删连线需 `Delete Associate`；无权限时「编辑」按钮禁用或隐藏（复用 `PermissionWrapper` 与后端对象级权限校验）。

### 2.2 拖拽连线 + 端口选择小窗

1. 编辑态下从设备 A 连接桩拖到设备 B 松手 → X6 生成**临时边（虚线、未确认）**并触发拦截。
2. 拦截后弹出**端口选择小窗**（`PortLinkModal`），分源设备端口、目标设备端口两个下拉：
   - 下拉数据源复用 `getAssociationInstanceList(deviceModel, deviceId)`，取 `interface_belong_<model>` 组的 `inst_list`，端口名展示剥掉设备前缀（复用 `stripDevicePrefix`）。
   - 某端**无端口**时，下拉空态内联「+ 新建端口」迷你表单：渲染 `interface` 模型的**必填字段 + 常用选填字段**（如端口别名、ifIndex，最终以 `interface` 模型属性定义为准），提交 `createInstance({model_id:'interface', ...})` 建端口实例，并**同时建该端口到设备的 `belong` 关联**（`interface_belong_<deviceModel>`，保证归属正确、采集与拓扑都认得）；建成后回填下拉并选中。
3. 确认 → `createInstanceAssociation({asst_id:'connect', model_asst_id:'interface_connect_interface', model_id:'interface', 两端接口 inst_id})`；成功后用真实 `relationship_id` 替换临时边、转实线、并入合并图。
4. 取消 / 失败 → 移除临时边，回到拖拽前状态。

### 2.3 删除连线

1. 编辑态点中一条边 → 高亮选中 → 浮出删除入口（边中点小按钮或右键菜单）。
2. 点删除 → 二次确认，文案明确：**「删除后，如果这条连线来自自动采集，下次采集可能会重新发现并加回来。确认删除？」**
3. 确认 → `deleteInstanceAssociation(relationship_id)`，成功后从画布与合并图移除该边。删后变孤点的设备保留在画布、不自动移除。

### 2.4 添加画布外设备（游离节点）

1. 编辑态工具栏「添加设备」→ `AddDevicePanel` 搜索面板：按**网络设备类模型**（有 `interface belong` 的模型）过滤，再按名称/IP 搜实例（复用 `searchInstances`）；支持多选；已在画布上的设备去重/置灰。
2. 选中设备作为 `floatingNode` 落在画布空白处（中心附近避让），**并入渲染节点集合**参与布局切换/导出/视口适配；可加淡标识表示「未连线」（可选）。
3. **游离孤点生命周期**：进出编辑态都保留；仅在**页面刷新 / 离开 / 切换到别的实例**（重新拉取拓扑、组件内存清空）时消失。一旦被连线，`relationship_id` 进合并图，下次加载因有连线自然出现，不再依赖 `floatingNodes`。

### 2.5 前端结构

在现有 `networkTopo.tsx` 上原地扩展，写操作逻辑拆分独立文件、不污染只读渲染管线：

```
relationships/networkTopo/
  EditToolbar.tsx     # 编辑开关 + 「添加设备」按钮
  PortLinkModal.tsx   # 端口选择小窗 + 内联新建端口迷你表单
  AddDevicePanel.tsx  # 画布外设备搜索面板（限网络设备）
  useTopoEditing.ts   # 编辑态 hook：connecting 启停、临时边回滚、连/删/加节点回调
```

### 2.6 异常与边界处理

1. 端口占用：选中端口已存在 `connect` 关联时给**非阻断提示**「该端口已连接 XX，确认继续？」。
2. 自连（A=B）、跨非网络设备：连线前拦截并提示，不生成临时边。
3. 新建端口成功但建连线失败：端口已落库，提示用户重试连线，不回滚端口。
4. 节点上限（100，`NETWORK_TOPO_NODE_LIMIT`）：编辑态加设备同样受约束，超限提示并阻止添加。
5. 重复连同一对端口：后端 `if data not in assos` 天然幂等，前端叠加占用提示。

## 3. 验收口径

1. **编辑开关**：默认只读、行为与现状一致；点「编辑」进入编辑态出现连接桩、「添加设备」按钮、点边可选中；退出恢复只读且未连线游离孤点仍在。
2. **拖拽连线**：A→B 拖拽弹出端口小窗，选两端端口确认后画布出现实线、刷新后仍在（已落库为 `connect` 关联）。
3. **空端口内联新建**：某端无端口时可在小窗内新建端口（含 `belong` 归属），建成后可直接选中并完成连线。
4. **删除连线**：点边删除弹出含「采集可能加回」提示的确认框，确认后边消失、刷新不回（除非被采集重新发现）。
5. **添加设备**：搜索面板按网络设备过滤、可多选加入画布，游离孤点参与布局/导出；连线后持久、未连线在刷新后消失。
6. **权限**：无 `Add Associate` 不能进编辑/连线，无 `Delete Associate` 不能删连线；无权限对端按现有规则过滤。
7. **边界**：端口占用提示、自连/跨非网络设备拦截、节点上限提示、建端口后连线失败提示均生效，无静默失败。
8. **测试**：`useTopoEditing` 状态机（进出编辑态、临时边回滚、游离孤点生命周期）、`PortLinkModal`（端口加载/空态/内联建端口）、`AddDevicePanel`（过滤/去重）前端单测通过；后端如端口列表数据源需新增过滤参数则补对应 service 单测（遵循 `server/docs/testing-guide.md`）。

## 4. 约束与边界

**In Scope**

- `web/src/app/cmdb` 网络拓扑 Tab 新增编辑模式、拖拽连线、删连线、添加设备、端口选择小窗（含内联新建端口）。
- 复用现有后端接口完成连线/删线/建端口/查端口/搜设备，**不新增 view**。

**Out of Scope**

- 编辑现有连线的端口（本期只支持删除 + 重连，不做独立编辑表单）。
- 连线来源区分（`collected` / `manual` 标记）与边的来源视觉样式区分；采集对手动边的覆盖治理。
- 画布节点布局状态持久化（游离孤点不跨刷新保存）。
- 非 `connect` 关联类型、非网络设备的拓扑编辑。
- 后端采集流水线改动。
