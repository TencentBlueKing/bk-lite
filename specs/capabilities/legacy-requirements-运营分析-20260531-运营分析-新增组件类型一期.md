# 运营分析-组件能力一期（KPI 卡片 / 仪表盘 / 事件表）

> Migrated from `spec/requirements/运营分析/20260531.运营分析-新增组件类型一期.md` as legacy capability evidence.

## 1. 背景与问题
运营分析作为可视化中枢，当前组件类型为折线、柱状、饼图、表格、单值、Top-N。结合日常运维高频场景与"日志模块、Grafana"两侧的能力对比，本期聚焦三个最痛的缺口：

- **现有"单值"组件能力弱**：只有一个数字，无趋势、无对比、无阈值染色，无法承担"概览页/大屏首页 KPI 卡片"的标准角色，用户得用"单值 + 折线"两个组件拼一张卡片。
- **缺仪表盘（Gauge）组件**：容量水位、SLA 达成率、磁盘/内存使用率这类"距离阈值有多远"的指标，单值给不出视觉感受。
- **缺事件表（message）组件**：当指标出现异常，用户希望在同一张仪表板看到相关的告警/审计/事件流，目前需要跳出运营分析去其他模块查。

## 2. 业务需求

### 2.1 强化"单值"为 KPI 卡片（升级现有组件）
- **场景**：概览页与大屏首页的关键指标卡片。
- **形态变化**：在现有"单值"基础上扩展为"大号当前数值 + 单位 + 对比值（环比箭头与百分比）+ 可选迷你趋势线 + 阈值染色"。
- **兼容性**：现有已创建的"单值"组件保持原显示；新能力（对比、趋势线、阈值染色）按需开启，不强制升级。
- 组件 code key 与日志模块对齐为 `kpiCard`。

![KPI 卡片示意（图源：Grafana 官方文档）](https://grafana.com/static/img/docs/v66/stat_panel_dark3.png)

### 2.2 仪表盘（Gauge，新增）
- **场景**：CPU/内存/磁盘使用率、SLA 达成率、容量水位等"百分比 + 阈值距离"指标。
- **形态**：刻度盘或圆环 + 中央数值；按阈值分段染色，超阈值时颜色按段切换。
- **配置要点**：最小值、最大值、单位、阈值分段、形态（半圆/圆环）。

![仪表盘示意（图源：Grafana 官方文档）](https://grafana.com/media/docs/grafana/panels-visualizations/screenshot-gauge-visualization-v13.0.png)

### 2.3 事件表（message，新增）
- **场景**：指标异常时同图查看相关事件流；告警概览、变更记录、审计日志的嵌入式展示。
- **形态**：时间倒序排列的列表，每行包含时间戳、级别、来源、消息正文；按级别染色，颜色规范与告警/日志模块一致。
- **战略价值**：是后续把告警中心的告警流、日志模块的日志条目、其他模块的事件，统一通过数据源接入运营分析仪表板的载体。
- 组件 code key 与日志模块对齐为 `message`。

![事件表示意（图源：Grafana 官方文档）](https://grafana.com/media/docs/grafana/panels-visualizations/screenshot-logs-v12.3.png)

## 3. 业务逻辑
- 三类组件均按现有机制绑定数据源、接收统一过滤条与时间范围、按命名空间路由取数。
- 阈值与颜色配置归属组件自身，同一数据源可被不同组件以不同阈值呈现。
- 数据源返回结构与组件期望不匹配时，组件区域显示"数据结构不符"并指明期望形态，不静默渲染空白。
- KPI 卡片对"单值"组件做能力扩展，不新增独立组件类型：用户在"单值"配置中按需启用对比、趋势线、阈值染色。

## 4. 交互
- 在仪表板编辑模式下，"单值"组件配置面板新增"对比方式（环比/无）、迷你趋势线显隐、阈值分段"三类新选项，默认全部关闭以保持现有视觉。
- "仪表盘"作为新组件出现在组件库；配置流程与现有组件一致。
- "事件表"作为新组件出现在组件库；级别染色复用与告警/日志模块一致的颜色规范。
- 阈值分段（KPI 卡片、仪表盘涉及）使用统一的"阈值-颜色"配置形态，避免每个组件一套独立 UI。

## 5. 关键技术选择
- KPI 卡片不新建组件类型，而是扩展现有"单值"组件，避免新旧并存造成用户选择困惑；命名上对外仍可称"KPI 卡片"。
- 事件表作为通用组件接入数据源（不绑定特定模块），以便后续打通告警中心/日志模块时不需要改组件本身。
- 不引入运营分析独有的新类型；命名（code key）与日志模块保持一致：`kpiCard` / `message`。

## 6. 验收口径
- "单值"组件可启用"对比方式（环比/无）"，正确显示对比值与上升/下降箭头。
- "单值"组件可启用迷你趋势线（sparkline）并正确渲染。
- "单值"组件可启用阈值分段染色；超阈值时颜色按段切换。
- 既有"单值"组件在不启用上述新选项时，显示与升级前完全一致。
- "仪表盘"可创建并绑定数据源，最小/最大/阈值分段染色生效。
- "事件表"可创建并绑定数据源，按时间倒序展示，按级别染色生效。
- 三类组件均受统一过滤条、时间范围、命名空间作用。
- 数据源返回结构不匹配时，组件区域显式提示，不渲染空白。
- 三类组件均可正常 YAML 导入导出。

## 7. 约束与边界
**In Scope**：单值组件升级为 KPI 卡片、新增仪表盘、新增事件表。

**Out of Scope**：
- 热力图、柱线组合、散点图、桑基图、状态时间线、Bar Gauge —— 评估为本期非最高优先级，留作后续阶段。
- 日志模块的 76 类采集器专用模板，属日志领域特化，不进入运营分析通用组件库。
- 事件表与告警中心/日志模块的直接打通（本期通过数据源接入即可，模块直连作为后续阶段需求）。
- 跨组件主题统一、组件间联动（Data Links）。
