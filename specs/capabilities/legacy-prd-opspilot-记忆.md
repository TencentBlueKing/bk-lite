# OpsPilot · 记忆

> Migrated from `spec/prd/OpsPilot/记忆.md` as legacy capability evidence.

## 1. 背景与目标

记忆为对话提供跨会话的长期上下文（用户画像、偏好、历史交互），让智能体的回答更连贯、更个性化。

## 2. 范围定义

- 记忆空间：记忆的容器与可见范围，并维护当前可用记忆引擎的配置。
- 记忆条目：记忆内容的存储（本地存储引擎下的落库形态）。
- 配置：本地存储引擎参数、默认模型、写入规则与可见范围。
- 存储引擎：本地存储引擎。

## 3. 关键能力

- 记忆空间作为对话记忆容器，可设可见范围（个人/团队）与写入规则。
- 记忆空间当前由本地存储引擎承载记忆，引擎以统一的读取/写入/删除/连接测试接口对上层透明，对话侧无需感知具体后端。
- 存储引擎：
  - 本地存储：记忆以标题 + 内容形式落库，按更新时间倒序提供最近上下文，并支持基于模型的智能合并写入。
- 引擎参数由后端按本地存储引擎声明的配置项动态描述，前端据此动态渲染配置表单（文本、密码、数字、下拉、JSON 等输入形态），并支持必填校验、默认值填充与连接测试。
- 配置默认模型、写入规则（自动存储的判定条件）与可见范围；写入时可临时指定模型覆盖默认模型。
- 用于在多轮对话间保留长期上下文。

## 4. 关键规则

- 记忆按个人或团队范围隔离。
- 写入规则决定哪些信息被自动沉淀为记忆。
- 记忆空间未指定存储引擎时默认使用本地存储。
- 记忆空间一经创建，其存储引擎不可切换；引擎配置可随时更新。
- 引擎配置中的敏感项（如 API 密钥）落库前加密存储；对外返回时一律脱敏，仅展示前缀加掩码或占位符。
- 个人记忆按用户隔离、团队记忆按组织隔离；同一记忆空间内每个用户/组织各自唯一一条记忆主体。

## 5. 关键技术架构选择

- 记忆采用可插拔引擎架构：引擎注册表统一管理引擎类型，当前仅注册本地存储引擎；记忆空间通过其存储引擎标识解析到具体引擎实例，业务侧仅依赖统一抽象接口。
- 本地存储引擎以关系型数据库承载记忆并通过异步任务完成模型驱动的合并写入。
- 引擎配置参数由当前引擎自描述，驱动后端校验与前端表单渲染；未来新增引擎时无需改动上层逻辑。

## 6. 验收标准

- 用户可创建记忆空间，并填写本地存储引擎配置（含连接测试）与可见范围、写入规则。
- 本地存储引擎下，记忆条目可按空间存储并在对话中提供长期上下文。
- 引擎配置中的敏感字段加密存储，API 返回时脱敏。
- 已创建记忆空间不可切换存储引擎，但可更新引擎配置。
