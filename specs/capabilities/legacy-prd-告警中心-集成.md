# 告警中心 · 集成

> Migrated from `spec/prd/告警中心/集成.md` as legacy capability evidence.

## 1. 背景与目标

集成定义告警中心如何接入外部告警源，把异构来源的事件归一化为统一的事件与告警，是告警数据的入口。

## 2. 范围定义

- 告警源：新增、编辑、删除、启停、连通性测试。
- 接入指引：各类型告警源的接入说明与字段映射。
- 事件接入：通过 Webhook / API 推送或轮询接入事件。

## 3. 关键能力

- 支持多种告警源类型：Prometheus、Zabbix、Webhook、日志、监控、云（多云）、NATS、RESTful。
- 每个告警源可配置名称、类型、连接配置与密钥。
- Webhook 类型对外暴露推送端点接收事件；支持按团队隔离的密钥校验。
- 支持查看各类型告警源的接入指引（接入步骤、字段映射、验证测试、排障）。
- 支持告警源连通性测试、启用/停用与软删除，并记录最近活跃时间。
- K8s 接入强制走"组织密钥"路径：生成部署清单前必须先选择组织并使用该组织的密钥，支持自签证书场景下跳过证书校验，并在密钥变更后重新部署时触发事件采集器滚动重启。

## 4. 关键规则

- 接入事件归一化为标准字段：标题、描述、级别、服务、事件类型（告警/恢复）、指标项、资源、起止时间、标签等。
- 事件按去重键去重，相同指纹窗口内的事件归并到同一告警。
- 告警源按团队隔离，支持多租户密钥映射；事件按上报所用密钥归属到对应组织。
- SNMP Trap 为多租户密钥映射的例外：不参与组织级密钥路由，桥接侧使用告警源级密钥接入，所有事件统一归属到默认组织；该类型不支持新增或重置组织密钥。
- K8s 接入必须传入组织密钥，且该密钥须确为当前 K8s 告警源已配置的组织密钥；缺失或不匹配时拒绝生成部署清单，以避免采集器长期持无效密钥上报而被拒。

## 5. 关键技术架构选择

- 接入管线统一：各源推送/轮询的原始事件经归一化与去重后进入事件，再聚合为告警。
- 支持将告警规则以清单形式导出到 K8s 集群（AlertManager）。

## 6. 验收标准

- 用户可创建、编辑、删除、启停各类型告警源并测试连通性。
- 各源事件能正确归一化并按去重规则归并。
- 接入指引可指导用户完成接入与验证。
- SNMP Trap 事件无论桥接来源均归属到默认组织，且该类型不可配置组织密钥。
- 生成 K8s 部署清单前未选择组织密钥或密钥不属于当前告警源时，系统拒绝并给出提示；自签证书场景可启用跳过证书校验，密钥变更后重新部署能触发采集器重启。
