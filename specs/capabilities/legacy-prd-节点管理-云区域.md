# 节点管理 · 云区域

> Migrated from `spec/prd/节点管理/云区域.md` as legacy capability evidence.

## 1. 背景与目标

云区域是节点管理的组织单元，代表一个独立的区域/环境，下辖节点、环境变量与采集器配置，是采集执行与节点纳管的根容器。

## 2. 范围定义

- 云区域：作为节点、环境、变量的归属容器。
- 节点：主机节点的纳管、控制器安装、状态追踪、组织绑定。
- 环境：云区域级环境变量管理。
- 变量：采集器配置级的变量覆盖。

## 3. 关键能力

### 3.1 节点

- 支持注册与管理主机节点（Linux/Windows），记录 IP、操作系统、CPU 架构与组织绑定。
- 控制器安装支持自动安装（SSH 密钥/密码）与手动安装（生成脚本/命令），并追踪安装进度。
- 追踪节点状态：IP 可达性、控制器运行状态、采集器部署状态。
- 支持节点标签（自定义标签与系统标签）。
- 单个节点可关联多个组织，实现权限隔离。
- 节点可绑定多个采集器配置。

### 3.2 环境

- 云区域级环境变量管理，为该区域所有节点提供统一配置。
- 变量分普通文本与密文（加密存储）。
- 同一云区域内变量键唯一。

### 3.3 变量

- 采集器配置级的环境变量覆盖机制，用于动态注入采集器参数与敏感配置。

## 4. 关键规则

- 节点生命周期：安装控制器 → 节点激活 → 下发采集器配置 → 数据采集。
- 节点与采集器配置均绑定到具体云区域。
- 节点、采集器配置按组织范围隔离权限。

## 5. 关键技术架构选择

- 云区域内节点与服务端通过 NATS 消息队列通信；NATS 连接配置存于云区域环境变量。
- 节点激活后注册并获得鉴权令牌，采集器配置与执行经 NATS 异步下发。

## 6. 验收标准

- 用户可在云区域下注册节点并完成控制器自动/手动安装。
- 节点状态（可达性、控制器、采集器）可正确追踪。
- 云区域环境变量与采集器变量覆盖可正确生效。
- 节点按组织范围正确隔离。
