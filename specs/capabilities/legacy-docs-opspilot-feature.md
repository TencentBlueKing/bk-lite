# 功能介绍

OpsPilot 将 AI 的构建管理链路划分为底层资源、中层逻辑加工和顶层视图触达，以下为您按平台核心导航模块拆解关键功能点。

## 1. 模型 (Provider)
作为全栈智能体的基石，“模型管理”专门用于在企业内部统一注册、存放与管理各家厂商的大模型接口资产。

*   **多维模型纳管**：支持集中接入及启停大语言 LLM、向量化 Embed、结果重排 Rerank、OCR 图像识别等多种模型品类（避免业务人员每次开发都重复配置大模型环境与鉴权信息）。
*   **同供应商批量管理**：优化模型添加交互体验，支持同一供应商模型批量管理集成，大幅降低维护成本。
*   **敏感配置加密**：对接后台的高安全性逻辑控制，自动对类似 API Key 等字段强制加密（解决密码明文在前端暴露或日志渗透的问题）。

## 2. 工具 (Tool)
通过工具管理使得大模型不仅只具有被动回答问题的能力，还能“长出手脚”与您现有的内部企业系统互通。

*   **开箱即用的运维库**：系统内置 Postgres 巡检、K8s 巡检、Web 应用巡检等专业助手工具集。此外提供内建 Redis、Elasticsearch 及 MSSQL 等运维基础工具，帮助运维场景极速落地。
*   **BrowserUse 浏览器操作**：新增的 Browser 自动化工具不仅让智能体能够直接操作网页，更能够在执行时自动生成关键截图，支持使用者可视化查看 Agent 完整的屏幕交互过程。
*   **基于 MCP 协议扩展**：底层强兼容 MCP 协议工具拉取，并支持最新的 streamable 传输类型生态（统一接入海量第三方接口资源）。
*   **上下文参数强约束**：针对工具执行入参设置明确的“文本”、“密码”、“数字”或“布尔值”区分设置（确保安全流转）。

## 3. 知识库 (Knowledge)
知识库是解决大语言模型“凭空捏造”风险的唯一指定解药。企业可以在此集中管理“记忆”。

*   **全态内容与多模态萃取**：不仅支持纯文本解析，更增强了对图片上传及文档中嵌套图片内容的 OCR 识别能力提取（有效消除包含复杂版式的说明文档语料盲区）。
*   **多层次召回体系**：除了常规搜索，更进阶支持基于分块（Chunk）与分段形式的深层召回模型，大幅提高最终 LLM 获取前置材料的精准命中率。
*   **文档预处理可视化**：支持针对文档解析的失败原因进行可视化溯源查询（极大减轻因文档格式非标导致的无预警训练中断问题）。

## 4. 智能体 (Skill)
智能体是一组逻辑单元载体，赋予具体场景下的大模型“执行角色”。

*   **问题主动优化机制**：智能体新增问题意图优化能力，在面临用户模糊提问时可通过内部 LLM 被动补充和优化查询语句再行作答。
*   **即插即拔增强挂载**：可直接勾选 RAG 能力并关联“知识库”，或挂载指定外部工具流执行，确保智能体能够在指定范畴内执行既定操作。

## 5. 工作台 (Studio) 及 ChatFlow
面向交付人员拼装智能体与对外发布服务的核心中枢。

*   **所见即所得画布**：基于可视化节点的 ChatFlow 画布，支持配置包含「意图识别」、「消息通知」等多维能力组件的工作流。
*   **在线即刻测试反馈**：画布原生支持在线调试，使得业务人员可以针对应用构建过程即时输入进行链路校验反馈。
*   **多元触发与多端覆盖**：针对配置好的应用提供三种基础界面触发方式：Web 对话框（AG-UI 协议支持）、移动端访问以及嵌入对话；且后台自动适配基于 API、定时器（包含时区识别与 Cron 表达式）的企业微信、钉钉等全渠道群播触发方式。
*   **全景对话历史回溯**：完整记录并复原用户通过 Chatflow 生成的应用交互链路，帮助管理员通过留存快照迅速定位问题与理解业务意图。
