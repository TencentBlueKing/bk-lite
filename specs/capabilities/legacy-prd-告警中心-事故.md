# 告警中心 · 事故

> Migrated from `spec/prd/告警中心/事故.md` as legacy capability evidence.

## 1. 背景与目标

事故是跨团队的告警响应与协作对象。原始告警经相关性规则聚合关联后形成事故，用于统一跟踪处置进展、协作沟通与复盘。

## 2. 范围定义

- 事故列表：分页查看与筛选。
- 事故详情：关联告警、时间线、资源信息、协作记录。
- 生命周期处置：认领、改派、关闭。
- 协作：观察、进展、结论、下一步等结构化更新与回复。

## 3. 关键能力

- 支持事故列表查看，按状态、级别、时间范围筛选。
- 支持查看事故详情：关联告警、事件时间线、资源信息、协作人与备注。
- 支持事故生命周期操作：认领（待处理→处理中）、改派、关闭。
- 支持添加结构化协作更新（观察、进展、结论、下一步，可带附件）并支持线程化回复。
- 支持将协作更新标记为关键信息。

## 4. 关键规则

- 事故状态生命周期：待处理 → 处理中 → 关闭（也可由待处理直接关闭）。
- 事故由相关性规则（智能降噪 / 缺失检测）从关联告警自动生成。
- 事故按组织（团队）隔离，用户仅可见所属团队的事故。
- 事故通过指纹实现告警的去重与归并。
- 事故持续时长由首条告警时间到最后告警/更新时间计算。

## 5. 关键技术架构选择

- 事故与告警是相互独立的实体：告警是原始事件的聚合，事故是跨团队的响应协作对象，两者状态各自独立。
- 关联告警与事故通过多对多关系绑定。

## 6. 验收标准

- 用户可查看、筛选事故并完成认领、改派、关闭。
- 事故详情可展示关联告警、时间线与协作记录。
- 协作更新支持结构化分类、附件与回复。
- 事故按团队范围正确隔离。
