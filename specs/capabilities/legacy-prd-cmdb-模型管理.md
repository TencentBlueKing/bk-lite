# CMDB · 模型管理

> Migrated from `spec/prd/CMDB/模型管理.md` as legacy capability evidence.

## 1. 背景与目标

模型管理是 CMDB 的数据地基。模型定义决定了资产实例能记录什么、实例之间能建立什么关系、以何种规则保证唯一性。

目标：让模型分类、模型、字段、字段分组、关系、唯一规则、自动关联规则、公共选项库可以统一定义和维护，使全平台的资产口径一致、可复用、可迁移。

## 2. 范围定义

- 模型分类：分类树的新增、编辑、删除、查看。
- 模型：新增、编辑、删除、详情查看、复制。
- 模型属性（字段）：新增、编辑、删除、列表查看。
- 字段分组：新增、编辑、删除、排序、字段归组、折叠状态。
- 模型关系：新增、删除、批量删除、列表查看。
- 联合唯一规则：新增、编辑、删除、查看。
- 自动关联规则：新增、编辑、删除、查看、启停。
- 公共选项库：新增、编辑、删除、列表查看、引用关系查看。
- 模型配置导入导出。
- 模型管理布局：分类与模型的全局可见性切换、拖拽排序（管理模式）。

## 3. 关键能力

### 3.1 模型分类与模型

- 模型归属于分类树，分类支持增删改查。
- 模型支持新增、编辑、删除、详情查看和复制。
- 模型复制至少选择一种复制方式：复制属性（含字段分组、唯一规则）或复制关系（含自动关联规则），两者可同时选择。
- 存在一个默认分组/组织，其下模型对所有用户可见，不受常规权限配置限制。

### 3.2 模型属性（字段）

- 字段类型支持：字符串、整数、枚举、标签、时间、用户、密码、布尔、组织、表格。
- 每个模型默认包含两个基础字段：实例名、所属组织。
- 标签字段：每个模型最多一个标签字段。
- 枚举字段的选项来源有两种：模型内自定义选项，或引用公共选项库。
- 枚举字段支持单选与多选。

### 3.3 字段分组

- 支持字段分组的新增、编辑、删除。
- 支持分组排序、组内字段排序、字段在分组间归属调整。
- 支持分组折叠状态配置。

### 3.4 模型关系

- 关系基于已存在的源模型和目标模型建立。
- 关系类型为预定义集合（如 1:1、1:n、n:n 等）。
- 支持关系的新增、删除和批量删除。

### 3.5 联合唯一规则

- 支持单字段唯一和多字段联合唯一。
- 每个模型最多配置 3 条唯一规则。
- 实例名始终参与唯一性校验。
- 枚举、标签、布尔类型字段不可加入唯一规则。
- 规则保存时会对照现有实例校验是否冲突。

### 3.6 自动关联规则

- 自动关联规则依附在具体的模型关系上，同一关系可配置多条规则，规则可启停。
- 规则以字段对匹配为准，匹配方式支持：精确、忽略大小写、包含。
- 枚举、标签、表格、组织、用户类型字段不参与匹配。
- 命中匹配后自动在对应实例之间建立关系。

### 3.7 公共选项库

- 支持公共选项库的新增、编辑、删除和列表查看。
- 枚举字段可引用公共选项库，统一选项定义。
- 支持查看选项库被哪些模型字段引用。
- 选项库被字段引用时不允许删除。
- 选项库内容更新后，同步刷新到所有引用它的字段及相关实例。

### 3.8 模型配置导入导出

- 支持以 Excel 导出全部模型定义（属性、字段分组、规则、关系）。
- 支持以 Excel 导入模型配置，用于模板复用与跨环境迁移。

### 3.9 模型管理布局（可见性与排序）

- 模型管理页提供「管理模式」，仅超级管理员可进入，用于对分类与模型做全局的可见性切换和拖拽排序。
- 可见性是全局生效的：超管在管理模式下隐藏某分类或模型后，所有用户的浏览视图都不再展示该项。
- 隐藏只是列表过滤，不删除任何数据；管理模式可拉取并展示包含已隐藏项的全量数据，重新设为可见即恢复。
- 排序在两个层级生效：分类之间排序、同一分类内的模型排序，均通过拖拽调整。
- 管理模式下分类与模型的全部改动（顺序 + 可见性）在「保存」时一次性提交；退出管理模式后，浏览视图立即按新顺序与可见性呈现。

## 4. 关键规则

- 删除模型前，必须先清理该模型下的实例和关系，否则不允许删除。
- 字段类型在创建后保持稳定，不直接切换类型。
- 枚举字段创建后，选项来源方式和单选/多选模式保持稳定。
- 联合唯一规则生效后，实例新增和编辑都必须满足唯一约束。
- 模型复制必须至少选择属性或关系中的一种复制方式。
- 公共选项库被引用时不可删除；选项内容变更后引用方保持同步。
- 模型与字段操作均产生模型管理类变更记录（见操作日志）。
- 进入管理模式、拉取全量（含已隐藏）数据、保存布局均仅限超级管理员；非超管即使带参请求也只返回过滤后的可见数据，且无法提交布局。
- 分类的可见性遵循联动规则：分类自身被设为隐藏时不展示；分类下存在模型但全部模型均被隐藏时，该分类在浏览视图中也一并隐藏；不含任何模型的空分类不受此联动影响。
- 可见性与排序对历史数据向后兼容：未设置过的旧分类排序默认排在最后、旧模型排序默认置 0，旧分类与旧模型默认均为可见。

## 5. 关键技术架构选择

- CMDB 采用图数据库 FalkorDB 作为模型、实例与关系的统一存储，关系天然以图的边表达，便于关系查询与拓扑展开。
- 公共选项库的选项变更采用异步任务同步到引用方，库与引用方之间为最终一致。
- 模型配置以 Excel 作为导入导出与跨环境迁移的载体。
- 分类排序/可见性、模型排序/可见性作为图节点属性持久化；浏览视图的过滤与排序在服务端完成。
- 保存布局时图数据库无原生事务，采用「先快照分类现状 → 写分类 → 写模型」的策略，模型写入失败则回滚分类到快照状态；布局保存为幂等操作，可安全重试。

## 6. 验收标准

- 产品管理员可独立完成模型分类、模型、字段、字段分组、关系、联合唯一规则、自动关联规则的配置。
- 模型复制能按所选方式正确复制属性与关系，并保持唯一规则、自动关联规则一致。
- 联合唯一规则、自动关联规则在实例侧按定义生效。
- 公共选项库可被枚举字段引用，被引用时无法删除，选项更新后引用方同步。
- 模型配置可完整导出并在另一环境导入还原。
- 删除模型在存在实例或关系时被正确拦截。
- 超管可进入管理模式，对分类与模型完成拖拽排序与可见性切换并保存；保存后浏览视图对所有用户即时生效。
- 被隐藏的分类与模型不出现在普通用户的浏览视图中，但数据保留，重新设为可见后可恢复。
- 分类下模型全部隐藏时，该分类在浏览视图中同步隐藏。
- 非超管无法进入管理模式、无法拉取已隐藏数据、无法提交布局。
