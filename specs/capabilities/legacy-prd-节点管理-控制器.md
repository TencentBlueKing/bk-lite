# 节点管理 · 控制器

> Migrated from `spec/prd/节点管理/控制器.md` as legacy capability evidence.

## 1. 背景与目标

控制器是部署在节点上的代理，负责执行安装指令、管理采集器生命周期并与服务端通信，是节点纳管与采集执行的基础。

## 2. 范围定义

- 控制器定义：按操作系统与 CPU 架构定义控制器。
- 控制器安装与卸载：自动/手动安装、进度追踪、卸载清理。
- 包管理：控制器安装包的版本管理。

## 3. 关键能力

### 3.1 控制器定义

- 按操作系统（Linux/Windows）与 CPU 架构定义控制器，记录名称、描述与版本获取方式。
- 支持系统预置控制器。

### 3.2 控制器安装与卸载

- 自动安装：服务端生成引导脚本/命令，节点执行后反向下载并启动，支持 SSH 密钥与密码认证。
- 安装过程分步骤追踪进度，支持超时保护与失败重试。
- 手动安装：获取安装脚本由管理员在节点手动执行后确认状态。
- 删除节点时异步执行控制器卸载清理。

### 3.3 包管理

- 支持控制器安装包的上传、版本管理与下载。

## 4. 关键规则

- 每个操作系统与 CPU 架构组合对应一个控制器。
- 安装任务以"任务 → 节点级进度"的方式记录步骤日志与失败信息。

## 5. 关键技术架构选择

- 控制器作为节点侧代理，通过 NATS 订阅指令并回复结果，承担采集器的下发与执行。
- 安装过程以流式事件实时推送进度。

## 6. 验收标准

- 用户可完成控制器的自动与手动安装，并实时查看安装进度。
- 控制器安装包可上传、版本化并下载。
- 删除节点时控制器可被正确卸载清理。
