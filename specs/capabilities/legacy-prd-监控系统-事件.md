# 监控系统 · 事件

> Migrated from `spec/prd/监控系统/事件.md` as legacy capability evidence.

## 1. 背景与目标

事件是监控的告警中枢：用户通过策略定义阈值与无数据条件，系统周期检测指标并生成、聚合、通知告警，并可基于模板快速创建策略。

## 2. 范围定义

- 告警：列表、筛选、详情、状态处理、通知记录查看。
- 策略：新增、编辑、删除、启停、导入导出。
- 模板：基于监控对象与插件的预置策略模板。

## 3. 关键能力

### 3.1 告警

- 支持按策略、监控实例、告警级别、告警状态、告警类型筛选告警列表。
- 支持查看告警详情，包括全生命周期的指标快照与事件序列。
- 支持更新告警状态、指定处理人、添加备注。

### 3.2 策略

- 支持监控策略的新增、编辑、删除、启停。
- 支持指定监控对象、采集类型与插件，配置告警条件：阈值、聚合算法、分组维度、查询周期。
- 支持无数据告警配置（无数据周期、级别、恢复周期），独立于阈值告警周期。
- 支持配置通知方式、通知人与通知开关。
- 支持指定策略适用范围（资源过滤）与所属组织。
- 支持策略模板的导入导出。
- 策略的 `organizations` 字段在新增、编辑、删除、启停前需通过对象级权限校验；批量模板创建需对所有 `asset_ids` 做授权预校验，越权整体回滚并不写入错误日志。

### 3.3 模板

- 提供按"监控对象 + 插件"组合的预置策略模板，每种组合至多一个模板。
- 支持基于模板快速创建策略。

## 4. 关键规则

- 执行周期为策略检测频率，数据周期为每次检测的数据窗口，两者独立。
- 阈值对比基于计算单位；支持 >、<、=、!=、>=、<= 操作符。
- 告警类型分为阈值告警与无数据告警。
- 告警状态在新告警、已关闭、已恢复之间流转；同一"策略 + 监控实例 + 指标实例"聚合为一条告警。
- 告警级别按权重排序：致命 > 严重 > 预警 > 提示。
- 恢复条件为连续若干周期不满足阈值后自动恢复。
- 策略与告警按组织隔离，权限由团队与监控对象控制。
- 告警的创建、升级、恢复、关闭会向告警中心与配置的通知渠道通知。

## 5. 关键技术架构选择

- 指标存储于 VictoriaMetrics，策略检测由 Celery 定时任务周期扫描，支持漏跑补偿（最多 30 个周期、24 小时内）。
- 告警原始数据与快照以压缩 JSON 存于对象存储。
- 通知经系统管理的通知渠道统一分发。

## 6. 验收标准

- 用户可创建策略、配置阈值与无数据条件、设置通知并启停。
- 系统按周期检测指标并正确生成、聚合、恢复、关闭告警。
- 告警详情可查看指标快照与事件序列。
- 可基于模板快速创建策略。
