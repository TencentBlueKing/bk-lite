## 一、模块定位

MLOps 提供从数据集管理、训练任务编排到能力发布与在线推理的统一闭环，覆盖六类运维算法场景（异常检测、时序预测、日志聚类、文本分类、图片分类、目标检测）。每个场景统一提供数据集、训练任务、能力发布、算法配置四大功能域；训练观测对接 MLflow，模型服务以容器化运行时承载并支持在线推理。本清单仅列已实现能力，不含算法实现细节与演进展望。

## 三、能力边界与约束

算法配置在"算法类型 + 算法标识"维度唯一，数据集发布版本在"数据集 + 版本号"维度唯一，服务 / 训练任务 / 数据集均按场景隔离。训练任务状态统一为待训练 / 训练中 / 已完成 / 训练失败，发布版本状态统一为待发布 / 发布中 / 已发布 / 发布失败 / 归档。服务状态分用户意图状态与容器运行状态两个口径，仅容器实际运行时可进入 active 语义；服务变更涉及模型版本或训练任务时须按约束重启生效。训练数据文件更新须清理旧文件，训练配置在数据库与对象存储间保持同步。本模块不含算法实现代码与训练框架底层细节，不提供外部数据治理 / 标签平台的独立产品能力，不定义监控 / 日志 / 告警等非 MLOps 业务页面。

## 四、平台协同

MLOps 训练观测对接 MLflow 进行运行记录、指标与模型版本管理；模型服务以容器化运行时承载，经统一运行时通道（webhookd）执行启停并回传容器实时状态。日志聚类、异常检测、时序预测等场景产出的能力可经在线推理供监控、日志、告警等模块消费。权限动作（View/Add/Edit/Delete/Train/Start/Stop/Predict）与系统管理的 RBAC 体系一致。
