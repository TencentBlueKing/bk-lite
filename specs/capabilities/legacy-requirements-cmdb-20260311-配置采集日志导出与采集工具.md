# 配置采集日志导出与采集工具

> Migrated from `spec/requirements/CMDB/20260311.配置采集日志导出与采集工具.md` as legacy capability evidence.

日期：2026-05-08（更新）

## 背景

在社区场景中，用户进行配置采集时，常出现"采集失败但无法快速定位原因"的问题。当前流程下，用户需要自行排查协议连通性、凭据可用性与采集返回内容，排障效率低，且难以将原始证据提供给原厂进行优化与修复。

为提升社区助力调试效率，CMDB 需要提供"采集工具"能力：在采集任务失败时，用户可快速跳转到对应协议工具页，自动带入部分任务上下文，手动补全当前诊断所需参数后执行采集，查看原始返回日志并导出文本文件。

## 需求描述

CMDB 一期在配置采集的"SOID特征库"下新增"采集工具"页面，支持 SNMP、IPMI 两类工具的参数填写、原始数据采集、日志查看与文本导出。

### 1）页面与入口

- 在"SOID特征库"下新增一级页面"采集工具"。
- "采集工具"作为独立一级页面新增，不与现有 SOID 页面合并为页面组。
- "采集工具"页面一期提供两个协议类型入口：SNMP 工具、IPMI 工具。
- 当配置采集任务状态为失败时，先在任务失败摘要区域提供"前往采集工具"快捷入口。
- 快捷入口点击后，自动跳转至与失败任务协议类型一致的工具页。

Nano Banana 提示词：

- 设计一个 CMDB「SOID特征库」下的「采集工具」页面原型。页面包含四个协议标签页：SNMP、IPMI、SMI-S、CLI。每个标签页由三部分组成：左侧为凭据与连接参数表单，中间为执行控制区（执行按钮、暂停按钮、执行状态），右侧为原始日志展示区与“导出文本”按钮。整体风格企业级、信息密度中等、优先突出排障效率。
- 在同一原型中补充“采集任务失败场景”联动：任务报错区域出现“前往采集工具”入口，点击后跳转到对应协议标签页，并自动带入任务上下文（任务标识、目标地址、协议类型、凭据引用）。自动带入的凭据字段以脱敏形式显示且可编辑。
- 表单交互要求：按协议动态显示必填项。SNMP 支持 v2/v2c/v3，v3 根据现有采集任务的安全级别联动鉴权/加密字段；CLI 支持密码认证与密钥认证二选一；SMI-S 支持 HTTP/HTTPS 且 HTTPS 展示证书校验开关；IPMI 支持可选 cipher suite。
- 结果区交互要求：点击“执行”后展示原始返回日志（成功或失败都可展示已有返回），并支持导出 `.txt` 文本文件。失败时在结果区顶部展示可读错误摘要（例如认证失败、网络不可达、协议参数错误），避免仅显示通用失败提示。

### 2）失败任务快捷跳转与上下文带入

- 仅失败任务支持"前往采集工具"快捷跳转。
- 跳转后根据失败任务协议类型自动进入对应工具页。
- 跳转后尽量自动带入失败任务中可解析出的字段，例如目标地址、端口、协议版本、接入点及其他可回填参数，用户可在工具页中修改。
- 若任务缺失必要上下文（例如协议类型为空），页面需提示"无法自动带入，请手动补全参数"，并允许用户继续手动执行，无需细分缺失原因。

### 3）参数填写规则

- 一期跳转到采集工具页后，应尽量自动带入失败任务中可稳定解析出的诊断参数；对无法获取或不宜回填的字段，再由用户手动输入。
- 页面中所有执行参数均以当前页面输入为准。
- 用户修改参数后执行采集，以页面当前输入为准。

### 3.1）一期协议参数录入项

#### 3.1.1 SNMP 参数录入项

- 必填：目标地址、端口。
- 必填：接入点（Stargazer 执行节点，下拉选择）。
- SNMP 参数填写方式应尽量复用现有 SNMP 采集任务表单逻辑与字段命名。
- 必填：SNMP 版本（v2/v2c/v3）。
- 当版本为 v2/v2c 时：必填 Community。
- 当版本为 v3 时：
	- 必填：用户名（username）、安全级别（authNoPriv / authPriv）。
	- 必填：鉴权算法（integrity）、鉴权密码（authkey）。
	- 当安全级别为 authPriv 时：必填加密算法（privacy）、加密密码（privkey）。

#### 3.1.2 IPMI 参数录入项

- 必填：目标地址、端口。
- 必填：接入点（Stargazer 执行节点，下拉选择）。
- 必填：用户名、密码。
- 选填：Cipher Suite（加密套件）。
- `privilege` 作为前端可选输入项提供；未填写时，执行层默认使用 `administrator`。

#### 3.1.3 Timeout 说明

- 页面不提供 Timeout 输入项，超时时间由系统后端按操作类型写死，用户无需感知。

### 3.2）执行链路约束

- 一期协议工具默认执行主链路为"CMDB 经 NATS 调用 Stargazer 执行采集并回传结果"。
- 测试连通性与原始采集均复用同一条"CMDB 经 NATS 调用 Stargazer"主链路执行。
- 一期不提供 CMDB 直连目标设备的兜底执行模式。
- 当失败任务跳入采集工具时，默认带入原任务中可稳定解析出的上下文字段，用户可按当前排障需要继续修改。
- 一期工具执行采用异步提交 + 轮询结果模式，用户在当前页面查看执行进度与最终诊断结果。

### 4）一期采集工具能力

#### 4.1 SNMP 工具

- 支持填写或自动带入 SNMP 参数。
- 提供三类操作按钮：
	- **测试连通性**：对固定 OID `1.3.6.1.2.1.1.1.0` 执行轻量 `snmpget`，快速判断网络与凭据是否可用，并返回单值 raw log。
	- **获取OID数据**：弹窗让用户输入指定 OID，对该 OID 执行 `snmpbulkwalk`，返回该 OID 范围内的 raw log。
	- **获取原始数据**：对固定根 OID `1.3.6.1.2.1` 执行 `snmpbulkwalk`，返回完整 raw log。
- 支持展示 SNMP 诊断 raw log。
- 支持将 raw log 导出为文本文件。

#### 4.2 IPMI 工具

- 支持填写或自动带入 IPMI 参数。
- 支持测试 IPMI 连通性，通过轻量指令快速判断 BMC 可达性与凭据是否可用。
- 支持执行 IPMI 原始数据采集。
- IPMI 工具页可选填写 `privilege`；未填写时，执行层默认使用 `administrator`。
- 支持展示 IPMI raw log。
- 支持将 raw log 导出为文本文件。

### 5）日志展示与文本导出

- 采集执行完成后，页面展示本次执行的 raw log。
- 导出文件格式为文本文件（`.txt`），内容仅包含本次执行的 raw log。
- 每次执行结果独立展示与导出，不覆盖历史已下载文件。
- 采集失败时仍可导出失败日志文本（若存在返回内容）。

### 6）错误提示与可用性

- 执行失败时，页面需给出可读错误提示，至少包含失败阶段与错误摘要。
- 当网络、认证或协议参数异常时，提示信息应可用于用户二次排查，不得仅返回通用"执行失败"。
- 从失败任务跳入后，页面需尽量减少用户重复输入，但一期默认仍允许用户手动补全诊断所需参数后再执行。
- 当执行超过系统设定时限时，页面需通过轮询拿到超时结果，并允许用户调整参数后重新执行。

## 适用范围

- CMDB：SOID特征库下"采集工具"页面新增。
- CMDB：配置采集任务失败场景下的快捷跳转。
- CMDB：SNMP/IPMI 两类工具的连通性测试、原始采集与文本导出。

## 非目标（本期不包含）

- 在一期页面中显示或开放 SMI-S、CLI 工具入口。
- 对采集引擎实现细节进行重构。
- 提供 CMDB 直连目标设备的兜底执行模式。
- 在失败任务跳转场景中自动填充全部凭据与诊断参数。
- 页面暴露 Timeout 设置项。
- 引入非文本格式导出（如 PDF、Excel、压缩包）。
- 构建自动修复或自动重试编排能力。

## 验收标准

- 在 SOID特征库下可见"采集工具"页面，并可访问 SNMP、IPMI 两类工具页。
- "采集工具"以独立一级页面提供，不与现有 SOID 页面合并为页面组。
- 页面不显示 Timeout 输入项。
- 配置采集任务失败时，任务失败摘要区域存在"前往采集工具"入口。
- 点击失败入口后可跳转至对应协议工具页，并尽量自动带入失败任务中可解析出的字段。
- 用户修改预填值后，以页面当前输入为准发起执行。
- 一期工具均有明确参数录入项校验，且按协议版本动态要求必填字段。
- SNMP 工具的字段命名与联动方式与现有 SNMP 采集任务表单保持一致。
- SNMP 工具提供"测试连通性"按钮，可对固定 OID 执行轻量探测并展示结果。
- SNMP 工具提供"获取OID数据"按钮，弹窗输入 OID 后可执行 `snmpbulkwalk` 并展示该 OID 范围内的 raw log。
- SNMP 工具提供"获取原始数据"按钮，可对固定根 OID `1.3.6.1.2.1` 执行 `snmpbulkwalk` 并展示 raw log。
- IPMI 工具提供"测试连通性"按钮，可快速验证 BMC 可达性与凭据有效性。
- IPMI 工具支持填写 cipher suite（选填），可获取原始数据并展示，且支持导出文本文件。
- IPMI 工具不要求用户填写 `privilege` 仍可完成诊断执行。
- 执行成功或失败后，若有返回内容，均可导出本次原始日志文本。
- 当自动带入字段缺失时，页面给出明确提示并允许用户手动补全后执行。
- 错误提示包含可读的失败摘要，不仅为通用错误文案。
- 一期 SNMP/IPMI 工具默认通过"CMDB 经 NATS 调用 Stargazer"主链路执行诊断请求。
- 一期工具执行采用异步提交 + 轮询结果模式；超时场景有明确结果反馈。
