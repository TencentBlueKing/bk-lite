# OpsPilot · 工具

> Migrated from `spec/prd/OpsPilot/工具.md` as legacy capability evidence.

## 1. 背景与目标

工具是智能体可调用的函数能力，让对话不止于问答，还能查询数据、执行动作，扩展智能体的行动边界。

## 2. 范围定义

- 内置工具：平台预置的函数工具，含数据库/缓存查询、监控数据查询、附件文件生成、shell/python 命令执行等。
- 自定义工具：用户定义的函数工具。
- 工具管理：工具的注册与绑定。
- 工具执行安全约束：命令执行类工具的高危操作黑名单。

## 3. 关键能力

- 工具为面向智能体执行的函数调用能力。
- 内置工具：如 MySQL、MSSQL、Oracle、Redis 等参数化的数据库/缓存查询工具。
- 监控数据查询工具：内置一组只读的监控查询能力，可列举监控对象与实例、列举对象/实例指标、查询指标数据、查询当前活跃告警与历史告警片段，供智能体在对话或工作流中获取监控现状。
- 附件文件生成工具：在工作流执行中根据传入内容生成可下载的附件文件，支持 Markdown、PDF、Word（docx）三类格式，生成后返回下载链接，作为工作流产出物交付给用户。
- 命令执行工具：内置 shell 命令执行与 Python 代码执行能力，用于自动化运维、信息获取等场景，受高危操作黑名单约束（见关键规则）。
- 自定义工具：用户按结构化定义（名称、说明、参数、回调）创建。
- 工具以可复用集合的形式注册，并以列表绑定到智能体。

## 4. 关键规则

- 工具按组织隔离。
- 工具通过结构化定义供智能体按函数调用方式使用。
- 附件文件类型限制：仅支持 Markdown、PDF、Word（docx）三类，传入不支持的类型直接报错拒绝生成。
- 附件去重与覆盖：同一次工作流执行内，同一来源节点重复生成时按既有标识更新覆盖，旧文件随之清理，避免产出重复附件。
- 附件留存期：生成的附件文件默认留存 3 天，超期由后台任务自动清理。
- Shell 命令执行黑名单：禁止执行高危及敏感命令，包括但不限于 rm -rf、dd、mkfs、curl/wget 等外联下载、env/printenv 等环境变量读取、读取系统口令与 SSH 授权密钥等敏感文件、python -c / sh -c / bash -c / cmd /c / powershell 等内联脚本执行、sudo/su/passwd 等提权命令；命中黑名单或命令为空时拒绝执行并返回错误。
- Python 代码执行限制：禁止 import 语句以及 eval、exec、__import__ 等高危调用，仅允许在受限的安全内置函数白名单内运行，命中限制即拒绝执行。

## 5. 关键技术架构选择

- 工具以函数调用（function-calling）形式集成进智能体执行流程。
- 附件文件以平台对象存储持久化，并通过下载链接对外交付；文件渲染按类型分别生成（Markdown 直出文本、PDF 与 Word 服务端排版）。
- 命令执行类工具的安全约束在执行前做静态校验：shell 侧基于命令模式匹配，Python 侧基于代码语法树解析，校验通过后才进入执行。

## 6. 验收标准

- 用户可使用内置工具并创建自定义工具。
- 工具可绑定到智能体并在对话中被正确调用。
- 监控数据查询工具可列举监控对象/实例、查询指标数据与告警，并向智能体返回结果。
- 附件文件生成工具可在工作流中生成 Markdown/PDF/Word 附件并返回可访问的下载链接；不支持的类型被拒绝。
- Shell 与 Python 命令执行工具命中高危/受限操作时拒绝执行并返回明确错误，合法命令可正常执行并返回结果。
