# CMDB 变更历史场景标记与默认推荐

> Migrated from `spec/requirements/CMDB/20260518.CMDB变更历史场景标记与默认推荐.md` as legacy capability evidence.

日期：2026-05-18

## 背景

当前 CMDB 历史记录只有底层操作流水，缺少“这次修改属于什么业务场景”的表达。

用户希望在实例历史中优先看到设备主线变化，例如归属调整、责任转移、状态流转、关键关系变化；而不是被采集任务调整、自动同步写回、普通属性编辑等低信号变化淹没。

本期场景判断以“修改入口 / 修改来源”为主，不以模型字段治理为前提。

## 需求描述

CMDB 新增“历史场景标记与默认推荐”能力。系统优先根据修改入口或来源给变更记录打场景；在少量语义不充分的入口中，系统给出默认推荐，用户可轻量修正。最终场景结果需写入变更记录，用于历史展示与筛选。

### 1）首期支持的历史场景

- 设备流转 `device_lifecycle`
- 关系变更 `relation_change`
- 普通属性变更 `ordinary_attribute_change`
- 自动采集/自动化变更 `collect_automation_change`
- 模型管理变更 `model_management_change`

### 2）固定场景入口

以下入口首期直接写固定场景，用户不可修改：

- 自动采集、自动同步、自动发现等自动化链路 → 自动采集/自动化变更
- 关系绑定、解绑、迁移等关系操作入口 → 关系变更
- 模型管理、模型属性管理、模型结构调整等入口 → 模型管理变更
- 专门的设备流转入口（如移交、上下线、回收、报废等）→ 设备流转

### 3）默认推荐与轻量修正

对于无法仅靠入口直接确定场景的入口，系统给出默认推荐。

首期典型入口包括：

- 通用实例属性编辑页
- 批量编辑页
- 批量导入或批量更新确认页

要求：

- 默认场景需自动选好，用户多数情况下无需干预
- 场景修正不是通用必填项
- 场景修正不应打断所有操作流程
- 首期仅允许在少量相邻场景之间修正，不开放任意切换

### 4）变更记录要求

- 变更记录需保留最终场景结果，用于历史默认展示与筛选
- 变更记录需保留来源分类，用于区分人工、自动采集、系统、导入、同步等来源
- 场景表示“这次变化属于什么业务含义”，来源表示“这次变化从哪里来”

### 5）历史默认展示

- 实例历史默认优先展示高信号场景
- 首期默认展示至少覆盖：
  - 设备流转
  - 关系变更
- 自动采集/自动化变更默认不与设备主线同优先级混排
- 普通属性变更保留可查看能力，但不要求默认与设备流转并列展示
- 模型管理变更不混入实例主线历史视图

## 适用范围

- CMDB 变更记录列表与详情
- 实例历史页
- 实例属性编辑页
- 批量编辑页
- 批量导入或批量更新确认页
- 关系操作入口
- 自动采集、自动同步、自动发现链路
- 模型管理与模型属性管理入口

## 非目标（本期不包含）

- 在实例数据中新增“历史场景”字段
- 将模型字段标签或字段元数据治理作为首期主方案
- 要求用户在所有编辑操作中手动选择场景类型
- 对所有入口开放任意场景切换能力
- 引入复杂规则引擎或用户自定义场景体系

## 验收标准

- 系统能够将变更记录归入 5 类场景：设备流转、关系变更、普通属性变更、自动采集/自动化变更、模型管理变更
- 自动采集、关系操作、模型管理、专门流转入口能够直接写入固定场景，且用户不可修改
- 通用实例编辑等入口能够获得默认场景推荐
- 在少量高价值歧义入口中，用户可以对默认推荐结果进行轻量修正
- 场景修正能力不是通用必填项，不会在所有编辑操作中强制展示
- 实例历史默认视图中，高信号场景优先于低信号场景展示
- 自动采集/自动化变更不会与设备流转主线默认同优先级混排
- 模型管理变更不会混入实例主线历史视图
