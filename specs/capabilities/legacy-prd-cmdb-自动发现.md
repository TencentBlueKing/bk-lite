# CMDB · 自动发现

> Migrated from `spec/prd/CMDB/自动发现.md` as legacy capability evidence.

## 1. 背景与目标

自动发现负责把外部环境中的真实资产自动采集进 CMDB，并与已有数据比对合并，减少人工录入与口径漂移。

目标：让采集任务的配置、执行、状态查看和结果合并形成闭环，并通过 SOID 特征库和采集工具支撑网络设备识别与协议联调。

## 2. 范围定义

- 采集对象树：按插件分类浏览可采集对象。
- 采集任务：新增、编辑、删除、列表、详情。
- 任务执行：手动执行与周期执行、并发保护、状态流转。
- 任务状态概览：按插件汇总与总体概览。
- 采集结果合并：新增、更新、删除、关联的比对与写回。
- IP 地址管理联动：对子网执行 IP 发现并回写地址台账与利用率。
- SOID 特征库：sysObjectID 映射维护。
- 采集工具：协议采集的独立联调诊断。
- 插件说明：采集插件文档查看。

## 3. 关键能力

### 3.1 采集对象树与插件

- 按插件分类浏览可采集对象（如主机、数据库、中间件、K8S、云资源、SNMP 网络设备、网络设备配置文件、IP 地址管理等）。
- 采集驱动分两类：协议采集与脚本采集。
- 支持查看采集插件的说明文档。

### 3.2 采集任务

- 支持采集任务的新增、编辑、删除、列表和详情查看。
- 任务名称在同一驱动类型与模型下唯一。
- 采集范围两种模式：选择已有的 CMDB 模型实例（推荐），或选择子网范围执行 IP 发现（用于回填 IP 地址台账与利用率）。
- 任务可绑定一个接入点（节点），通过该接入点代理执行。
- 同一采集任务可为采集目标配置一组凭据（凭据池），最多 3 组；同一任务内各组凭据的字段结构必须保持一致，密钥类字段以掩码展示。
- 凭据池支持新增、编辑、删除与排序；编辑时按新增/删除/修改进行差异比对，被删除的凭据其历史命中状态一并清除（仅调整顺序不视为修改）。
- 网络设备配置文件采集面向已纳管的网络设备实例，任务中可维护登录账号、登录口令、特权模式口令、目标命令与配置名称，采集结果归档为配置文件版本。

### 3.3 任务执行

- 支持手动执行和周期执行。
- 同一任务执行中不允许重复触发。
- 执行状态在未开始、执行中、成功、失败、部分成功、超时、强制停止等之间流转。
- 任务可配置数据清理策略（默认不清理，可设置过期天数清理旧数据）。
- 当任务配置了多组凭据时，执行入口启用多凭据派发：先把任务范围拆分为可连接目标，再为每个目标逐一尝试凭据，命中即采集成功并记录该目标命中的凭据。
- 多凭据派发支持的任务类型：主机、数据库、中间件、配置文件、SNMP 网络设备及通用协议采集；仅配置单组凭据或不在支持范围内的任务沿用原单凭据采集链路。
- IP 发现任务以选定子网为边界，周期执行时按本轮在线结果回写 IP 台账，并同步刷新子网利用率。

### 3.4 凭据命中、冷却与重试

- 命中状态按"目标 × 凭据"记录为未探测、成功、已知失败三态；某目标命中某组凭据成功后，会记住该凭据并在后续执行中优先复用，同目标其他凭据的成功态会被重置为未探测。
- 凭据类失败（认证、口令、社区字符串等鉴权相关错误）会切换到下一组凭据继续尝试。
- 进入冷却的失败有两类：凭据类失败，以及连接不可达类失败（目标拒绝连接、TCP 连接/拨号超时、路由不可达等"连不上目标主机"的情况）。两类都按连续失败次数递增冷却时长（1 小时、4 小时、24 小时），冷却期内该目标不再被探测与执行，到期后重新参与。
- 不纳入冷却的失败：接入点无响应、已建立连接但脚本执行超时（慢主机而非不可达）等"非目标主机不可达"的情况，以及其它任务类失败——这些不冷却，下轮照常重试。
- 冷却不区分单凭据与多凭据任务：单凭据任务的失败同样进入冷却并在下轮被跳过；多凭据任务在此基础上还会切换到未冷却的其它凭据。
- 失败态以采集结果中的失败标识准确判定，失败不会被误记为命中成功。
- 当某目标的全部可用凭据都处于冷却时，本轮直接跳过该目标、不再探测与下发。

### 3.5 任务状态概览

- 按插件维度汇总各任务的成功、失败、执行中、部分成功数量。
- 提供总体概览：任务总数、正常/异常/部分成功数、最近执行时间、覆盖模型数。

### 3.6 采集结果合并

- 采集数据与 CMDB 已有实例比对，归类为新增、更新、删除、关联四种结果。
- 周期（自动）执行走完整的新增/更新/删除流程；手动（一次性诊断）执行只做更新，不做新增与删除。
- 满足自动关联规则的结果会自动建立关系。
- 采集合并产生变更记录，归入自动采集场景。
- 配置文件采集会把命中的网络设备命令输出归档为配置文件版本，供资产详情查看历史与差异。
- IP 发现只更新自动发现产生的地址记录；手工维护的地址条目不被覆盖。

### 3.7 SOID 特征库

- 维护 sysObjectID 到网络设备类型、品牌、型号的映射，支持新增、编辑、删除、列表查询。
- OID 保持唯一；区分内置条目与用户自定义条目。
- SNMP 网络设备采集时据此自动识别设备分类。

### 3.8 采集工具

- 面向协议采集的独立联调诊断，不依赖采集任务的增删改。
- 支持对目标（主机、端口、凭据）发起一次性诊断并查询结果。
- 支持从已有任务预填诊断参数，凭据以掩码展示并在执行时还原。

## 4. 关键规则

- 任务名称在同一驱动类型与模型下不可重复。
- 同一任务执行中不允许并发触发。
- 手动执行仅更新已知实例，不新增、不删除；周期执行才执行完整的新增/更新/删除合并。
- 关联结果按自动关联规则建立，重复关联视为成功。
- 采集凭据不能为空，单任务最多 3 组，且同一任务内各组凭据字段结构必须一致。
- 多凭据派发在凭据类失败时切换下一组凭据；命中成功的凭据被记忆并优先复用。凭据类失败与连接不可达类失败均按 1/4/24 小时阶梯冷却，冷却同时覆盖单凭据与多凭据任务；接入点无响应、执行超时等"非目标不可达"情况不冷却。
- 采集任务按团队/组织隔离，并依用户权限过滤可见节点与云区域。
- 采集执行与任务增删改均产生变更记录。
- 网络设备配置文件采集只面向已纳管网络设备实例，按任务定义的命令清单采集并归档。
- IP 发现按所选子网范围更新地址台账；未在本轮在线结果中的自动发现地址会被置为离线，手工地址不受影响。

## 5. 关键技术架构选择

- 采集执行下沉到 Stargazer 采集服务，平台通过 RPC 异步派发，经节点管理的接入点节点代理执行。
- 周期采集基于 Celery 定时任务调度，并对周期变更做延迟补偿，避免漏跑。
- 采集结果经比对引擎与图数据库 FalkorDB 中的已有实例比对后写回。
- 非 K8S 任务的节点参数同步到节点管理；K8S 任务直接携带集群标识，不走节点参数同步。

## 6. 验收标准

- 用户可完成采集任务的配置、执行、状态查看和结果摘要查看。
- 周期执行能完成新增/更新/删除/关联的完整合并；手动执行只做更新。
- 单任务可配置至多 3 组凭据，执行时按目标逐一尝试并记住命中凭据；凭据类失败与连接不可达类失败均按阶梯冷却后重试，冷却对单凭据与多凭据任务同样生效，某目标全部凭据冷却时当轮被跳过。
- SOID 特征库可维护映射并在 SNMP 采集中用于设备识别。
- 采集工具可在不创建任务的情况下完成协议联调诊断。
- 采集相关操作均可在变更记录中追溯。
- 网络设备配置文件采集可把网络设备命令输出沉淀为配置文件版本，并在资产详情查看历史与差异。
- IP 发现任务执行后可回写子网下在线 / 离线地址状态与利用率。
