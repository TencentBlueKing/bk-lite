# 运营分析-仪表板可折叠分组

> Migrated from `spec/requirements/运营分析/20260531.运营分析-仪表板可折叠分组.md` as legacy capability evidence.

## 1. 背景与问题
仪表板的所有组件平铺在同一栅格中。一张仪表板装载 8~10 个以上组件、覆盖多主题时（主机健康/业务指标/容量水位/告警概览…），用户只能靠组件标题区分上下文，长页面浏览负担重。

## 2. 业务需求
- 仪表板新增"分组"作为一种特殊组件，呈现为横跨整宽的标题条。
- 标题条下方的组件归属该分组；折叠时下属组件整体收起，仅保留标题条。

![仪表板分组示意（图源：Grafana 官方文档）](https://grafana.com/media/docs/grafana/dashboards/screenshot-dashboard-w-groupings-v13.0.png)

示意来自 Grafana 官方文档，体现的就是这种"按主题切分 + 标题条 + 折叠/展开"形态。

## 3. 业务逻辑
- 分组下属组件 = 编辑态栅格顺序上，处于本分组标题条与下一个分组标题条（或仪表板末尾）之间的所有组件。
- 拖动一个组件越过分组标题条，其归属随之改变；无需"加入分组"的显式操作。
- 折叠状态按"用户 + 仪表板"维度记忆，不写入仪表板配置、不参与导入导出。
- 折叠时下属组件停止取数；展开时按当前过滤条与时间范围重新取数。
- 分组不持有独立的过滤条、时间范围、命名空间；视图级统一过滤条对所有分组下的组件生效。

## 4. 交互
- 编辑模式下可新增分组、重命名、整组拖动顺序、整组删除（删除前提示，连同下属组件一起删除）。
- 查看模式下点击标题条左侧箭头展开/折叠；折叠状态由当前用户的浏览状态决定。
- 仪表板可以没有分组（保持当前形态），也可以同时存在"未分组区"与多个分组。

## 5. 关键技术选择
- 分组是布局组件，不绑定数据源、不发起查询。
- 归属由栅格位置决定，避免维护显式的"分组-组件"关系，让顺序调整与归属变化保持一致。

## 6. 验收口径
- 可在编辑模式新增/重命名/排序/删除分组；删除时提示并联动删除下属组件。
- 折叠后下属组件区域收起、停止取数；展开后按当前过滤条与时间范围重新取数。
- 折叠状态在同一用户再次访问时保持，不进入 YAML 导入导出内容。
- 分组本身不响应过滤条、不发起查询。

## 7. 约束与边界
**In Scope**：分组的新增/重命名/排序/删除/折叠展开、归属随栅格位置自动变化。

**Out of Scope**：分组级时间范围/过滤条/命名空间、按变量循环（repeat）、分组嵌套、拓扑图与架构图的分组形态。
