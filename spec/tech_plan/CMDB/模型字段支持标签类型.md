## 2) 核心数据结构 / Schema 定义

```python
from dataclasses import dataclass, field
from typing import Literal

TagMode = Literal["free", "strict"]

@dataclass(frozen=True)
class TagOption:
    """模型字段候选项。"""
    key: str
    value: str

@dataclass
class TagFieldConfig:
    """tag 字段配置，挂在 attr.option 中。"""
    mode: TagMode
    options: list[TagOption] = field(default_factory=list)

@dataclass(frozen=True)
class TagValueItem:
    """实例存储的标准化标签项。"""
    raw: str  # 规范格式: "key:value"

@dataclass
class TagValidationResult:
    """标签校验结果。"""
    normalized_values: list[TagValueItem]
    errors: list[str]
```

```python
# 固定规则口径（强约束）
# 1) 单模型仅允许一个 tag 字段。
# 2) tag 字段 attr_id 固定为 "tag"。
# 3) 实例值存储为 string[]，每项格式必须为 "key:value"。
# 4) 单实例最多 20 对标签。
# 5) key/value 均不能为空，且 value 禁止包含空格、冒号、换行。
# 6) 搜索仅支持大小写敏感精确匹配。
```

## 4) 核心逻辑伪代码 (Step-by-Step Logic)

```text
主链路：实例写入 + 自由模式候选回写 + 精确搜索

1) 模型字段定义阶段
   1.1 接收 attr_type=tag 的字段定义。
   1.2 校验 attr_id 必须等于 "tag"。
   1.3 校验当前模型 attrs 中 tag 字段数量（含本次变更后）<= 1。
   1.4 校验 option 结构：mode ∈ {free, strict}；options 为 [{key,value}]。
   1.5 写入模型 attrs。

2) 实例写入阶段（新增/编辑/批量/导入统一）
   2.1 若模型无 tag 字段：忽略输入中的 tag 值。
   2.2 若模型有 tag 字段：读取配置 mode/options。
   2.3 解析输入为 list[str]，元素必须是 "key:value"。
   2.4 执行逐项校验：
       - key/value 非空
       - value 不含空格、冒号、换行
       - 总量 <= 20
       - mode=strict 时每项必须存在于 options
   2.5 校验通过后去重并写回实例（string[]）。
   2.6 mode=free 时，将新增合法项按 (key,value) 精确去重后回写至模型 options。

3) 搜索阶段
   3.1 前端将所选标签值生成 selectedValues: string[]（每项 "key:value"）。
   3.2 构造查询条件：type=list_any[]，accurate=true。
   3.3 后端按 OR 语义做大小写敏感精确匹配。
   3.4 不走模糊检索分支。
```
