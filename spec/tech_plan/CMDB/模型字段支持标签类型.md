# 模型字段支持标签类型 - Tech Plan

日期：2026-03-04

## 技术目标与非目标

### 技术目标
- 在现有 CMDB 架构内新增字段类型 `tag`，不引入新服务与新存储。
- 打通模型字段配置、实例新增/修改、列表展示、标签搜索、导入导出、批量更新全链路。
- 建立“单一标签校验口径”，保证新增/修改/导入/批量更新规则一致。
- 一个模型最多能存在一个字段类型 `tag`的属性

### 非目标
- 不实现三级及以上标签树。
- 不实现标签 value 模糊检索。
- 不实现标签推荐、治理统计等运营能力。
- 不改造非标签类型字段。

## 2) 核心数据结构 / Schema 定义

### 2.1 后端 Dataclass

```python
# server/apps/cmdb/types/tag_schema.py
from dataclasses import dataclass, field
from typing import Literal

TagMode = Literal["free", "strict"]

@dataclass(frozen=True)
class TagPair:
    """单个标签键值对，格式 key:value。"""
    key: str
    value: str

@dataclass
class TagFieldOption:
    """标签字段配置：模式 + 候选项。"""
    mode: TagMode
    candidates: list[str] = field(default_factory=list)  # 每项格式 key:value

@dataclass
class TagValidationResult:
    """统一校验结果。"""
    normalized_tags: list[str]  # 去重后的 key:value 列表
    errors: list[str]
```

### 2.2 前端 TypeScript

```ts
// web/src/app/cmdb/types/tag.ts
export type TagMode = "free" | "strict";

export interface TagFieldOption {
  mode: TagMode;
  candidates: string[]; // key:value
}

export interface TagTreeNode {
  title: string;
  value: string;
  children?: TagTreeNode[];
}
```

### 2.3 规则口径（固定）
- 单模型最多 1 个标签字段，`attr_id` 固定 `__tag__`。
- 标签格式：`key:value`。
- `key` 不可空；`value` 不可含空格、冒号、换行。
- 单实例最多 20 对标签。
- 重复项按完整 `key:value` 去重。
- 严谨模式下仅允许候选项；导入/批量更新遇到非候选项严格失败。
- 搜索多选语义固定为 OR（命中任一标签）。

## 4) 核心逻辑伪代码 (Step-by-Step Logic)

### 4.1 写入链路统一校验（新增/修改/批量更新/导入）

```text
1. 识别当前模型 attrs 是否存在 __tag__ 字段。
   - 不存在：忽略标签输入，不入库，不参与展示/搜索。
   - 存在：读取 option(mode, candidates)。

2. 解析输入标签。
   - 支持 string | list[string] | list[dict]。
   - 分隔符支持: ,  ，  \n。
   - trim + 去空 + 去重。

3. 执行统一校验。
   - 每项必须满足 key:value。
   - key 非空。
   - value 不包含 空格/冒号/换行。
   - 总数 <= 20。
   - mode=strict 时，每项必须在 candidates 中。

4. 校验通过后，写回标准化值。
   - 存储结构统一为 list[str]，每项 key:value。
   - 进入原有实例 create/update/import/batch_update 流程。
```

### 4.2 模型字段删除语义（按确认口径）

```text
1. 用户在模型字段页点击删除标签字段。
2. 前端展示二次确认弹窗，确认后发起删除请求。
3. 后端仅从模型 attrs 移除 __tag__ 字段定义。
4. 历史实例数据中已存标签值保留，但所有读取链路按 attrs 控制：
   - 无 __tag__ 时不返回标签字段
   - 标签搜索入口隐藏
   - 导入导出不再暴露标签列
```

### 4.3 标签搜索（二层树 + OR 精确匹配）

```text
1. 用模型候选项构建二层树：key -> values。
2. 用户多选后生成查询条件：type=list_any[]，accurate=true（固定）。
3. 执行 OR 精确匹配（任一标签命中）。
4. 会话内跨页保留 selectedTags；切换筛选字段时清空 selectedTags。
```

## 影响范围与改动点

### 后端
- 类型体系：新增 `tag` attr_type。
- 模型字段：新增单模型唯一 + 固定 `__tag__` 约束。
- 实例写入：统一标签校验接入 create/update/import/batch_update。
- 查询：新增标签 OR 精确匹配参数构造。
- 导入导出：接入标签分隔符解析与错误定位。

### 前端
- 字段类型枚举：新增 `tag`。
- 模型字段配置：新增标签模式与候选维护、删除二次确认。
- 实例表单：Combo/Select 双模式输入与实时校验。
- 搜索：标签专用二层树组件与会话态管理。
- 列表：标签列逗号展示 + 超长 tooltip。

### 外部交互边界
- 不新增 API 路径。
- 在现有模型/实例接口中扩展 `attr_type=tag` 的请求与响应字段。

## 测试方案与发布回滚策略

### 测试方案
- 后端单测：
  - 标签格式校验（合法/非法）。
  - 20 对上限。
  - strict 模式候选校验。
  - 导入分隔符解析与错误定位。
  - 查询参数构造（list_any[] + accurate=true）。
- 前端测试/回归：
  - 字段配置：创建/编辑/删除二次确认。
  - 实例录入：自由/严谨模式行为差异。
  - 搜索：二层树多选、跨页保留、切换清空。
  - 列表展示：截断与 tooltip。
- 门禁命令：
  - `cd server && make test`
  - `cd web && pnpm lint && pnpm type-check`

### 发布策略
- 先后端后前端灰度上线。
- 先启用模型字段定义能力，再开放实例端录入与搜索入口。

### 回滚策略
- 代码回滚：`git revert <commit>`。
- 功能回滚：前端隐藏 `tag` 类型入口 + 后端禁用 `tag` 分支逻辑。
- 数据回滚：不涉及结构迁移，按业务开关回退即可。
