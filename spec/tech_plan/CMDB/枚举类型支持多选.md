# 枚举类型支持多选 - Tech Plan

日期：2026-03-10

## 技术目标与非目标

### 技术目标
- 在现有 CMDB 架构内，为 `enum` 字段落地 single/multiple 选择模式，不新增服务、不扩展跨模块架构。
- 保证字段配置、实例写入、列表展示、搜索筛选、批量更新、导入导出语义一致。
- 固化数据口径：single/multiple 均使用数组存储，`_display` 冗余字段使用英文逗号拼接。
- 固化导入失败策略：全量错误汇总后一次返回。

### 非目标
- 不改造非 `enum` 字段类型。
- 不做存量单选字段自动迁移。
- 不改造 CMDB 之外系统。

## 2) 核心数据结构 / Schema 定义

```python
from dataclasses import dataclass
from typing import Literal, Any

EnumSelectMode = Literal["single", "multiple"]

@dataclass(frozen=True)
class EnumFieldAttr:
    """
    挂载在模型字段 attr 下的枚举配置。
    关键规则：
    1) enum_select_mode 必填，默认 single
    2) 字段创建后 enum_select_mode 不可变更
    """
    enum_select_mode: EnumSelectMode
    option: list[dict[str, str]]  # [{"id": "...", "name": "..."}]

@dataclass(frozen=True)
class EnumValuePayload:
    """
    实例值统一数组存储：
    - single: len(values) == 1
    - multiple: len(values) >= 1 (若字段 required=False 可为空数组)
    """
    values: list[str]

@dataclass(frozen=True)
class EnumDisplayPayload:
    """
    冗余展示字段，英文逗号拼接。
    示例: ["A", "B"] -> "A,B"
    """
    display: str

@dataclass(frozen=True)
class ImportCellError:
    row: int
    field_key: str
    raw_value: str
    code: Literal[
        "EMPTY_NOT_ALLOWED",
        "INVALID_ENUM_OPTION",
        "SINGLE_MODE_TOO_MANY_VALUES",
        "VALUE_TYPE_ERROR",
    ]
    message: str

@dataclass(frozen=True)
class ImportValidateResult:
    """
    全量错误汇总后一次返回：
    - errors 非空: 不落库
    - errors 为空: 返回 normalized_rows 给后续写入
    """
    normalized_rows: list[dict[str, Any]]
    errors: list[ImportCellError]
```

固定规则（强约束）：
```python
# 1) 仅 enum 字段启用 enum_select_mode
# 2) enum_select_mode 存在模型字段 attr 属性下，不新增独立持久化字段
# 3) single / multiple 实例值均为数组存储
# 4) _display 冗余字段直接 ",".join(values)
# 5) 搜索语义：字段内 OR；字段间 AND
# 6) 导入失败策略：全量错误汇总一次返回
```

## 4) 核心逻辑伪代码 (Step-by-Step Logic)

### A. 字段配置保存（最关键：模式不可切换）

```text
1. 接收字段保存请求，若 type != enum：跳过本方案逻辑。
2. 从 attr 读取 enum_select_mode；缺省则写入 single。
3. 若为编辑场景：
   3.1 读取旧字段 attr.enum_select_mode。
   3.2 若新旧 mode 不同，返回 MODE_IMMUTABLE 错误。
4. 持久化 attr（含 enum_select_mode）。
5. 返回字段详情。
```

### B. 实例写入（新增/编辑/复制/批量统一）

```text
1. 读取模型字段元数据：mode、required、option_ids。
2. 将输入值标准化为 string[]（single/multiple 统一数组）。
3. 执行校验：
   - required 且空 -> EMPTY_NOT_ALLOWED
   - mode=single 且 len(values)!=1 -> SINGLE_MODE_TOO_MANY_VALUES
   - 任一值不在 option_ids -> INVALID_ENUM_OPTION
4. 保存 values 到字段值。
5. 同步写入 <field>_display = ",".join(values)。
6. 返回成功。
```

### C. 搜索筛选（字段内 OR、字段间 AND）

```text
1. 对每个筛选字段读取 selected_values。
2. 构建该字段 Q：value contains 任一 selected_value（OR）。
3. 将多个字段 Q 用 AND 合并。
4. 执行查询并返回结果。
```

### D. 导入处理（全量错误汇总一次返回）

```text
1. 遍历每行每字段，解析 raw -> values[]（支持 ',', '，', '\n'）。
2. 调用与在线写入一致的校验逻辑。
3. 失败则记录 ImportCellError(row, field_key, raw_value, code, message)，继续下一项。
4. 全量扫描后：
   - 若 errors 非空：一次返回全部 errors，不写入
   - 若 errors 为空：批量写入 normalized_rows
```

测试方案与验收映射：
```text
- UT:
  - normalize_enum_values: 分隔符混用、空值、顺序保持
  - validate_enum_values: single/multiple、required、非法值
  - build_enum_display: 数组->逗号串
  - build_enum_filter_q + merge_filter_q_by_and: OR/AND 语义
- IT:
  - 字段创建/编辑模式锁定
  - 实例新增/编辑/复制/批量一致性
  - 导入全量错误一次返回
  - 导出格式稳定为英文逗号
- 回归:
  - 存量 single 行为不变（值仍为数组但长度=1）
```

## 5) 第三方依赖与环境要求 (Dependencies)

- 无新增依赖。  
- 运行环境沿用现有：
  - 后端：Python 3.12 + Django 4.2
  - 前端：Next.js + TypeScript（现有依赖）
- 发布策略：
  - 先灰度模型配置与实例写入入口，再灰度导入导出与筛选入口。
  - 监控指标：写入失败率、导入错误率、筛选命中准确率。
- 回滚策略：
  - 代码回滚到变更前版本。
  - 不做数据回迁；既有数组值继续可读。
  - 恢复旧校验分支时，single 按数组长度=1处理。

## 6) 要求主要函数和主要逻辑补充相关注释和关键日志

主要函数注释要求：
```text
- 每个核心函数必须有“输入/输出/失败条件”注释。
- 类内非简单函数必须额外说明“主要作用 + 触发场景 + 副作用”。
- 禁止写无信息注释（如“这里进行校验”）。
```

关键日志要求（必须落点）：
```text
1) 字段模式锁定失败：
   level=WARNING
   keys: model_id, field_key, old_mode, new_mode, operator

2) 实例写入校验失败：
   level=WARNING
   keys: model_id, instance_id, field_key, mode, error_code

3) 导入校验结束：
   level=INFO
   keys: model_id, total_rows, success_rows, error_count, duration_ms

4) 导入失败明细输出（仅结构化，不打敏感值全文）：
   level=WARNING
   keys: row, field_key, error_code
```

排查优先级建议：
```text
- P0: MODE_IMMUTABLE、批量写入失败、导入全量失败
- P1: 单字段筛选命中偏差
- P2: _display 展示不一致
```
