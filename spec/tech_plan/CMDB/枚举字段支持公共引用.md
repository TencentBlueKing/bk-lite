# 枚举字段支持公共引用 - Tech Plan

日期：2026-03-09

## 技术目标与非目标

### 技术目标
- 在现有 CMDB 架构内，为 `enum` 字段新增“公共选项库”引用能力，不新增服务、不改动跨模块架构。
- 落地字段规则：`custom/public_library`，默认 `custom`，字段创建后规则类型不可切换。
- 落地删除规则：公共选项库被引用时禁止删除，并返回引用该库的模型/属性字段明细。
- 落地一致性策略：公共选项库变更后通过 Celery 异步刷新引用字段快照。
- 落地写入策略：历史值保留；新建/编辑/导入/批量更新统一按当前绑定库校验。
- 落地文案策略：错误提示与前端 i18n 规范对齐（后端返回结构化错误，前端词条渲染）。

### 非目标
- 不支持 `model_config` 接入公共选项库。
- 不支持非 `enum` 字段接入。
- 不实现审批流、版本管理、跨租户共享。
- 不改造 CMDB 之外模块。

## 2) 核心数据结构 / Schema 定义

### 2.0 后端持久化模型（Django Models）

```python
from django.db import models

from apps.core.models.maintainer_info import MaintainerInfo
from apps.core.models.time_info import TimeInfo


class PublicEnumLibrary(MaintainerInfo, TimeInfo):
  """公共选项库主表。"""

  library_id = models.CharField(max_length=64, unique=True, db_index=True)
  name = models.CharField(max_length=128, db_index=True)
  team = models.JSONField(default=list, help_text="关联组织")
  options = models.JSONField(default=list)

  class Meta:
    db_table = "cmdb_public_enum_library"
    indexes = [
      models.Index(fields=["library_id"], name="idx_public_enum_lib_id"),
      models.Index(fields=["name"], name="idx_public_enum_lib_name"),
    ]
```

```text
模型设计说明（强约束）
1. options 结构固定为 [{"id": str, "name": str}]，服务层负责内容校验。
2. 删除语义采用物理删除（hard delete）：若被字段引用则直接拦截，不进入删除。
3. 不新增“引用关系表”，引用关系实时扫描模型 attrs（保持最小改动）。
4. team 直接复用项目现有“关联组织”口径，不新增 include_subgroups 字段。
5. library_id 全局唯一，name 非唯一（同名是否允许由服务层按业务规则校验）。
```

### 2.1 后端（Python Dataclasses）

```python
from dataclasses import dataclass, field
from typing import Literal

EnumRuleType = Literal["custom", "public_library"]


@dataclass(slots=True, frozen=True)
class EnumOptionItem:
    """枚举选项标准结构。"""
    id: str
    name: str


@dataclass(slots=True)
class EnumPublicBinding:
    """挂在 enum 字段上的公共选项库绑定配置。option 直接复用之前的option即可无需新开字段"""
    enum_rule_type: EnumRuleType = "custom"
    public_library_id: str | None = None
    option: list[EnumOptionItem] = field(default_factory=list)


@dataclass(slots=True, frozen=True)
class PublicEnumLibrary:
    """公共选项库实体。"""
    library_id: str
    name: str
    team: list[str]
    options: list[EnumOptionItem]


@dataclass(slots=True, frozen=True)
class LibraryReferenceItem:
    """删除拦截时返回的引用明细。"""
    model_id: str
    model_name: str
    attr_id: str
    attr_name: str
```

## 4) 核心逻辑伪代码 (Step-by-Step Logic)

### 4.1 删除拦截（被引用不可删 + 返回引用明细）

```text
1. 接收 DELETE /public_enum_libraries/{library_id}
2. 调用 find_library_references(library_id)
3. 若 references 非空：
   3.1 返回 HTTP 409
   3.2 响应体：error_code=CMDB_PUBLIC_ENUM_LIBRARY_IN_USE
   3.3 携带 references=[{model_id, model_name, attr_id, attr_name}, ...]
   3.4 终止删除
4. 若 references 为空：
   4.1 执行删除
   4.2 返回 success
```

### 4.2 公共选项库变更后 Celery 异步刷新快照

```text
1. 公共库 create/update 成功后，调用 enqueue_library_snapshot_refresh(...)
2. Celery 任务拉取 library 最新 options
3. 查询所有引用该库的 enum 字段
4. 逐字段更新：
   4.1 更新 attr.option（快照）
   4.2 保持 enum_rule_type/public_library_id 不变
5. 对受影响 enum 字段触发实例 display 刷新（复用现有能力）
6. 记录统计并返回：affected_models/affected_attrs/affected_instances/failed_items
```

### 4.3 实例写入校验（历史值保留，新写入强校验）

```text
1. 所有写入口（新增/编辑/导入/批量更新）统一走 validate_instance_data
2. 当 attr_type == enum：
   2.1 若 enum_rule_type == custom：按字段本地 option 校验
   2.2 若 enum_rule_type == public_library：按当前绑定公共库 options 校验
3. 校验失败：拦截本次写入并返回结构化错误
4. 不自动修复历史实例旧值；仅保证新写入合法
```

## 测试方案与发布回滚策略

### 测试方案（最小可执行）
- 后端单测：
  - 公共库被引用删除拦截与明细返回。
  - enum_rule_type 创建后不可切换。
  - public_library 校验通过/失败分支。
  - Celery 任务快照刷新统计结果。
- 前端验证：
  - 管理入口与弹窗可用。
  - 字段配置规则切换限制生效。
  - 删除拦截弹窗正确展示引用明细。
  - 错误文案走 i18n 词条。
- 回归点：
  - 枚举筛选、导入导出、批量更新与现有行为一致。

### 发布回滚策略
- 发布策略：后端接口与字段规则先上线，前端功能随后启用；Celery 任务同步上线。
- 回滚策略：
  - 若新逻辑异常，可回退到 `custom` 解析分支（解析层回滚，不改历史数据）。
  - 关闭公共库入口后，不影响已有自定义枚举字段。
