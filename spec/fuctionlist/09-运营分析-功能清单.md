## 一、模块定位

运营分析将多源运维数据统一编排为可消费的分析视图，提供"数据接入—命名空间—分析资产—可视化呈现"的一体化工作台。模块由主视图（仪表盘、拓扑图、架构图）与管理页（数据源、命名空间）两部分构成；数据既可经命名空间 NATS 通道实时拉取，也可通过数据库、REST API、Excel 数据源直接获取。本清单仅列已实现能力，不含移动端、跨仪表盘共享筛选、表格回写/计算列等明确排除项。

## 三、能力边界与约束

目录层级上限 3 层，同一父目录下目录名称唯一；仪表盘、拓扑图、架构图、大屏、报表名称全局唯一；数据源名称与其连接标识组合保持唯一；命名空间名称唯一且密码加密存储。统一筛选本期仅适用于仪表盘与大屏画布，拓扑图、架构图、报表不支持统一筛选；不支持组件间点击联动、钻取联动等事件型筛选，也不支持跨仪表盘共享筛选状态。表格组件不含在线编辑回写、计算列、汇总行、文件导出等高级能力。报表当前为基础模式，尚未提供完整编排器。本模块不提供移动端能力，不含模块内自动化任务编排，不定义模块外算法能力与数据质量治理流程。后端 datasource_view.py 中静态演示数据源短路块（namespace=="demo"）为注释代码，不构成运行时能力，不计入本清单。

## 四、平台协同

运营分析通过命名空间经 NATS 通道向 CMDB、告警、监控、日志、业务等平台数据源实时拉取数据，也可通过数据库、REST API、Excel 数据源接入外部分析数据并聚合呈现；数据源标签内置 CMDB / 告警 / 监控 / 日志 / 业务 / 其他六类，对应平台各数据来源。用户与团队上下文随取数请求透传，数据可见性与系统管理的组织及权限体系保持一致。命名空间连接参数依赖部署环境的 NATS 服务配置。

### 5.6 命名空间（NameSpace）

命名空间用于经 NATS 通道隔离取数主题，默认随包预置「默认命名空间」（`namespace.json`，NATS 命名空间前缀默认 `bklite`），支持 TLS 开关与连接凭据加密存储。数据源、拓扑图/架构图取数节点均通过关联命名空间确定取数通道。

> 说明：上述枚举均为已实现落地项、**全部为 GA**（源码中无 Beta/试验标注），不含移动端、跨仪表盘共享筛选等排除项。图表/组件类型以前端 widgetRegistry 注册为准（9 种）；后端 `DataSourceAPIModel.chart_type` 为 JSON 透传字段、`is_active` 为内部预留兼容字段，均不构成独立产品枚举。

| 枚举项 | 取值 | 中文含义 |

### 结构化单位 ID（unitId）

取自 `utils/unitFormat.ts`，共 9 种（GA）：

| 枚举项 | 取值 | 自动缩放规则 |
|---|---|---|
| 二进制字节 | `bytesIEC` | 1024 进位：B→KiB→MiB→GiB→TiB |
| 十进制字节 | `bytesSI` | 1000 进位：B→KB→MB→GB→TB |
| 数据速率 | `bps` | bps→Kbps→Mbps→Gbps |
| 毫秒时间 | `ms` | ms→s→m→h→d 自动进位 |
| 百分比（0-100） | `percent` | 原样 + % 后缀 |
| 百分比（0-1） | `percentunit` | ×100 后 + % 后缀 |
| 短数字 | `short` | 1.2K / 3.4M / 5.6B 自动缩放 |
| 无单位 | `none` | 原样展示，无后缀 |
| 自定义后缀 | `custom:<后缀>` | 原值 + 指定后缀；兼容旧自由文本 unit 字段 |

`unitId` 优先于自由文本 `unit` 字段；两者同时存在时以 `unitId` 为准。
