**文档版本：** V1.0

## 一、模块定位

作业管理是平台统一的脚本执行、文件分发与作业调度中心，覆盖"作业准备 → 执行 → 追踪 → 复用 → 治理"五类场景。模块通过目标管理维护异构主机的连接驱动与凭据，通过脚本库与 Playbook 库实现模板化与参数化复用，通过高危命令与高危路径规则降低误操作风险，并以作业记录与作业详情保留完整执行档案。本清单仅列已实现能力。

## 三、能力边界与约束

作业类型含脚本执行、文件分发、Playbook；触发来源含手动执行、定时任务、API 调用。Playbook 执行依赖 Ansible 驱动主机，其他驱动不支持 Playbook，选择目标时不兼容主机会被禁用并提示原因。执行状态共 6 种：等待中、执行中、成功、失败、超时、已取消；其中成功、失败、超时、已取消为终态。队列中的作业支持取消，已完成作业支持重新执行。高危命令与高危路径规则当前不提供系统内置只读规则，规则状态仅含已启用、已停用，处理策略为"禁止"时阻断执行、为"二次确认"时需确认后继续，规则按组织范围生效。高危路径匹配仅支持精确匹配与正则匹配。加密参数与凭据不明文回显。

## 四、平台协同

作业管理从节点管理同步可执行目标，并复用节点管理提供的本地执行通道（同步来源目标无需额外连接凭据）；执行目标清单可由 CMDB 资产数据辅助提供；组织范围、用户与凭据管理由系统管理统一治理；执行结果与异常可经系统管理配置的通知渠道送达，并向控制台消息中心推送通知。

## 五、支持的执行对象与能力范围

以下枚举均以后端常量 `apps/job_mgmt/constants/choices.py` 为准，状态机取值与默认值取自该文件及 `models/scheduled_task.py`。

### 5.1 脚本语言（ScriptType）

| 语言 | 取值 | 执行 shell 映射 | 状态 |
|---|---|---|---|
| Shell | `shell` | `sh` | GA |
| Python | `python` | `python` | GA |
| PowerShell | `powershell` | `powershell` | GA |
| Batch（批处理） | `bat` | `cmd` | GA |

共 4 种；脚本默认超时 60 秒，参数采用 Jinja2 模板语法。

### 5.2 操作系统与连接方式

| 操作系统（OSType） | 取值 | 连接方式 | 默认端口 | 认证方式 |
|---|---|---|---|---|
| Linux | `linux` | SSH | 22 | 密码 / 密钥（密钥支持口令） |
| Windows | `windows` | WinRM | 5986 | Basic / NTLM（默认）/ Kerberos / CredSSP；协议 HTTP / HTTPS（默认 HTTPS） |

操作系统 2 种；SSH 凭据类型 2 种（password、key）；WinRM 协议 2 种、认证传输方式 4 种。凭据来源 2 种：手动录入（manual）、凭据管理（credential）。

### 5.3 执行驱动（ExecutorDriver）与目标来源

| 类别 | 枚举值 | 说明 |
|---|---|---|
| 执行驱动 | Ansible（`ansible`）、Sidecar（`sidecar`） | 后端驱动枚举共 2 种；目标管理界面手动录入目标提供 Ansible、SSH 两类选项，Playbook 仅支持 Ansible 驱动目标 |
| 目标来源 | 同步（`sync`）、手动（`manual`）、节点管理（`node_mgmt`） | 共 3 种；同步/节点管理来源走本地执行，手动来源走 SSH/远程执行 |

### 5.4 作业类型与触发来源

| 类别 | 枚举值 | 数量 |
|---|---|---|
| 作业类型（JobType） | 脚本执行（`script`）、文件分发（`file_distribution`）、Playbook（`playbook`） | 3 |
| 触发来源（TriggerSource） | 手动执行（`manual`）、定时任务（`scheduled`）、API 调用（`api`） | 3 |

### 5.5 执行状态机（ExecutionStatus）

| 状态 | 取值 | 是否终态 |
|---|---|---|
| 等待中 | `pending` | 否 |
| 执行中 | `running` | 否 |
| 成功 | `success` | 是 |
| 失败 | `failed` | 是 |
| 超时 | `timeout` | 是 |
| 已取消 | `cancelled` | 是 |

共 6 种，其中成功、失败、超时、已取消为终态（`TERMINAL_STATES`）。

### 5.6 定时调度与并发策略

| 类别 | 枚举值 | 说明 |
|---|---|---|
| 调度类型（ScheduleType，后端） | 单次执行（`once`）、周期执行（`cron`） | 后端枚举共 2 种 |
| 执行策略（前端落地） | 简单策略（每天一次 / 每 N 小时一次 / 只执行一次）、复杂策略（Cron 表达式编辑） | 简单策略最终映射为 `once` 或 `cron` |
| 并发策略（ConcurrencyPolicy） | 跳过（`skip`，默认）、运行（`run`）、排队（`queue`） | 共 3 种 |

定时任务默认超时 300 秒。

### 5.7 文件分发覆盖策略（OverwriteStrategy）

| 策略 | 取值 |
|---|---|
| 覆盖已存在文件 | `overwrite` |
| 跳过已存在文件 | `skip` |

共 2 种。

### 5.8 高危规则处理策略与匹配方式

| 类别 | 枚举值 | 数量 |
|---|---|---|
| 处理策略（DangerousLevel） | 二次确认（`confirm`）、禁止分发（`forbidden`） | 2 |
| 路径匹配方式（MatchType） | 精确匹配/路径前缀（`exact`）、正则匹配（`regex`） | 2 |

> 说明：上述枚举均直接来自后端常量定义，不含演进展望项。高危命令规则的命令匹配支持正则；高危路径匹配仅含精确匹配与正则匹配两种，不支持通配符。前端"执行策略"的简单/复杂划分是后端 ScheduleType（once/cron）的界面封装，二者并不一一对应。

| 枚举项 | 取值 | 中文含义 |
|---|---|---|
| 同步 | `sync` | 从 Node 同步的目标（兼容旧值，本地执行） |
| 手动 | `manual` | 手动新增的目标，走 SSH 远程执行 |
| 节点管理 | `node_mgmt` | 来自节点管理的目标，本地执行 |

### 脚本语言

| 枚举项 | 取值 | 中文含义 |
|---|---|---|
| Shell | `shell` | Shell 脚本，Linux 下用 bash 执行 |
| Python | `python` | Python 脚本，用 python 解释器执行 |
| PowerShell | `powershell` | PowerShell 脚本，Windows 下执行 |
| Batch | `bat` | Windows 批处理脚本，用 cmd 执行 |

### 覆盖策略

| 枚举项 | 取值 | 中文含义 |
|---|---|---|
| 覆盖已存在文件 | `overwrite` | 分发时覆盖已存在的同名文件 |
| 跳过已存在文件 | `skip` | 分发时跳过已存在的同名文件 |

### 触发来源

| 枚举项 | 取值 | 中文含义 |
|---|---|---|
| 手动执行 | `manual` | 由用户手动触发执行 |
| 定时任务 | `scheduled` | 由定时任务触发执行 |
| API调用 | `api` | 由 API 调用触发执行 |
