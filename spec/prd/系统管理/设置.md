# 系统管理 · 平台设置

## 1. 背景与目标

平台设置维护平台级的系统配置、对外访问密钥与平台运行的可观测入口，统一管理 API 密钥、门户品牌等平台配置，并提供审计日志与错误日志的查看入口。敏感信息保护已迁移至用户管理统一治理，不在本模块范围内。

## 2. 范围定义

- 密钥：API 密钥/系统密钥的管理。
- 系统配置：门户品牌、水印等平台级配置（许可证、门户配置等）。
- 审计日志：操作日志与登录日志的统一查看入口。
- 错误日志：平台运行错误日志的查看入口。

## 3. 关键能力

- 支持 API 密钥/系统密钥的管理，用于程序化访问。
- 支持门户品牌配置（名称、Logo、图标）等平台级设置，部分门户配置可免鉴权公开读取。
- 支持审计日志查看，按操作日志、登录日志分页签呈现平台操作与登录记录。
- 支持错误日志查看，便于排查平台运行异常。

## 4. 关键规则

- 系统配置以键值方式存储，部分门户配置对外公开。
- 密钥用于对外鉴权访问。
- 许可证、门户配置作为平台设置下的受控资源；敏感信息保护资源归属用户管理（含安全角色相应权限），不在平台设置内维护。

## 5. 关键技术架构选择

- 平台级配置以统一的键值配置存储，按需公开或鉴权访问。
- 审计日志聚合操作日志与登录日志两类记录，统一入口呈现。

## 6. 验收标准

- 管理员可管理 API 密钥并用于程序化访问。
- 门户品牌等平台级配置可设置并生效。
- 管理员可在审计日志中查看操作日志与登录日志，可在错误日志中查看平台运行异常记录。
