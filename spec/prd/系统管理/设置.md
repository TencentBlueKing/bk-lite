# 系统管理 · 设置

## 1. 背景与目标

设置维护平台级的系统配置与 API 密钥，包括门户品牌、敏感信息保护开关与对外访问密钥。

## 2. 范围定义

- 密钥：API 密钥/系统密钥的管理。
- 系统配置：门户品牌、水印、敏感信息保护等平台级配置。

## 3. 关键能力

- 支持 API 密钥/系统密钥的管理，用于程序化访问。
- 支持门户品牌配置（名称、Logo、图标）等平台级设置，部分门户配置可免鉴权公开读取。
- 支持敏感信息保护开关（如审计日志中邮箱/电话脱敏）与水印设置。

## 4. 关键规则

- 系统配置以键值方式存储，部分门户配置对外公开。
- 密钥用于对外鉴权访问。

## 5. 关键技术架构选择

- 平台级配置以统一的键值配置存储，按需公开或鉴权访问。

## 6. 验收标准

- 管理员可管理 API 密钥并用于程序化访问。
- 门户品牌等平台级配置可设置并生效。
- 敏感信息保护开关可控制审计日志脱敏。
