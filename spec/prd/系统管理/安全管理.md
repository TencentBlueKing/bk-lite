# 系统管理 · 安全管理

## 1. 背景与目标

安全管理统一管控登录安全策略、认证源与审计日志，保障平台访问安全与可追溯。

## 2. 范围定义

- 登录设置：密码策略与锁定策略。
- 认证源：多种外部认证后端与用户同步。
- 登录日志、操作日志、错误日志：安全审计。

## 3. 关键能力

### 3.1 登录设置

- 配置密码策略：长度、字符类型要求、有效期、重试次数、锁定时长、过期提醒等。
- 登录失败按次累计，超过阈值锁定账号至超时；改密成功后重置。

### 3.2 认证源

- 支持多种认证后端（如企业微信、钉钉、自定义 OAuth、内置域认证），记录应用标识与密钥（加密）及同步计划。
- 支持从认证源周期同步用户与组织，并据来源映射归属组织。

### 3.3 审计日志

- 登录日志：记录登录成功/失败、用户名、来源 IP、操作系统/浏览器、位置、失败原因。
- 操作日志：审计跨应用的新增/修改/删除/执行操作，含时间、用户、IP、应用、动作与摘要。
- 错误日志：记录未处理异常的用户、应用、模块、错误信息与堆栈。

## 4. 关键规则

- 登录失败达阈值锁定账号，锁定到期或改密后解除。
- 认证源密钥加密存储。
- 敏感信息（邮箱/电话）可按开关在审计日志中脱敏。

## 5. 关键技术架构选择

- 支持多种认证后端与多域用户同步，认证为平台中央能力。
- 审计分登录、操作、错误三类分别记录。

## 6. 验收标准

- 管理员可配置密码与锁定策略并正确生效。
- 可配置认证源并周期同步用户与组织。
- 登录、操作、错误日志可完整审计并支持敏感信息脱敏。
