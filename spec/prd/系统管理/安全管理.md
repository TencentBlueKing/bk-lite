# 系统管理 · 安全管理

## 1. 背景与目标

安全管理统一管控登录安全策略、认证源与审计日志，保障平台访问安全与可追溯。安全管理不再作为系统管理的独立一级菜单，相关能力按用户与平台两条主线分布：认证源与安全策略归入「用户管理」，审计日志与错误日志归入「平台设置」。

## 2. 范围定义

- 安全策略（原登录设置）：密码策略与锁定策略，归属用户管理。
- 认证源：多种外部认证后端与用户同步，归属用户管理。
- 审计日志：操作日志与登录日志的统一入口，归属平台设置。
- 错误日志：未处理异常审计，归属平台设置。

## 3. 关键能力

### 3.1 安全策略

- 配置密码策略：长度、字符类型要求、有效期、重试次数、锁定时长、过期提醒等。
- 登录失败按次累计，超过阈值锁定账号至超时；改密成功后重置。

### 3.2 认证源

- 支持多种认证后端（如企业微信、钉钉、自定义 OAuth、内置域认证），记录应用标识与密钥（加密）及同步计划。
- 支持从认证源周期同步用户与组织，并据来源映射归属组织。

### 3.3 审计日志

- 操作日志与登录日志合并为单一「审计日志」页面，以页签切换两类内容，默认进入操作日志页签；两类日志各自维持原有的接口边界与交互差异，不强行合并为单一数据模型。
- 登录日志：记录登录成功/失败、用户名、来源 IP、操作系统/浏览器、位置、失败原因，支持导出。
- 操作日志：审计跨应用的新增/修改/删除/执行操作，含时间、用户、IP、应用、动作与摘要，支持按操作模块、操作类型、时间范围等维度筛选。

### 3.4 错误日志

- 记录未处理异常的用户、应用、模块、错误信息与堆栈。

## 4. 关键规则

- 登录失败达阈值锁定账号，锁定到期或改密后解除。
- 认证源密钥加密存储。
- 敏感信息（邮箱/电话）可按开关在审计日志中脱敏。
- 审计日志页面采用统一的页面级查看权限控制访问，操作日志与登录日志不再保留各自独立的查看权限；不具备该权限的用户不暴露审计日志入口。
- 被迁移页面的原安全管理路径整体废弃，不提供兼容跳转。

## 5. 关键技术架构选择

- 支持多种认证后端与多域用户同步，认证为平台中央能力。
- 审计日志将操作日志与登录日志合并为统一页面入口，错误日志独立记录；操作与登录两类日志在统一页面内仍各自保留独立的接口与能力边界。

## 6. 验收标准

- 管理员可在用户管理下配置密码与锁定策略并正确生效。
- 可在用户管理下配置认证源并周期同步用户与组织。
- 审计日志在平台设置下提供统一页面，操作日志与登录日志以页签呈现且默认进入操作日志，分别支持各自的筛选与导出能力。
- 错误日志在平台设置下可完整审计；审计日志支持敏感信息脱敏。
- 具备统一审计日志查看权限的用户可访问审计日志页面，不具备者不暴露入口。
