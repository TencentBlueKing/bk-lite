# 控制台 · 控制台

## 1. 背景与目标

控制台是登录后的统一门户首页，以应用卡片的形式聚合用户有权访问的各模块入口，并支持个性化定制，作为平台的导航中枢。

## 2. 范围定义

- 应用入口：以卡片展示可访问的应用模块。
- 个性化定制：用户级的应用选择与排序。
- 门户头部：欢迎信息与日期等全局展示。

## 3. 关键能力

- 以栅格卡片展示用户可访问的应用，卡片含图标、名称、描述、标签与进入入口。
- 内容来源两层：有个性化配置时按用户配置展示，否则展示全部可访问应用；自动排除控制台自身。
- 支持个性化定制：通过设置弹窗拖拽排序、移除或添加应用，并保存。
- 展示门户欢迎信息与本地化日期等全局头部。

## 4. 关键规则

- 可见应用由后端按用户角色/权限过滤，用户只看到有权访问的应用。
- 个性化配置按用户隔离（用户 + 域），各自管理自己的门户，无全局默认或管理员覆盖。
- 首次登录会引导用户确认组织归属。
- 控制台仅为导航中枢，不聚合其他模块的运营指标数据。

## 5. 关键技术架构选择

- 应用清单来自核心应用注册接口，用户个性化配置独立持久化，前端合并后渲染。

## 6. 验收标准

- 用户登录后可在控制台看到有权访问的应用卡片并进入。
- 个性化定制可拖拽排序、增删应用并保存，按用户隔离生效。
- 首次登录可正确引导组织归属。
