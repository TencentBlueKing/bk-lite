# 监控系统 · 集成

## 1. 背景与目标

集成定义"监控什么、怎么采、采哪些指标"，并把被监控资源沉淀为监控实例，按对象、分组、资产组织起来，是监控数据的来源底座。

## 2. 范围定义

- 集成（插件管理）：监控插件的查看、新增、编辑、删除与配置。
- 资产：监控实例的查看、创建、编辑、删除与可达性检验。
- 流量监控接入：基于 NetFlow/sFlow 的网络设备流量监控，通过接入向导完成资产纳管与设备导出引导。
- 分组：实例分组规则的配置与组织归属。
- 对象：监控对象与对象类型的维护。

## 3. 关键能力

### 3.1 集成（插件管理）

- 支持查看内置与自定义监控插件，支持自定义插件的新增、编辑、删除。
- 插件包含采集器、采集类型、配置模板与指标定义。
- 配置：维护表单 UI 模板与配置文件模板（如 SNMP 配置导入导出）。
- 指标：定义指标组与指标，配置查询语句、单位、维度、数据类型。
- 采集：查看采集与节点关联配置，触发手动采集。

### 3.2 资产

- 支持查看监控实例列表，按对象、名称、采集状态筛选。
- 支持创建手动监控实例（如 K8s 集群、云资源），编辑名称、采集间隔、自动发现标识。
- 支持删除实例并同步清理其子配置与分组规则，删除主机等远程监控对象时一并刷新其复用关系。
- 支持实例可达性检验。

### 3.5 流量监控接入

- 面向交换机、路由器、防火墙、负载均衡四类网络设备，提供 NetFlow 与 sFlow 两种流量协议的内置监控插件及指标（如总入/出流量及对应阈值告警）。
- 提供「流量接入引导向导」，分三步完成接入：绑定资产 → 接入指引 → 完成。
  - 绑定资产：可新建流量资产或复用已有资产，登记云区域、设备 IP、名称、组织归属与兜底采样率，并标记本次启用的流量协议。
  - 接入指引：展示当前云区域的流量监听接入地址（按协议区分端口）与设备侧导出配置说明，并可检测最近时间窗内是否已收到该资产对应协议的实际流量数据。
  - 完成：引导用户跳转查看监控视图或返回插件列表。
- 同一云区域 + IP 的流量资产唯一；重复绑定会合并已启用协议而非新建实例。
- 主机等对象的远程监控支持复用已有监控实例，避免重复纳管。

### 3.3 分组

- 支持实例分组规则的新增、编辑、删除，按对象、名称、启用状态筛选。
- 规则按实例维度自动将实例归入指定组织，可针对特定实例或全对象级别。
- 启用时可同步更新已分组实例的组织归属。

### 3.4 对象

- 支持查看监控对象树（父子对象），按类型、名称筛选。
- 支持自定义监控对象类型与对象的新增、编辑、删除，支持对象继承与显示排序。
- 支持编辑对象显示名、图标、描述与补充指标。

## 4. 关键规则

- 监控对象分为基础对象与衍生对象，衍生对象继承父对象，构成树状结构。
- 实例唯一标识由维度组合（实例标识键）决定。
- 采集间隔可自定义（默认 10 秒）。
- 实例与组织关联，权限基于实例所属组织。
- 对象可见性可控制其是否在界面展示。
- 流量监控仅支持交换机、路由器、防火墙、负载均衡四类对象与 NetFlow、sFlow 两种协议；不在支持范围内的对象或协议会被拒绝。
- 流量资产以「云区域 + 设备 IP」作为唯一约束；命中冲突时按权限校验后合并协议，且实例名称在同一对象下不可重复。
- 采样率统一以 effective_sampling_rate 消费：设备已上报则直接采用；否则按 SAMPLING_INTERVAL、SAMPLING_ALGORITHM、sampling_rate、samplingRate 的优先级归一化；仍缺失时回退到资产兜底采样率（默认 1000）。
- 流量资产变更（新增、绑定、删除）会触发对应云区域流量采集配置的环境变量刷新，使资产映射保持一致。
- 主机远程监控通过实例标识适配统一其存储/逻辑标识，复用同一实例时按插件隔离指标分组。
- Telegraf HTTP 类插件（主机、腾讯云、VMware）的登录凭据等敏感字段以加密环境变量形式下发，不在配置模板中明文保存。

## 5. 关键技术架构选择

- 指标统一存储于 VictoriaMetrics，指标定义以查询语句 + 维度组合表达。
- 采集支持多种采集器（HTTP / SNMP / Prometheus / NetFlow / sFlow 等），采集配置与节点关联，经节点代理执行。
- 配置模板以模板渲染方式生成实际采集配置；敏感凭据以加密环境变量注入而非写入明文模板。
- 流量监控复用 Telegraf 监听采集，按云区域下发资产映射环境变量，并以协议默认端口（NetFlow 2055、sFlow 6343）拼接监听接入地址。

## 6. 验收标准

- 用户可维护监控插件并完成配置、指标、采集三部分配置。
- 可创建并管理监控实例，按对象、分组、资产正确组织。
- 分组规则可按维度自动归组并按需同步组织归属。
- 监控对象树可维护并支持继承与排序。
- 用户可通过流量接入向导完成网络设备的资产绑定与设备导出引导，并能检测到该资产的实际流量数据。
- 重复绑定同一云区域 + IP 的流量资产时合并协议而非新建实例，采样率按既定优先级归一化后生效。
- 删除实例时其子配置与分组规则被同步清理，相关云区域流量采集配置随之刷新。
- Telegraf HTTP 插件凭据以加密方式下发，配置模板中不出现明文密码。
