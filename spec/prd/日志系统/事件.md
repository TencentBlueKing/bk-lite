# 日志系统 · 事件

## 1. 背景与目标

事件是日志侧的告警能力：用户通过策略定义基于日志模式/计数的触发条件，系统周期评估并生成告警，用于从日志中发现异常。

## 2. 范围定义

- 告警：列表、筛选、关闭、原始事件查看。
- 策略：基于日志的告警策略的新增、编辑、删除、启停。

## 3. 关键能力

### 3.1 告警

- 支持告警列表查看，按状态、级别、采集类型、时间范围、策略名筛选。
- 支持关闭告警、查看关联事件与原始事件数据快照。

### 3.2 策略

- 支持基于日志的告警策略增删改与启停。
- 触发条件基于日志模式查询 + 聚合规则（分组字段、计数/数值阈值比较），在数据窗口内满足条件则触发（如"最近 5 分钟错误数 > 10"）。
- 策略作用于所选日志分组，可全局或按采集类型。
- 支持配置检测频率与回看周期、告警名称与级别、可选的通知渠道与接收人。
- 支持启停，并记录最近执行时间。

## 4. 关键规则

- 日志策略基于日志计数/模式（文本）触发，绑定日志分组；区别于监控模块基于指标阈值（数值时序）的策略。
- 策略、分组、实例按组织隔离，受权限控制。

## 5. 关键技术架构选择

- 策略评估由 Celery 定时任务周期驱动。
- 告警原始数据与累积事件快照以压缩 JSON 存于对象存储。

## 6. 验收标准

- 用户可创建基于日志的告警策略并配置触发条件、周期与通知。
- 系统按周期评估日志并正确生成告警。
- 告警可查看关联事件与原始数据并关闭。
