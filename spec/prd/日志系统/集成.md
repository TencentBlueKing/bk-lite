# 日志系统 · 集成

## 1. 背景与目标

集成定义日志从哪里来、如何接入、如何分组，是日志数据的入口与组织底座。

## 2. 范围定义

- 日志集成：采集器类型与采集实例的配置管理。
- 日志接收：外部日志（Syslog / SNMP Trap）的接收。
- 日志分组：按规则把日志归入分组，供检索与策略引用。

## 3. 关键能力

### 3.1 日志集成

- 提供预置采集器类型（如 MySQL 慢日志、Docker 日志、Apache 访问日志、Syslog、SNMP Trap、Kubernetes 等），各含名称、图标、配置项与分类。
- 采集实例按"采集器类型 + 节点"部署。
- 配置：在节点上新增实例并设置采集参数（路径、解析规则、提取字段等），支持批量导入与批量编辑、编辑、删除。
- 简介：查看集成概览（采集器类型、实例数、最近更新）。

### 3.2 日志接收

- 支持配置 Syslog 接收端点供外部系统推送日志。
- 支持 SNMP Trap 接收并转换为日志。
- 支持按组织管理接收器的新增、编辑、删除。

### 3.3 日志分组

- 支持按规则（字段匹配、包含、正则等）定义日志分组。
- 分组按组织划分，可跨多个采集类型。
- 检索与策略均引用日志分组，实现跨源检索与多源告警。

## 4. 关键规则

- 采集实例按"采集器类型 + 节点"唯一对应。
- 分组、实例、接收器按组织隔离，受权限控制。

## 5. 关键技术架构选择

- 接入管线：采集实例采集的日志与外部接收的日志统一汇入 VictoriaLogs；采集器支持 Filebeat（apache/nginx/mysql/redis/kafka 等中间件日志）、Vector（syslog/file/docker/kubernetes）、Packetbeat（网络流量）、Auditbeat（文件完整性）、Winlogbeat（Windows 事件日志）、Snmptrapd（SNMP Trap）共 6 类。
- 日志存储与检索基于 VictoriaLogs。

## 6. 验收标准

- 用户可配置采集器实例并完成采集参数设置与批量导入。
- 外部日志可经 Syslog / SNMP Trap 接收并入库。
- 日志分组可按规则定义并被检索与策略引用。
