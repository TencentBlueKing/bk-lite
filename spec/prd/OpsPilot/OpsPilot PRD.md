# OpsPilot PRD

- 文档日期：2026-03-03

- 依据：项目代码（`server/apps/opspilot`、`web/src/app/opspilot`、`mobile/src/api/bot.ts`）与官网功能文档（https://bklite.ai/docs/opspilot/feature）

## 1. 背景与目标

OpsPilot 用于构建可配置、可发布、可运营的智能应用能力，支撑从模型接入、知识管理、智能体编排到多渠道对话交付的全链路。

目标：
- 统一管理 LLM/Embed/Rerank/OCR 等基础模型能力，并为上层能力复用。
- 提供工具接入与知识库管理能力，保障智能体可调用外部数据与私域知识。
- 提供智能体与 ChatFlow 工作流能力，支持多触发方式与多渠道发布。
- 提供会话日志、执行日志与统计能力，满足运营分析与审计追踪。

## 2. 范围定义

### 2.1 In Scope

- 模型管理：LLM、Embed、Rerank、OCR、模型分组。
- 工具管理：工具列表、新增/编辑/删除、MCP 工具拉取。
- 知识库管理：知识库、文档、问答对、知识图谱、检索与重排配置。
- 智能体管理：智能体列表、创建、设置、执行、模板能力。
- 工作台（Studio）：应用机器人、ChatFlow 设计、渠道配置、日志与统计。
- 对话应用能力：会话列表、会话消息、引导语查询、会话删除。
- 对外触发能力：RESTful/OpenAI/AG-UI/定时/企业渠道触发。

### 2.2 Out of Scope

- 非 OpsPilot 模块的资产管理、作业编排与监控能力。
- 第三方平台内的组织权限模型定义。
- 与 OpsPilot 无关的通用系统管理页面。

## 3. 目标用户

- AI 平台管理员：维护模型、工具、知识库与权限分组。
- 智能应用配置人员：创建智能体、编排 ChatFlow、发布渠道。
- 业务使用者：在 Web/移动端/企业渠道中发起对话与查询。
- 运维与审计人员：查看执行日志、会话记录与用量统计。

## 4. 信息架构

### 4.1 一级模块

- 模型（Provider）
- 工具（Tool）
- 知识库（Knowledge）
- 智能体（Skill）
- 工作台（Studio）
- 设置（Settings）

### 4.2 工作台内结构

- ChatFlow 画布：节点编排、节点配置、执行测试

## 5. 核心功能需求

### 5.1 模型管理

- 支持 LLM、Embed、Rerank、OCR 四类模型的统一管理。
- 支持模型新增、编辑、删除、启停与分组展示。
- 支持模型分组维护与排序调整。
- 支持模型密钥类配置加密存储。
- 支持按团队分组控制模型可见范围。

### 5.2 工具管理

- 支持工具列表展示与按名称/标签检索。
- 支持工具新增、编辑、删除。
- 支持配置 MCP 链接与变量。
- 变量支持类型区分（含文本与密码类型）。
- 支持从 MCP 拉取可用子工具能力。
- 支持按团队分组控制工具可见范围。

### 5.3 知识库管理

- 支持知识库新增、编辑、删除、详情查看。
- 支持知识来源：本地文件、网页链接、自定义文本。
- 支持文档预处理、文本提取、分块预览与训练。
- 支持文档训练状态展示（待处理、分块中、训练中、就绪、错误）。
- 支持知识库检索配置：检索模式、阈值、返回规模。
- 支持 Rerank 模型与召回配置。
- 支持问答对生成、导入、编辑、删除、预览。
- 支持知识图谱创建、查看、更新与社区重建。

### 5.4 智能体管理

- 支持智能体列表展示、搜索与置顶。
- 支持智能体创建、编辑、删除。
- 支持智能体模板列表与模板化创建。
- 支持智能体基础配置：模型、提示词、温度、简介、分组。
- 支持智能体增强配置：聊天历史、RAG、工具。
- 支持每知识库独立阈值配置与严格模式配置。
- 支持智能体执行（流式响应）与 AG-UI 协议执行。

### 5.5 工作台（Studio）与 ChatFlow

- 支持应用机器人列表管理与上线/下线。
- 支持应用类型：Pilot、LobeChat、ChatFlow。
- 支持 ChatFlow 画布编排与节点配置。
- 支持节点类别：触发、应用、智能体、逻辑判断、动作。
- 支持节点执行测试与执行过程查看。
- 支持执行日志检索、会话日志查看、输出数据查看。
- 支持统计视图（会话量、活跃用户、Token 消耗等）。

### 5.6 渠道与会话能力

- 支持渠道类型：企业微信、企业微信机器人、微信公众号、钉钉、Web、GitLab。
- 支持渠道参数配置并对敏感字段脱敏展示。
- 支持 Web/Mobile 会话列表查询。
- 支持会话历史消息查询。
- 支持技能引导语查询。
- 支持按会话删除历史记录。

### 5.7 对外接口与触发

- 支持 OpenAI 风格聊天补全接口。
- 支持 LobeChat 兼容聊天补全接口。
- 支持 ChatFlow 执行接口（按 bot_id/node_id）。
- 支持企业微信、微信公众号、钉钉触发入口。
- 支持 RESTful、OpenAI、AG-UI、定时触发等工作流执行类型。

## 6. 关键业务规则

### 6.1 可见性与权限规则

- 资源可见范围以团队分组为边界。
- 非超级管理员仅可访问有权限团队内资源。
- 关键资源编辑需通过模块权限点校验。

### 6.2 应用与工作流规则

- ChatApplication 由 BotWorkFlow 保存时自动同步，不通过应用接口直接创建。
- 应用上线后进入可对外服务状态；下线后停止对外响应。
- 工作流执行结果需保留主任务与节点级结果。

### 6.3 会话与日志规则

- 会话列表按用户维度返回，并按最近会话时间倒序。
- 会话消息支持 Web 与 Mobile 入口类型查询。
- 会话删除仅作用于当前用户指定会话。
- 日志能力覆盖触发日志、执行明细与会话日志。

### 6.4 知识库规则

- 知识库名称需唯一。
- 变更知识库基础模型后需进行文档重训练。
- 知识库删除前需检查是否被智能体引用。
- 文档状态机需覆盖从待处理到可用/失败的全生命周期。

### 6.5 智能体规则

- 智能体支持置顶状态切换。
- 关闭 RAG 时需清空关联知识库与阈值映射。
- 工具变量中的密码类型字段需加密保存。

### 6.6 渠道安全规则

- 渠道配置中的密钥字段必须加密存储。
- 渠道配置对外返回时敏感字段以掩码显示。

## 7. 非功能需求

- 可用性：核心配置流程支持在单模块内闭环完成。
- 安全性：模型密钥、工具密码、渠道密钥全程加密或脱敏。
- 可审计性：执行日志、会话日志、统计数据可回溯。
- 可扩展性：支持新增模型类型、工具能力与渠道类型。

## 8. 验收标准

- 用户可完成模型、工具、知识库、智能体、工作台五大模块的基本 CRUD 与配置。
- 用户可基于 ChatFlow 配置一条可执行流程并进行测试执行。
- 用户可完成至少一个渠道配置并触发对话交互。
- 用户可在会话列表查看历史并查看会话消息详情。
- 用户可在日志/统计页面查看执行轨迹与关键指标。
- 知识库可完成“上传 -> 处理 -> 训练 -> 可检索”闭环。
- 智能体可完成“创建 -> 配置 -> 执行 -> 结果输出”闭环。
- 关键敏感配置项在读取时均不以明文展示。
