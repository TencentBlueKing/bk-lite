# OpsPilot · 知识库

## 1. 背景与目标

知识库管理 RAG 所需的知识来源：文档摄入、切分嵌入、问答对与知识图谱，并提供检索测试，是智能体回答准确性的来源。

## 2. 范围定义

- 文档：多来源文档摄入与管理。
- 切分与嵌入：分块与向量化。
- 问答对：高精度问答匹配。
- 知识图谱：实体关系抽取。
- 检索测试：检索效果预览。

## 3. 关键能力

- 文档摄入来源：
  - 文件：Markdown、Word、Excel、CSV、PPT、PDF、TXT、图片（可选 OCR）。
  - 网页：抓取并可按计划自动同步，支持多层级。
  - 手动文本：直接录入内容。
  - 问答对：以问答元组实现高精度匹配。
- 切分与嵌入：支持固定大小切分（含重叠）或语义切分，按配置的嵌入模型向量化。
- 检索模式：朴素 RAG（分块检索）+ 问答对匹配 + 可选知识图谱。
- 检索：基于相似度阈值或 Top-K，可选重排模型；按知识库设置阈值。
- 知识图谱：可选抽取实体关系图用于结构化检索。
- 检索测试：在知识库上测试检索，预览召回结果。

## 4. 关键规则

- 知识库按组织隔离。
- 文档索引为异步处理，带切分进度。
- 知识图谱为可选能力，需显式开启。

## 5. 关键技术架构选择

- 向量存储基于 PostgreSQL pgvector，配合元数据检索。
- 文档嵌入、网页同步、检索测试等经异步任务执行。

## 6. 验收标准

- 用户可通过文件/网页/手动/问答对四种来源摄入知识。
- 文档可正确切分、嵌入并建立索引。
- 检索测试可预览召回，支持阈值、Top-K 与重排。
- 知识图谱可按需开启并用于检索。
