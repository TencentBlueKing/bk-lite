# MLOps · 异常检测

## 1. 背景与目标

异常检测面向时间序列数据，训练并发布异常识别能力，供监控、告警等模块对时序指标做异常判定。

## 2. 范围定义

MLOps 各能力共享统一管道：数据集 → 数据标注 → 训练任务 → 能力发布 → 算法配置。本能力覆盖该管道在时序异常检测场景下的应用。

- 数据集：时序数据的上传与版本管理。
- 数据标注：标记时序中的异常点/范围。
- 训练任务：选择算法、配置超参、运行与跟踪。
- 能力发布：将训练模型发布为推理服务。
- 算法配置：维护可用算法及其超参表单。

## 3. 关键能力

### 3.1 数据集

- 创建数据集容器并管理多版本，数据为时序 CSV（时间戳 + 数值）。
- 数据可标记为训练集、验证集、测试集；发布时生成压缩包、版本号与统计元数据。

### 3.2 数据标注

- 在时序图表上标记异常点/范围（支持拖拽框选），保存为带标签的样本。

### 3.3 训练任务

- 创建任务时指定算法、数据集版本与超参配置；超参搜索按设定的评估次数进行。
- 训练状态在待执行、运行中、完成/失败之间流转，由实验跟踪同步。

### 3.4 能力发布

- 从完成的训练任务选择模型版本，配置服务名与端口，发布为容器化推理端点；创建后自动启动对应容器。
- 支持服务生命周期操作：启动、停止、删除（移除容器），以及在线推理（predict）。
- 列表与详情实时同步容器运行状态（container_info）；变更模型版本、关联训练任务或端口等配置时自动重启服务。
- 支持版本控制（最新或指定版本）；其他模块经动态生成的推理地址调用。

### 3.5 算法配置

- 维护本能力可用算法清单，每个算法记录展示名、适用说明、镜像与超参表单定义。
- 前端按表单定义动态渲染超参，后端训练前校验并注入。

## 4. 关键规则

- 属传统机器学习：最终训练前合并训练集与验证集。
- 所有资源按组织隔离，训练数据继承所属数据集的组织权限。

## 5. 关键技术架构选择

- 实验跟踪基于 MLflow，每个训练任务对应一个实验，按命名规则便于溯源。
- 训练与推理的容器编排经 webhookd 执行，支持 Docker 与 Kubernetes 两种运行时；训练/推理镜像由算法配置（AlgorithmConfig）动态指定。
- 推理地址不持久化，按部署环境（K8s/Docker/主机）在调用时动态生成。
- 数据集与训练配置存于对象存储，元数据存于数据库；训练任务经异步任务执行并同步状态。

## 6. 验收标准

- 用户可上传时序数据、标注异常并训练模型。
- 训练任务可配置算法与超参并跟踪状态。
- 训练模型可发布为推理服务并被监控/告警模块调用。
