# MLOps · 文本分类

## 1. 背景与目标

文本分类面向文本数据，训练并发布分类能力，供日志/告警等场景做文本归类。

## 2. 范围定义

MLOps 各能力共享统一管道：数据集 → 数据标注 → 训练任务 → 能力发布 → 算法配置。本能力覆盖该管道在文本分类场景下的应用。

- 数据集：文本数据的上传与版本管理。
- 数据标注：为文本分配分类标签。
- 训练任务：选择算法、配置超参、运行与跟踪。
- 能力发布：将训练模型发布为推理服务。
- 算法配置：维护可用算法及其超参表单。

## 3. 关键能力

### 3.1 数据集

- 创建并管理多版本数据集，数据为带文本与标签列的 CSV。
- 数据可标记为训练/验证/测试集；发布生成压缩包、版本号与统计元数据。

### 3.2 数据标注

- 为文本行分配分类标签（表格化标注）。

### 3.3 训练任务

- 指定算法、数据集版本与超参，按设定评估次数搜索超参；状态由实验跟踪同步。

### 3.4 能力发布

- 选择训练模型版本，配置服务名与端口，发布为容器化推理端点，支持版本路由。

### 3.5 算法配置

- 维护可用算法清单与超参表单定义，前端动态渲染、后端校验注入。

## 4. 关键规则

- 属传统机器学习：最终训练前合并训练集与验证集。
- 资源按组织隔离，训练数据继承所属数据集的组织权限。

## 5. 关键技术架构选择

- 实验跟踪基于 MLflow；模型经 BentoML 容器化发布，推理地址按部署环境动态生成。
- 数据与配置存于对象存储，训练任务异步执行并同步状态。

## 6. 验收标准

- 用户可上传文本数据、标注标签并训练模型。
- 训练任务可配置算法与超参并跟踪状态。
- 训练模型可发布为推理服务并被相关场景调用。
