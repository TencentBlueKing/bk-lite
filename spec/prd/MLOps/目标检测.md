# MLOps · 目标检测

## 1. 背景与目标

目标检测面向图像数据，训练并发布目标检测能力，供视觉事件检测、图片告警验证等场景调用。

## 2. 范围定义

MLOps 各能力共享统一管道：数据集 → 数据标注 → 训练任务 → 能力发布 → 算法配置。本能力覆盖该管道在目标检测场景下的应用。

- 数据集：图像与标注数据的上传与版本管理。
- 数据标注：绘制边界框与类别。
- 训练任务：选择算法、配置超参、运行与跟踪。
- 能力发布：将训练模型发布为推理服务。
- 算法配置：维护可用算法及其超参表单。

## 3. 关键能力

### 3.1 数据集

- 创建并管理多版本数据集，数据为图像 + 标注（边界框坐标）压缩包。
- 数据按训练/验证/测试集划分；发布生成压缩包、版本号与统计元数据。

### 3.2 数据标注

- 在图片上绘制边界框并赋类别，集成图片标注工具，支持标注格式的导入导出。

### 3.3 训练任务

- 指定算法、数据集版本与超参，按设定评估次数搜索超参；状态由实验跟踪同步。

### 3.4 能力发布

- 选择训练模型版本，配置服务名与端口，发布为容器化推理端点，支持版本路由。

### 3.5 算法配置

- 维护可用算法清单与超参表单定义，前端动态渲染、后端校验注入。

## 4. 关键规则

- 属深度学习：训练集/验证集/测试集保持分开，按目录结构组织。
- 资源按组织隔离，训练数据继承所属数据集的组织权限。

## 5. 关键技术架构选择

- 实验跟踪基于 MLflow；模型经 BentoML 容器化发布，推理地址按部署环境动态生成。
- 数据与配置存于对象存储，训练任务异步执行并同步状态。

## 6. 验收标准

- 用户可上传图像数据集、标注边界框并训练模型。
- 训练任务可配置算法与超参并跟踪状态。
- 训练模型可发布为推理服务并被相关场景调用。
