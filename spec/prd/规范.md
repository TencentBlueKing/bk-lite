# PRD 编写规范

## 1. 目标
- PRD 仅描述产品的最终状态与稳定规则，用于对齐“要做成什么”。

## 2. 必须遵循
- 只写结果，不写过程：
  - 写“当前应是什么”。
  - 不写“从什么改成什么”。
- 不记录迁移过程、替换过程、讨论过程。
- 文案使用结论式短句，避免实现细节。

## 3. 变更记录边界
- 变更过程（例如隐藏/新增/替换/迁移）必须写入同业务目录下的更新日志。
- PRD 正文只保留变更后的规则与口径。

## 4. 推荐结构
- 背景与目标（简短）
- 范围定义
- 关键能力
- 关键规则
- 验收标准

## 5. 禁止项
- 禁止在 PRD 中使用“本次调整为/由…改为…”等过程化措辞。
- 禁止堆砌技术实现细节（接口字段、代码路径、算法细节）。
