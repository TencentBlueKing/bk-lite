# CMDB · 搜索

## 1. 背景与目标

搜索面向"我不确定资产在哪个模型下"的场景，提供跨模型的全文检索，先看命中分布再深入查看，帮助用户快速定位资产。

## 2. 范围定义

- 全文检索：按关键词跨模型检索资产。
- 命中统计：按模型汇总命中数量。
- 分模型结果：按模型分页查看命中结果。
- 检索历史：保存与复用最近检索词。
- 结果查看：在结果旁查看资产详情并跳转。

## 3. 关键能力

- 输入关键词后跨模型全文检索，先返回总命中数与各模型命中数（以模型标签呈现）。
- 选择某个模型标签后，分页查看该模型的命中结果。
- 支持精确匹配开关，控制检索是否区分大小写（默认不区分）。
- 检索结果左侧为命中列表，右侧查看选中资产详情，并可跳转到资产详情页。
- 命中项中实例名与标签字段始终展示，匹配字段高亮。
- 保存最近检索历史，可一键复用或清除。

## 4. 关键规则

- 检索结果受用户组织权限与实例级权限过滤。
- 检索历史保存在用户本地，不是后端持久化的常用条件。
- 全文检索以"按模型统计 + 分模型分页"的方式组织，不依赖单一模型入口。

## 5. 关键技术架构选择

- 全文检索基于图数据库 FalkorDB 的资产数据，先做按模型的命中统计，再按模型分页取数，避免一次性返回全部命中。

## 6. 验收标准

- 用户输入关键词后可先看到按模型汇总的命中分布，再进入某模型查看分页结果。
- 精确匹配开关能正确切换大小写敏感行为。
- 检索结果可查看详情并跳转，且严格限定在用户权限范围内。
