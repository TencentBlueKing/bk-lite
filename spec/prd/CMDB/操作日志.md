# CMDB · 操作日志

## 1. 背景与目标

操作日志是 CMDB 的变更审计中心，记录模型与资产相关操作，并以"变更场景"标注每次变化的业务含义，支撑追溯与排查。

## 2. 范围定义

- 变更记录列表：分页查看。
- 多维筛选：按操作类型、变更场景、操作人、时间范围、关键词筛选。
- 详情查看：查看单条记录的前后快照。
- 导出：按当前筛选导出变更记录。

## 3. 关键能力

- 支持变更记录的分页列表查看，默认按时间倒序。
- 支持按以下维度筛选：操作人、操作类型（新增/更新/删除/执行/建关系/删关系）、变更场景、关键词、时间范围。
- 支持查看单条记录的操作人、时间、操作类型、变更场景及前后数据快照。
- 支持按当前筛选条件导出变更记录。

### 变更场景

- 变更场景共 6 类，表示"这次变化属于什么业务含义"：
  - 设备流转
  - 关系变更
  - 普通属性变更
  - 自动采集
  - 模型管理变更
  - 自定义上报变更
- 场景由系统在变更产生时自动判定。
- 用户仅可在实例编辑时，于"普通属性变更"和"设备流转"两类之间轻量修正；非法选择回落为普通属性变更。
- 其余场景由系统锁定，不可修改。

## 4. 关键规则

- 变更场景是唯一的分类口径，系统不另设独立的"来源分类"维度。
- 关系操作、自动采集/自动化、模型管理等入口的场景由系统直接写入且不可修改。
- 操作日志列表默认聚焦实例视角，默认不展示模型管理变更，用户可手动加入该场景查看。
- 变更记录列表按时间倒序排列。
- 导出单次最多 5 万行，并遵循当前筛选条件。

## 5. 关键技术架构选择

- 变更记录保留操作类型、变更场景、操作人与前后数据快照，作为审计与历史展示的统一来源。

## 6. 验收标准

- 用户可分页查看变更记录并按多维度筛选。
- 变更场景由系统正确判定，固定入口场景不可改，实例编辑可在相邻两类场景间轻量修正。
- 操作日志默认聚焦实例视角，模型管理变更默认不混入、可手动加入。
- 变更记录可按筛选条件导出，且不超过单次行数上限。
