# 作业管理 · 作业模版

## 1. 背景与目标

作业模版沉淀可复用的脚本与 Playbook，供作业执行引用，实现运维操作的标准化与复用。

## 2. 范围定义

- 脚本库：脚本的管理与参数化。
- Playbook 库：Playbook 的管理与版本化。

## 3. 关键能力

### 3.1 脚本库

- 支持脚本的创建、编辑、删除、批量删除。
- 脚本类型支持 shell、python、powershell、bat。
- 支持参数化：脚本内容用模板语法，参数以结构化方式定义（名称、标签、说明、默认值、是否加密），可加密存储。
- 支持脚本级默认超时；标记为内置的脚本不可删除。
- 按组织隔离。

### 3.2 Playbook 库

- Playbook 以压缩包形式上传（含 playbook 与说明文档），仅支持 .zip、.tar.gz、.tgz 格式。
- 上传时自动提取名称、版本、说明内容、文件清单与参数定义。
- 上传（含新建与版本升级）前对压缩包进行体量校验，超限直接拒绝并返回校验错误：压缩包整体最大 20MB、包内文件数不超过 2000 个、单个文件最大 5MB、解压后总量最大 50MB。
- 支持多版本共存（同名不同版本）。
- 删除时级联清理存储文件。

## 4. 关键规则

- 内置脚本不可删除。
- 脚本参数支持加密存储，敏感参数受保护。
- 模版按组织隔离。
- Playbook 压缩包上传受体量上限约束：超出压缩包大小（20MB）、文件数（2000 个）、单文件大小（5MB）或解压总量（50MB）任一上限时，不予处理并提示对应原因，避免超大或解压膨胀的压缩包进入系统。

## 5. 关键技术架构选择

- Playbook 压缩包与脚本密钥文件存于对象存储，并具备自动清理机制。
- 参数解析与注入分阶段进行，支持回放与审计。

## 6. 验收标准

- 用户可管理脚本库并定义参数化脚本。
- Playbook 可上传、自动解析元数据并多版本共存；超出体量上限的压缩包在新建或版本升级时被拒绝并给出明确原因。
- 模版可被作业执行正确引用。
