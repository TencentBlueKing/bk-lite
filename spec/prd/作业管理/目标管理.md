# 作业管理 · 目标管理

## 1. 背景与目标

目标管理维护作业执行的目标主机及其连接凭据，是作业执行的对象底座，并可与节点管理打通复用已纳管资产。

## 2. 范围定义

- 目标：手动目标的定义与管理。
- 凭据：SSH 与 WinRM 连接凭据管理。
- 目标操作：连通性测试、批量导入删除、节点同步。

## 3. 关键能力

### 3.1 目标

- 支持手动目标的定义，记录名称、IP（IPv4/IPv6）、操作系统，并关联云区域。

### 3.2 凭据

- 凭据来源：手动录入，或关联凭据管理系统。
- SSH 凭据：端口（默认 22）、密码或密钥认证、密钥文件与口令。
- WinRM 凭据（Windows）：端口（默认 5986）、协议、传输方式、证书校验开关。

### 3.3 目标操作

- 支持 SSH/WinRM 连通性测试。
- 支持批量导入与删除目标。
- 支持从节点管理同步节点，直接复用已纳管资产作为目标。

## 4. 关键规则

- 凭据中的密码与密钥口令加密存储。
- 目标按组织隔离。

## 5. 关键技术架构选择

- 连通性测试与执行经统一执行服务驱动（SSH/WinRM）。
- 与节点管理集成，可查询节点作为执行目标。

## 6. 验收标准

- 用户可定义手动目标并配置 SSH/WinRM 凭据。
- 连通性测试可返回成功/失败信息。
- 可批量导入目标并从节点管理同步复用资产。
