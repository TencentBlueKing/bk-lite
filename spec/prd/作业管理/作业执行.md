# 作业管理 · 作业执行

## 1. 背景与目标

作业执行是作业管理的核心，提供脚本快速执行、文件分发、定时任务与执行记录，让用户在目标主机上安全、可追溯地执行运维操作。

## 2. 范围定义

- 快速执行：在目标上即时执行脚本。
- 文件分发：向目标分发文件。
- 定时任务：单次或周期调度执行。
- 作业记录：执行历史查询、详情、重跑与取消。

## 3. 关键能力

### 3.1 快速执行

- 三种来源：选择脚本库脚本、选择 Playbook、临时输入脚本（shell/python/powershell/bat）。
- 目标来源：手动指定，或从节点管理导入。
- 临时脚本解释器：Shell 类型默认解释器为 bash；若脚本首行声明 Shebang（如 `#!/bin/bash`、`#!/usr/bin/env python3`），自动按 Shebang 识别解释器执行，识别结果限定于受支持的解释器白名单（sh/bash/python/python3/powershell/pwsh），白名单外则回退到默认解释器。
- 支持参数化（模板语法注入参数）与自定义超时。
- 执行前对脚本内容做高危命令检测，命中禁止规则则拒绝执行。
- 异步执行，创建后即返回执行标识，可查询进度。

### 3.2 文件分发

- 上传文件并分发到目标目录，支持多文件。
- 覆盖策略：覆盖或跳过。
- 执行前对目标路径做高危路径检测，命中禁止规则则拒绝。

### 3.3 定时任务

- 调度类型：单次（指定时间）或周期（cron 表达式）。
- 并发策略：跳过、排队、并发运行。
- 可关联脚本库、Playbook 或临时脚本，支持启停，记录最近执行时间与执行次数。

### 3.4 作业记录

- 按作业类型、状态、时间范围查询执行历史。
- 查看批量视图（目标总数、成功/失败数）与单条详情（关联脚本/Playbook、参数、各目标结果）。
- 对于 Playbook 来源的执行，记录并展示执行时所用的 Playbook 版本，并支持在记录中预览该 Playbook 的文件内容。
- 支持基于历史记录重新执行，支持取消处于待执行/执行中的任务。
- 对外开放接口（经 NATS，供补丁管理等第三方 App 调用）：触发脚本执行、触发文件分发、批量查询作业状态、查询作业详情、查询目标列表。

## 4. 关键规则

- 目标来源影响执行路径：节点管理来源优先经 Sidecar 执行；手动来源 Windows 强制经 Ansible/WinRM。
- 解释器选择以脚本 Shebang 优先、默认解释器（Shell 为 bash）兜底；经 Ansible 执行时仅对 sh/bash 切换 Shell 可执行程序，python3/powershell 等非 Shell 解释器改以脚本入口方式执行，避免破坏 Ansible 连接。
- Playbook 执行版本在创建执行与重新执行时一并记录，作为执行记录的不可变快照。
- 高危命令/路径检测在执行前置校验，命中禁止策略直接拒绝。
- 执行终态为成功、失败、超时、取消；待执行与执行中为中间态。
- 作业按组织隔离，关键操作记录操作日志，操作受权限控制。
- 分发文件均设过期时间（默认 7 天，可设 1-365 天），由每日 00:00 定时任务到期后自动清理，不存在永久保存。

## 5. 关键技术架构选择

- 执行基于 Celery 异步任务，多目标经线程池并发执行，单次执行聚合各目标结果。
- 两种执行驱动：Sidecar（经节点管理代理）与 Ansible（含 Windows WinRM）。
- 周期任务由定时调度器按 cron 触发，并按并发策略控制重复执行。
- 可选回调通知，回调带签名校验。

## 6. 验收标准

- 用户可通过脚本库/Playbook/临时脚本完成快速执行并查看进度。
- 临时 Shell 脚本默认以 bash 执行；声明 Shebang 时按白名单内的解释器执行。
- 文件分发可按覆盖策略分发到目标并受高危路径保护。
- 定时任务可按单次/周期调度并正确执行并发策略。
- 执行记录可查询、查看详情、重跑与取消；Playbook 来源记录可查看执行时版本并预览 Playbook 文件。
