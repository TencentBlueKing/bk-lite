# 告警中心 · 配置

## 1. 背景与目标

配置集中管理告警中心的运行规则：相关性规则、告警分派、屏蔽策略、全局配置与操作日志，决定告警如何聚合成事故、如何通知、如何抑制以及如何审计。

## 2. 范围定义

- 相关性规则：定义告警聚合为事故的规则。
- 告警分派：定义告警路由到处理人并通知。
- 屏蔽策略：在维护窗口或已知问题期间抑制通知。
- 全局配置：通知频率、默认接收人与渠道等系统默认值。
- 操作日志：告警中心操作审计。

## 3. 关键能力

### 3.1 相关性规则

- 定义哪些告警应聚合关联为同一事故，支持新增、编辑、删除、启停。
- 两种规则类型：
  - 智能降噪：按匹配条件与分组维度归并相关告警，支持观察窗口（确认前为会话告警）与不活跃超时后的自动关闭。
  - 缺失检测：基于 cron 检测预期事件缺失（心跳检查），支持宽限期、激活方式、自动恢复与告警模板。
- 规则可指定生效团队与接收事故的分派团队。

### 3.2 告警分派

- 把告警路由到指定处理人并通过渠道通知，支持新增、编辑、删除、启停。
- 匹配方式：全部告警或按条件筛选。
- 支持通知场景（分派时/恢复时）、通知频率（最大次数 + 间隔）、生效时段与星期过滤。
- 提醒任务按间隔轮询，未达上限则持续提醒。

### 3.3 屏蔽策略

- 在维护窗口或已知问题期间抑制匹配告警的通知，支持新增、编辑、删除、启停。
- 匹配方式：全部或按条件筛选；抑制时间支持全天、时间段、按星期/月。

### 3.4 全局配置

- 配置通知频率阈值、默认接收人、默认通知渠道等系统默认值。
- 全局配置作用于所有告警，可被分派规则覆盖。

### 3.5 操作日志

- 记录对事件、告警、事故、系统配置的新增、修改、删除、执行操作。
- 记录操作人、动作、对象类型、对象、摘要与时间，用于审计。

## 4. 关键规则

- 相关性、分派、屏蔽规则均为模板式匹配，规则变更不回溯历史告警，仅作用于后续告警与状态变化。
- 屏蔽在通知分发前评估，命中则抑制通知。
- 分派在告警创建或状态变化时执行匹配。

## 5. 关键技术架构选择

- 相关性引擎实时对新增/更新告警做规则匹配，命中后按分组与策略关联到事故。
- 通知分发结合分派规则、提醒轮询任务与屏蔽规则共同决定是否及如何通知。

## 6. 验收标准

- 用户可配置相关性规则并使匹配告警正确聚合为事故。
- 分派规则可把告警路由给处理人并按频率与时段通知。
- 屏蔽策略可在指定时间窗抑制匹配告警的通知。
- 全局配置作为默认值生效，可被分派规则覆盖。
- 操作日志可完整审计告警中心的关键操作。
