# 告警中心 · 告警

## 1. 背景与目标

告警是来自各告警源的原始事件按指纹聚合后的统一记录；此外，"即时告警"策略命中的事件可在入库同一环节直接生成告警，无需聚合等待。本菜单提供告警的查看、筛选、趋势观测与生命周期处置。

## 2. 范围定义

- 告警列表：分页查看与多维筛选。
- 趋势图：按时间窗口的告警频次。
- 告警详情：原始事件、邻近告警。
- 生命周期处置：认领、分派、改派、关闭、批量操作。

## 3. 关键能力

- 支持告警列表查看，展示状态、级别、标题、持续时长、事件数、资源与指标项。
- 支持按状态、级别、来源、资源、时间范围筛选，支持"我的告警"过滤。
- 支持查看所选时间段的告警频次趋势（按状态堆叠）。
- 支持查看告警详情：原始事件（含完整事件数据）、±60 分钟窗口内的邻近告警。
- 支持"即时告警"：命中即时告警策略的事件实时直接生成告警，无需等待聚合周期。
- 支持告警生命周期操作：认领、分派、改派、关闭，支持批量操作。

## 4. 关键规则

- 告警由原始事件按指纹聚合而成：相同指纹（基于服务、位置、指标项、资源等维度）的事件归并为一条告警。
- 告警继承事件的标题、内容、级别、起止时间与资源/指标项/位置等信息。
- 即时告警：当事件命中"即时告警"类型策略时，在事件入库同一环节直接生成一条告警，不进入指纹聚合主路径。每条命中事件对应独立的一条即时告警（事件数恒为 1），告警级别强制等同事件级别；标题与内容按策略配置的模板渲染，未配置时沿用事件原始标题与内容。
- 即时告警按"策略 + 事件"唯一标识去重：同一策略对同一事件重复投递不会产生重复告警；其去重标识与聚合告警相互隔离，互不影响。
- 即时告警生成后初始为未分派状态，并自动触发分派规则匹配；即时告警不参与自动关闭，仅可经人工处置或自动恢复结束生命周期。
- 告警状态：未分派、待处理、处理中、已解决、已关闭、自动关闭、自动恢复。
- 自动关闭由相关性规则配置的超时触发，仅作用于聚合告警；会话告警（观察中/已确认/已恢复）在确认前为虚拟告警，不计入常规列表。
- 告警与组织关联，可见性按团队隔离。

## 5. 关键技术架构选择

- 接入管线：原始事件经去重键去重后归入事件，再由指纹引擎按维度聚合为告警（一个指纹对应一条告警、多条事件），该路径异步周期化执行。
- 即时告警旁路：与聚合主路径并行，在事件入库后实时按内存匹配命中"即时告警"策略并直接落库告警。聚合路径已显式排除即时告警策略，两条路径完全互斥。该旁路对主流程无侵入——未配置即时告警策略时零开销，且自身异常不阻断事件入库与聚合。
- 即时告警写入采用同步优先、异步兜底：单次命中量在阈值内（50 条）时同步落库以避免中转开销，超阈值时降级到异步任务批量处理；单次调用命中数设有上限（5000 条）保护。
- 规则（相关性、分派、屏蔽）为模板式匹配，规则变更不回溯历史告警，仅作用于后续状态变化。

## 6. 验收标准

- 用户可查看、筛选告警并查看频次趋势。
- 相同指纹的事件正确聚合为同一条告警。
- 命中即时告警策略的事件实时生成独立告警，级别等同事件级别、事件数为 1，不进入聚合且不被自动关闭。
- 同一策略对同一事件重复投递不产生重复即时告警。
- 告警可完成认领、分派、改派、关闭与批量操作。
- 告警按团队范围正确隔离。
