# 运营分析（Operation Analysis）PRD

## 1. 背景与目标

运营分析用于将多源运维数据统一编排为可消费的分析视图，帮助运维与平台团队在同一工作台完成“观察、对比、定位、决策”。

本模块的产品目标如下：

- 提供统一的数据接入与管理入口，降低多系统数据使用门槛。
- 提供目录化分析空间，支持团队按业务场景组织分析资产。
- 提供多种分析载体（仪表盘、拓扑图、架构图），支持不同分析表达方式。
- 提供组织与功能权限控制，保障团队协作的可控与可审计。

## 2. 范围定义

### 2.1 在范围内

- 运营分析主视图（View）。
- 管理页：数据源管理、命名空间管理。
- 分析资产：目录、仪表盘、拓扑图、架构图。
- 数据连接资产：数据源、数据源标签、命名空间。
- 权限控制：查看/新增/编辑/删除等操作权限。

### 2.2 不在范围内

- 移动端运营分析能力。
- 与本模块无关的跨产品编排功能。
- 模块外的算法能力定义与数据质量治理流程。

## 3. 用户与角色

### 3.1 目标用户

- 平台运维人员：搭建分析看板与图谱。
- 业务运维人员：按团队查看并使用已有分析资产。
- 系统管理员：维护命名空间、数据源等基础配置。

### 3.2 角色能力边界

- 管理员角色具备完整运营分析菜单与操作能力。
- 普通角色默认具备查看类能力，可在授权后执行新增/编辑/删除。
- 权限粒度覆盖：
	- 主视图：查看、目录增改删、图表增改删。
	- 管理页：数据源增改删查、命名空间增改删查。

## 4. 核心能力

### 4.1 分析工作台

- 提供左侧目录树 + 右侧内容区的双栏布局。
- 内容区支持三类分析对象切换展示：
	- 仪表盘
	- 拓扑图
	- 架构图
- 支持侧边栏折叠，保证复杂场景下的可视区域。
- 用户切换分析对象或离开当前内容时，若存在未保存变更，系统进行二次确认。

### 4.2 目录与分析资产管理

- 支持在目录下创建子目录与分析对象。
- 支持目录、仪表盘、拓扑图、架构图的编辑与删除。
- 支持按层级结构浏览分析资产，目录树统一展示四类节点。
- 删除目录时遵循“存在可用子内容不可直接删除”的保护逻辑。

### 4.3 数据源管理

- 提供数据源列表、搜索、分页查看。
- 支持新增、编辑、删除数据源。
- 数据源可关联多个命名空间，实现多来源聚合拉数。
- 数据源可关联标签，便于分类管理与后续筛选使用。
- 数据源支持配置可用图表类型，限定该数据源可用于的展示形态。

### 4.4 命名空间管理

- 提供命名空间列表、搜索、分页查看。
- 支持新增、编辑、删除命名空间。
- 命名空间支持 TLS 开关，满足不同网络安全要求。
- 命名空间维护连接域名、账号、密码、逻辑命名空间等连接信息。

### 4.5 数据拉取与聚合

- 支持通过数据源触发实时取数。
- 单个数据源可面向多个命名空间并行获取结果并聚合返回。
- 请求自动附带当前用户与团队上下文，保证数据权限语义一致。
- 当某个命名空间拉取失败时，不阻断其他命名空间结果返回。

## 5. 关键业务规则

### 5.1 目录规则

- 目录层级上限为 3 层。
- 同一父目录下目录名称唯一。
- 目录可被启用/停用，目录树仅展示启用状态目录。

### 5.2 分析对象规则

- 仪表盘、拓扑图、架构图名称全局唯一。
- 分析对象可挂载到目录，也允许以空目录形态存在。
- 分析对象具备描述、视图配置与扩展配置能力。

### 5.3 数据源规则

- 数据源名称与 REST API 组合唯一。
- 数据源支持参数模板配置，参数按固定值、可编辑值等类型组织。
- 数据源可配置图表类型；当前可见类型包括 line、bar、pie、single。

### 5.4 命名空间规则

- 命名空间名称唯一。
- 命名空间密码以加密形式持久化存储。
- 连接地址协议由 TLS 开关决定（TLS 打开时使用安全连接协议）。

### 5.5 权限与可见性规则

- 页面按钮与操作能力受权限项控制，未授权用户不可执行对应动作。
- 分析资产与数据源遵循组织维度可见性策略。
- 查看权限与编辑权限独立控制，避免“可见即可改”。

### 5.6 列表与分页规则

- 管理页列表默认分页返回，统一返回总数与当前页数据。
- page_size 为空或为 0/-1 时，允许返回全量数据。
- 单次分页返回上限为 10000 条。

### 5.7 默认初始化规则

- 系统初始化时自动写入默认命名空间。
- 系统初始化时自动写入内置数据源标签（CMDB、告警、监控、日志、业务、其他）。
- 系统初始化时自动写入预置数据源模板，支持快速开箱使用。

## 6. 关键业务流程

### 6.1 配置与使用主流程

1. 管理员进入管理页，先配置命名空间连接信息。
2. 管理员创建或维护数据源，并关联命名空间与标签。
3. 用户在主视图创建目录结构与分析对象。
4. 用户在分析对象内选择数据源并配置展示方式。
5. 用户触发查询，系统按命名空间拉取并汇总返回。
6. 用户保存视图配置，形成可复用分析资产。

### 6.2 协作流程

1. 具备目录管理权限的用户建立统一目录规范。
2. 具备图表管理权限的用户在目录内持续沉淀分析对象。
3. 仅具查看权限的用户按团队可见范围消费分析结果。

## 7. 异常与降级策略

- 数据拉取异常时，单命名空间返回空数据，不中断整体结果。
- 管理页增删改失败时，前端给出明确失败提示，保持当前列表状态可恢复。
- 用户切换内容且存在未保存修改时，系统强提醒，避免误丢失编辑结果。
- 当无选中分析对象时，主内容区显示空态引导。

## 8. 非功能要求

- 安全性：连接密码加密保存；支持 TLS 连接策略。
- 可维护性：目录、视图、数据源、命名空间统一采用标准化 CRUD 管理。
- 一致性：主视图与管理页权限语义一致，操作反馈一致。
- 可扩展性：支持新增数据源模板与命名空间扩容。

## 9. 验收标准

### 9.1 功能验收

- 用户可在主视图完成目录、仪表盘、拓扑图、架构图的新增/编辑/删除。
- 用户可在管理页完成数据源、命名空间的新增/编辑/删除/查询。
- 数据源可关联多个命名空间并返回聚合结果。
- 未保存变更场景下的离开确认可正常触发。

### 9.2 规则验收

- 目录超过 3 层时不可创建。
- 同父目录重名目录不可创建。
- 同名且同 REST API 的数据源不可重复创建。
- 命名空间密码不可明文持久化。

### 9.3 权限验收

- 未授权用户不可见或不可用对应操作按钮。
- 仅查看权限用户不可执行新增、编辑、删除动作。
- 团队维度的数据可见性与授权一致。

## 10. 边界与约束

- 本 PRD 定义运营分析模块当前稳定能力，不包含技术迁移过程。
- 本 PRD 以代码可观测行为为准，若历史文档与当前实现冲突，以当前实现口径执行。
- 本 PRD 不扩展未落地能力（如模块内自动化任务编排、移动端同等功能）。
