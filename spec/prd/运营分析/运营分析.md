# 运营分析 · 运营分析

## 1. 背景与目标

运营分析提供可视化分析视图，让用户以仪表板、拓扑、架构图等形式自助编排和查看跨来源的运营数据。

## 2. 范围定义

- 视图类型：仪表板、拓扑、架构图。
- 视图组织：自定义目录组织视图。
- 过滤与命名空间：统一过滤条与命名空间选择。

## 3. 关键能力

- 支持三类视图，按自定义目录分层组织：
  - 仪表板：栅格化拖拽布局，支持折线、柱状、饼图、表格、单值、Top-N 等图表，每个组件绑定数据源查询。
  - 拓扑：图形可视化，节点可展示单值或图表并实时刷新，支持节点与边。
  - 架构图：等距图编辑，可拖入云厂商、K8s、资产等图标设计系统拓扑。
- 视图通过数据源（在管理中配置）取数，查询支持固定参数、用户筛选与运行时参数。
- 提供统一过滤条（下拉、时间范围、单值），过滤项绑定组件查询并作用于整个视图。
- 提供命名空间选择，决定视图取数的来源连接。
- 支持编辑模式（增删/重排组件、改查询）与自动保存、未保存离开提醒。

## 4. 关键规则

- 视图按组织分组，用户仅可见所属组织的视图。
- 内置视图对组织可见但常规用户不可删改。
- 组件不直接查询其他模块，统一经数据源取数。

## 5. 关键技术架构选择

- 模块定位为"可视化 + 配置层"，本身不做数据转换或聚合，取数全部下沉到用户定义的 REST 数据源。
- 取数经 NATS 按命名空间路由到下游服务；请求携带用户上下文用于下游鉴权。
- 视图配置支持 YAML 导入导出，并解析依赖（如仪表板带出其数据源）。

## 6. 验收标准

- 用户可创建并查看仪表板、拓扑、架构图三类视图。
- 组件可绑定数据源并按过滤条、命名空间取数。
- 视图按组织范围正确隔离，内置视图不可删改。
