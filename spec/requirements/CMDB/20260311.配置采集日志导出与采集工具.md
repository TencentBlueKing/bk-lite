# 配置采集日志导出与采集工具

日期：2026-03-11

## 背景

在社区场景中，用户进行配置采集时，常出现“采集失败但无法快速定位原因”的问题。当前流程下，用户需要自行排查协议连通性、凭据可用性与采集返回内容，排障效率低，且难以将原始证据提供给原厂进行优化与修复。

为提升社区助力调试效率，CMDB 需要提供“采集工具”能力：在采集任务失败时，用户可快速跳转到对应协议工具页，自动带入任务上下文并执行采集，查看原始返回日志并导出文本文件。

## 需求描述

CMDB 在配置采集的“SOID特征库”下新增“采集工具”页面，支持 SNMP、IPMI、SMI-S、CLI 四类工具的凭据填写、原始数据采集、日志查看与文本导出。

### 1）页面与入口

- 在“SOID特征库”下新增一级页面“采集工具”。
- “采集工具”页面提供四个协议类型入口：SNMP 工具、IPMI 工具、SMI-S 工具、CLI 工具。
- 当配置采集任务状态为失败时，在任务报错信息区域提供“前往采集工具”快捷入口。
- 快捷入口点击后，自动跳转至与失败任务协议类型一致的工具页。

Nano Banana 提示词：

- 设计一个 CMDB「SOID特征库」下的「采集工具」页面原型。页面包含四个协议标签页：SNMP、IPMI、SMI-S、CLI。每个标签页由三部分组成：左侧为凭据与连接参数表单，中间为执行控制区（执行按钮、重置按钮、执行状态），右侧为原始日志展示区与“导出文本”按钮。整体风格企业级、信息密度中等、优先突出排障效率。
- 在同一原型中补充“采集任务失败场景”联动：任务报错区域出现“前往采集工具”入口，点击后跳转到对应协议标签页，并自动带入任务上下文（任务标识、目标地址、协议类型、凭据引用）。自动带入的凭据字段以脱敏形式显示且可编辑。
- 表单交互要求：按协议动态显示必填项。SNMP 支持 v1/v2c/v3，v3 根据安全级别联动鉴权/加密字段；CLI 支持密码认证与密钥认证二选一；SMI-S 支持 HTTP/HTTPS 且 HTTPS 展示证书校验开关；IPMI 支持可选 cipher suite。
- 结果区交互要求：点击“执行”后展示原始返回日志（成功或失败都可展示已有返回），并支持导出 `.txt` 文本文件。失败时在结果区顶部展示可读错误摘要（例如认证失败、网络不可达、协议参数错误），避免仅显示通用失败提示。

### 2）失败任务快捷跳转与上下文带入

- 仅失败任务支持“前往采集工具”快捷跳转。
- 跳转后自动带入任务上下文字段，至少包含：任务标识、目标地址、协议类型、凭据引用。
- 若任务缺失必要上下文（例如协议类型为空），页面需提示“无法自动带入，请手动补全参数”，并允许用户继续手动执行。

### 3）凭据自动填充与编辑规则

- 跳转到采集工具页后，凭据字段默认自动填充。
- 凭据以脱敏形式展示，用户可在页面中编辑后再执行。
- 页面不展示完整明文凭据；保存与执行链路遵循现有凭据安全策略。
- 用户修改凭据后执行采集，以页面当前输入为准。

### 3.1）四类协议凭据录入项

#### 3.1.1 SNMP 凭据录入项

- 必填：目标地址、端口。
- 必填：SNMP 版本（v1/v2c/v3）。
- 当版本为 v1/v2c 时：必填 Community。
- 当版本为 v3 时：
	- 必填：用户名、安全级别（noAuthNoPriv / authNoPriv / authPriv）。
	- 安全级别包含鉴权时必填：鉴权协议、鉴权密码。
	- 安全级别包含加密时必填：加密协议、加密密码。

#### 3.1.2 IPMI 凭据录入项

- 必填：目标地址、端口。
- 必填：用户名、密码。
- 选填：加密套件（cipher suite）。

#### 3.1.3 SMI-S 凭据录入项

- 必填：目标地址、端口、命名空间（namespace）。
- 必填：用户名、密码。
- 必填：传输协议（HTTP/HTTPS）。
- 当传输协议为 HTTPS 时：支持证书校验开关配置。

#### 3.1.4 CLI 凭据录入项

- 必填：目标地址、端口。
- 必填：登录协议（SSH/Telnet）。
- 必填：用户名。
- 凭据认证方式支持二选一：
	- 密码认证：必填密码。
	- 密钥认证：必填私钥内容或私钥文件引用。
- 选填：Enable 密码（设备需要特权模式时使用）。

### 4）四类采集工具能力

#### 4.1 SNMP 工具

- 支持填写或自动带入 SNMP 凭据。
- 支持执行 OID Tree 原始数据采集。
- 支持展示 OID Tree 原始返回内容。
- 支持将原始返回内容导出为文本文件。

#### 4.2 IPMI 工具

- 支持填写或自动带入 IPMI 凭据。
- 支持执行 IPMI 原始数据采集。
- 支持展示 IPMI 原始返回内容。
- 支持将原始返回内容导出为文本文件。

#### 4.3 SMI-S 工具

- 支持填写或自动带入 SMI-S 凭据。
- 支持执行 SMI-S 原始数据采集。
- 支持展示 SMI-S 原始返回内容。
- 支持将原始返回内容导出为文本文件。

#### 4.4 CLI 工具

- 支持填写或自动带入 CLI 凭据。
- 支持执行 CLI 原始数据采集。
- 支持展示 CLI 原始返回内容。
- 支持将原始返回内容导出为文本文件。

### 5）日志展示与文本导出

- 采集执行完成后，页面展示本次执行的原始日志内容。
- 导出文件格式为文本文件（`.txt`），内容为本次执行的原始采集返回。
- 每次执行结果独立展示与导出，不覆盖历史已下载文件。
- 采集失败时仍可导出失败日志文本（若存在返回内容）。

### 6）错误提示与可用性

- 执行失败时，页面需给出可读错误提示，至少包含失败阶段与错误摘要。
- 当网络、认证或协议参数异常时，提示信息应可用于用户二次排查，不得仅返回通用“执行失败”。
- 从失败任务跳入后，用户仅需点击“执行”即可完成一次默认采集尝试。

## 适用范围

- CMDB：SOID特征库下“采集工具”页面新增。
- CMDB：配置采集任务失败场景下的快捷跳转。
- CMDB：SNMP/IPMI/SMI-S/CLI 四类工具原始采集与文本导出。

## 验收标准

- 在 SOID特征库下可见“采集工具”页面，并可访问 SNMP、IPMI、SMI-S、CLI 四类工具页。
- 配置采集任务失败时，任务报错区域存在“前往采集工具”入口。
- 点击失败入口后可跳转至对应协议工具页，并自动带入任务上下文与凭据信息。
- 自动填充的凭据以脱敏形式展示，用户可编辑并发起执行。
- 四类工具均有明确凭据录入项校验，且按协议版本/认证方式动态要求必填字段。
- SNMP 工具可获取 OID Tree 原始数据并展示，且支持导出文本文件。
- IPMI、SMI-S、CLI 工具可获取各自原始数据并展示，且支持导出文本文件。
- 执行成功或失败后，若有返回内容，均可导出本次原始日志文本。
- 当自动带入字段缺失时，页面给出明确提示并允许用户手动补全后执行。
- 错误提示包含可读的失败摘要，不仅为通用错误文案。
- 从失败任务跳入后，在默认参数完整场景下，用户无需额外配置即可一键执行。
