# 枚举字段支持公共引用

日期：2026-03-06

## 背景

品牌、厂商等枚举数据在多个模型中重复维护，口径不一致、维护成本高、变更同步困难。

CMDB 需要提供“公共选项库”能力，作为跨模型复用的枚举数据源，统一管理并在模型字段配置中可引用，降低重复配置与数据漂移风险。

## 需求描述

枚举类型字段新增“公共引用”能力：模型字段可选择“自定义选项”或“公共选项库”作为校验数据源。仅作用于 `enum` 类型字段。

### 1）公共选项库入口与范围

- 在模型管理列表页面提供“管理公共选项”入口。
- 点击入口后打开弹窗，支持在当前组织上下文下维护公共选项库。
- 公共选项库支持“包含子组织”范围控制：
  - 未勾选“包含子组织”时，仅当前组织可见可用。
  - 勾选“包含子组织”时，当前组织与子组织可见可用。
- 列表展示当前组织可见范围内的公共选项库（包含可见的子组织库）。

### 2）公共选项库维护规则

- 支持新增、修改、删除公共选项库。
- 新增公共选项库时默认组织为当前组织。
- 选项内容交互与枚举 option 一致：按行维护 `Key` / `Display`。
- 保存校验规则：
  - 选项库名称不能为空。
  - 选项内容不能为空。
  - 每行 `Key` 与 `Display` 均不能为空。

### 3）模型字段配置规则（枚举）

- 在枚举字段配置中新增“校验规则”单选项：`自定义`、`公共选项库`。
- 默认值为 `自定义`。
- 历史已存在字段初始化为 `自定义`。
- 字段创建完成后，不允许调整“校验规则”类型（`自定义`、`公共选项库`）。
- 选择 `自定义` 时，保持现有枚举 option 维护交互。
- 选择 `公共选项库` 时：
  - 展示公共选项库下拉框，仅可选择当前组织条件下可见可用的公共选项库。
  - 选中后展示该库的 option 内容，仅展示不允许在当前页面修改。
  - 如需修改 option，需前往“管理公共选项”入口进行维护。

### 4）实例相关页面规则

- 当枚举字段使用公共选项库时，实例新增、编辑、复制、详情编辑等交互方式与现有枚举一致。
- 实例列表展示、搜索筛选、导入导出、批量更新等与枚举类型保持一致。
- 公共选项库变更后，历史实例值保留；新建与编辑写入按当前已绑定公共选项库校验。

## 适用范围

- 模型管理：公共选项库管理入口与弹窗
- 模型字段配置：枚举字段校验规则与数据源选择
- 实例维护与实例数据相关页面：按枚举现有规则执行

## 非目标（本期不包含）

- `model_config` 配置支持公共选项库
- 非枚举字段（如 `tag`、`table` 等）接入公共选项库
- 公共选项库的审批流、版本管理、跨租户共享能力

## 验收标准

- 模型管理页可进入“管理公共选项”弹窗，并可查看当前组织可见范围内的公共选项库。
- 公共选项库支持新增、修改、删除，新增时默认归属当前组织。
- 公共选项库选项维护交互与枚举 option 一致；名称、选项库内容、行内 `Key`/`Display` 均不能为空。
- 枚举字段配置中存在“校验规则=自定义/公共选项库”单选，默认 `自定义`。
- 历史字段进入编辑时，校验规则默认显示为 `自定义`。
- 枚举字段创建完成后，编辑场景不允许切换“校验规则”类型（`自定义`、`公共选项库`）。
- 选择“公共选项库”后，仅可选择当前组织条件下可见可用的公共选项库，并可只读查看其 option 内容。
- 在字段配置页面无法直接修改公共选项库 option，需通过“管理公共选项”入口修改。
- 实例相关页面在公共引用场景下交互与枚举一致；历史值保留，新建/编辑按当前绑定库校验。
