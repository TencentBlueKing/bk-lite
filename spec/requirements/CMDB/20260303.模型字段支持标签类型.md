# 模型字段支持标签类型

日期：2026-03-03

## 背景

当前实例描述能力主要依赖固定字段，用户侧难以对数据进行低成本、灵活的语义标注（如环境、地域、业务属性）。

为下放数据描述权并提升实例检索与治理效率，CMDB 模型字段需支持内置“标签类型”字段，打通模型配置、实例录入/修改、列表展示、搜索、导入导出与批量更新链路。

## 需求描述

CMDB 模型字段新增内置标签能力（单模型单标签字段）。标签数据以 `key`、`value` 结构维护，展示与回填格式为 `key:value`。

### 1）模型字段定义与模型管理交互

- 每个模型最多存在 1 个标签类型字段。
- 标签类型字段的属性 ID 固定为 `tag`。
- 在模型管理的字段设置中，当字段类型选择为 `tag` 时，属性 ID 自动填充为 `tag`，且输入框禁用编辑。
- 删除标签字段需二次确认。

### 2）标签字段编辑规则

- 字段 ID 固定为 `tag`，且不允许修改。
- 字段名称、属性分组允许修改。
- 内置约束固定且不允许修改：
  - 可编辑：是
  - 唯一：否
  - 必填：否
- 模式仅约束实例维护时的选择行为，不约束模型管理中的标签选项维护行为。
- 自由模式与严谨模式下，均允许在模型管理中编辑标签选项内容。

### 3）标签选项配置规则

- 标签选项采用与枚举类型一致的行编辑交互：每行两个输入框，分别为 `key`、`value`。
- 标签项支持同 key 多 value 并存（例如 `env:test`、`env:prod`、`env:uat`）。
- 标签 key 不允许为空。
- 标签 value 不允许包含空格、冒号、换行符。
- 前端支持快速配置：点击“批量修改”后展示 `textarea`，用户可输入多行 `key:value`，每行代表一个标签项。
- 后端建议以数组对象形式存储标签类型数据：`[{"key":"xxx","value":"zzzzz"}]`。

### 4）实例维护交互规则

- 实例维护相关页面（新增、修改、详情页修改、复制）统一使用级联选择交互。
- 级联选择规则：
  - 第一层展示标签 key 列表。
  - 第二层展示所选 key 对应的 value 列表。
  - 当存在多个相同 key 时，第一层仅展示一个 key，第二层展示该 key 的多个 value。
- 选择完成后，页面回填展示格式为 `key:value`。
- 严谨模式下，仅允许从标签候选数据源中选择。
- 自由模式下，允许输入自定义 `key` 与 `value`。
- 自由模式下，若用户输入的 `key:value` 不在当前模型字段标签候选范围内，系统自动将该项新增到模型字段标签候选配置。
- 录入/修改阶段实时执行标签合法性校验：
  - 格式校验：必须为 `key:value`
  - key 校验：不可为空
  - value 字符校验：不可包含空格、冒号、换行符
  - 数量校验：最多 20 个标签对

### 5）实例列表展示与批量操作规则

- 实例列表支持展示标签列，展示格式为逗号分隔的 `key:value` 序列。
- 标签内容超长时自动截断，鼠标移入显示完整 tooltip。

### 6）实例搜索规则（标签专用）

- 当搜索字段选择“标签”时，展示标签专用下拉筛选组件。
- 组件形态为二层树选择：
  - 第一层：标签 key 列表
  - 第二层：对应 key 的 value 列表
- 支持多选，已选内容在输入区域回显。
- 同一查询会话内支持跨页多选；切换筛选条件后清空已选集合。
- 标签搜索仅支持精确匹配。
- 标签搜索时 `Accurate` 固定为开启状态，不允许切换为模糊匹配。

### 7）导入/导出与批量更新规则

- Excel 中标签字段格式为 `key:value`。
- 多个标签对分隔符兼容：英文逗号（`,`）、中文逗号（`，`）、换行。
- 导入、导出、批量更新均复用同一标签校验规则：
  - key 不允许为空
  - value 不允许空格、冒号、换行符
  - 单实例最多 20 个标签对
  - 非法数据需返回可定位的错误信息（至少包含行号与字段名）

## 适用范围

- 模型字段定义与编辑
- 实例新增/修改
- 实例列表展示
- 实例搜索
- 实例导入/导出
- 实例批量更新

## 验收标准

- 标签类型字段属性 ID 固定为 `tag`，单模型最多仅允许存在 1 个标签类型字段。
- 在模型管理中选择字段类型为 `tag` 时，属性 ID 自动填充为 `tag` 且不可编辑。
- 标签字段编辑时，字段 ID 不可变更；名称与分组可变更；可编辑/唯一/必填内置约束不可改。
- 自由模式与严谨模式下，模型管理均可编辑标签候选内容。
- 标签候选采用一行 `key` + `value` 双输入交互；支持“批量修改”并按换行解析多行 `key:value`。
- 后端支持按 `[{"key":"xxx","value":"zzzzz"}]` 结构存储标签类型数据。
- 实例新增、修改、详情页修改、复制场景下使用级联选择：第一层 key、第二层 value；同 key 仅展示一次，value 可多选；回填格式为 `key:value`。
- 严谨模式下仅可从数据源选择；自由模式下可自定义输入 `key`、`value`。
- 自由模式下，用户输入超出模型字段既有候选范围的 `key:value` 时，系统自动新增到模型字段标签候选。
- 标签 key 为空时保存失败并提示；value 包含空格、冒号、换行符时保存失败并提示；单实例超过 20 个标签对时保存失败并提示。
