# 告警中心-新增"告警升级策略"

## 1. 背景与问题
现有告警分派规则只解决"通知给谁、通知多频"两个问题，没有回答"通知了但没人响应该怎么办"。告警分派后，处理人因休假、会议、通知未到达等原因长时间不响应时，告警会一直停留在"待处理"状态直到故障扩大。

业界对标产品（PagerDuty、Opsgenie、Grafana OnCall 等）均提供独立的升级策略实体来兜底这一场景。BK-Lite 缺位，需要补齐。

本期仅做**超时升级**：告警在规定时长内未被认领时，自动切换到下一层处理人。其他升级形态（严重度升级、频次升级、手动升级）不在本期。

## 2. 业务需求
- 在「配置」下新增菜单"升级策略"，与`告警分派`、`屏蔽策略`并列，作为可复用的独立实体。
- 升级策略由有序的多个层级组成，每个层级指定处理人（个人或角色）与等待时长。
- 告警分派规则可引用一条升级策略；`全局配置`可设置默认升级策略，作为兜底。
- 告警在某一层的等待时长内未被认领，自动进入下一层并通知该层处理人；走完最后一层仍未认领则停止升级。
- 告警被认领、已解决、已关闭后，立即终止后续升级。

## 3. 业务逻辑

### 3.1 升级策略配置项
| 分类 | 字段 | 必填 | 说明 |
|---|---|---|---|
| 基础 | 策略名称、生效组织 | ✅ | 与现有相关性规则、分派规则一致 |
| 基础 | 状态（启用/停用） | — | 沿用现有规则惯例 |
| 路径 | 升级层级列表 | ✅ | 有序，至少 1 层；每层包含：处理人 / 等待时长 / 通知渠道（可选，未指定则沿用触发本次升级的分派规则的渠道） |

### 3.2 引用模型
- 告警分派规则新增字段"升级策略"（可选下拉）。
- `全局配置`新增字段"默认升级策略"。
- 一条告警的有效升级策略按以下优先级解析：
  1. 命中的分派规则上指定的升级策略
  2. 全局配置中的默认升级策略
  3. 都未配置 → 不升级

### 3.3 触发与终止
- **触发口径**：以告警是否被**认领**为信号。告警停留在"待处理"超过当前层等待时长 → 进入下一层。
- **终止口径**：告警被认领（进入"处理中"）、已解决、已关闭 → 立即终止后续升级。
- **走完仍未响应**：到达最后一层后停止升级，告警保持"待处理"，由用户后续处置。
- **改派/分派变更**：发生改派时，升级计时**重置**到第一层开始；新分派人若仍不响应，再按完整路径走。

### 3.4 与现有分派提醒的关系
- 分派规则原有的"通知频率（最大次数 + 间隔）"语义是**同一人多次催**，解决"叫不醒"。
- 升级策略语义是**没响应就换人**，解决"叫错人/此人无法响应"。
- 两者并存、互不替代。同一条告警在等待升级的同时，仍可被原分派人按提醒频率重复通知。

### 3.5 与屏蔽策略的关系
- 升级触发的通知同样接受现有屏蔽策略评估；命中屏蔽则抑制通知，但**不阻断升级**（仍按时间推进到下一层）。

### 3.6 与即时告警的关系
- 即时告警同样可被升级策略覆盖（前提是命中的分派规则或全局配置指定了升级策略）。两类规则正交，互不干扰。

### 3.7 不挂事故
- 本期升级策略只服务于"告警"实体，不挂"事故"。未来若需事故升级，可复用同一实体，本期不展开。

## 4. 关键技术架构选择
- **升级策略是独立实体**，不嵌入告警分派规则；通过被引用的方式复用，呼应 PagerDuty / Opsgenie 等业界标杆的实体边界。
- **挂载点采用"分派规则引用 + 全局配置兜底"双层模型**：分派规则可针对性指定策略以体现优先级差异，全局配置提供入门默认值，降低 PLG 入门成本。
- **驱动方式为时间驱动**：以告警停留在"待处理"的时长作为升级判据，绑定告警状态机（认领/已解决/已关闭即终止）。实时性要求为**分钟级**，不需要秒级，与现有分派提醒轮询机制同一量级。

## 5. 验收口径
- 可在「配置」下新建、编辑、删除、启停升级策略；策略可被多条分派规则引用。
- 分派规则可选择引用任一升级策略；全局配置可设置默认升级策略。
- 告警在某一层等待时长内未被认领 → 自动进入下一层并通知该层处理人；超过最后一层仍未认领则停止升级。
- 告警被认领、已解决、已关闭后，不再触发后续层级的升级通知。
- 改派后升级计时从第一层重置开始。
- 升级通知与原分派提醒并存，互不替代；命中屏蔽策略时该次通知被抑制，但升级层级仍按时间推进。
- 未命中任何分派规则、且全局未配置默认策略时，告警不升级。
- 升级策略按生效组织正确隔离。

## 6. 约束与边界
**In Scope**：升级策略独立菜单、分派规则引用字段、全局配置默认值、超时升级触发与终止逻辑、与提醒/屏蔽机制的并存关系。

**Out of Scope**：严重度升级、频次升级、手动升级动作、事故级别的升级、跨告警的批量升级、升级过程中变更升级策略的热替换。
