# 运营分析 — 网络拓扑大屏画布 设计规格

> 状态: 已通过 brainstorming 流程与用户确认,等待最终评审
> 日期: 2026-07-07
> 适用模块: `bk-lite/server/apps/operation_analysis/` + `bk-lite/web/src/app/ops-analysis/`
> 关联模块: `bk-lite/server/apps/monitor/`(指标源,本设计仅消费其 API)

## 1. 背景与目标

### 1.1 背景

bk-lite 监控中心已具备对网络设备的指标采集能力(SNMP,Stargazer 采集 → VictoriaMetrics 存储 → `monitor` app 暴露 REST 查询)。运营分析模块已存在 5 类画布(`dashboard / topology / screen / architecture / report`),其中 `topology` 画布基于 `@antv/x6` 实现了通用节点拖入 + 连线 + 数据源绑定能力。

运维侧痛点:网络故障定位时,运维需在监控中心逐个查设备 / 接口状态,无法在"以链路为视角"的大屏上看到实时连接质量。

### 1.2 目标

在 `operation_analysis` 模块下**新增第 6 类画布「网络拓扑大屏」**,让运维能:

1. 拖入已监控的网络设备作为节点,展示设备名 + 多个指标 + 接口摘要 + sparkline
2. 在节点间用"多个接口组合"创建连线,配置每接口的展示指标
3. 连线状态按规则自动着色(活跃绿 / 中断红),判定标准默认「任一接口 `ifOperStatus==2(Down)` 即中断」
4. 按可配置周期(默认 60s)自动刷新,大屏可挂在值班屏

### 1.3 范围

**In Scope**:

- 1 个新画布类型(`network_topology`):NetworkTopology 模型、view_sets 布局、permission_key、URL 路由
- 1 个旁路业务表 + 1 个子表:NetworkLinkConfig / NetworkLinkConfigInterface,存连线业务配置
- 1 个后端服务:NetworkLinkStatusService 聚合 PromQL,按判定规则返回状态
- 1 个新前端画布入口:(pages)/view/networkTopology/,复用 X6 hooks、节点库侧栏、widget 数据源机制
- 节点走现有 DataSourceAPIModel;连线状态走专用聚合接口(数据拉取混合模式)
- 节点拖入 / 节点配置 / 连线创建 / 连线配置 / 实时刷新 / 故障定位 / 删除 / 导入导出 / 权限 / 审计

**Out of Scope**: 见 §7。

## 2. 架构总览

### 2.1 模块定位

在 `operation_analysis` app 内**新增第 6 类画布**「网络拓扑大屏」,与 `dashboard / topology / screen / architecture / report` 平级。

### 2.2 命名与 Key

| 项 | 值 |
|---|---|
| 中文名 | 网络拓扑大屏 |
| 英文 canvas key | `network_topology` |
| 后端 model 名 | `NetworkTopology` |
| URL 前缀 | `/api/operation_analysis/api/network_topology/` |
| 前端路由 | `/ops-analysis/view/networkTopology` |
| permission_key | `directory.network_topology`(view / AddCatalogue / EditCatalogue / DeleteChart) |
| 导航顺序 | 在「拓扑图」之后、「架构图」之前 |

### 2.3 模块边界

- 后端 `apps/operation_analysis/`:新增 model 文件、旁路 model、service、view、URL 增量;**不动**现有 5 类画布
- 前端 `web/src/app/ops-analysis/`:新增 `(pages)/view/networkTopology/` 目录;**复用** X6 hooks、`useGraphOperations`、`useDataSourceApi`、`useTopologyRefresh`

### 2.4 数据源边界

- 设备数据:**只**从 `MonitorInstance`(type=Network Device)取,不调 CMDB
- 指标数据:**只**调 `monitor` app 的 `metrics_instance/query*` API
- 连线状态数据:由本 app 内 `NetworkLinkStatusService` 用 PromQL 聚合 VictoriaMetrics

### 2.5 数据拉取模式(混合)

- **节点上的指标** → DataSourceAPIModel + 现有 widget 数据流(`useDataSourceApi().getSourceDataByApiId()`)
- **连线状态 + 连线接口指标** → 专用聚合接口(后端一次性按判定规则聚合,前端订阅聚合结果)

## 4. 数据模型与业务规则

### 4.1 概念模型

#### 4.1.1 NetworkTopology(画布)

代表"一张网络拓扑大屏"。每张画布独立,与其他画布互不影响。

| 字段 | 含义 | 备注 |
|---|---|---|
| `id` | 画布主键 | 系统生成 |
| `name` | 画布名 | 用户可编辑,默认 "未命名网络拓扑大屏" |
| `directory_id` | 所属目录 | 外键,沿用现有 `Directory` 树 |
| `view_sets` | 画布布局 JSON | 含节点(x/y/类型)、边(source/target 引用)、画布缩放 |
| `refresh_interval_seconds` | 刷新周期 | 默认 60,可选 10/30/60/300/600 |
| `is_build_in` | 是否内置示例 | 默认 false,预留一个内置 demo |
| `created_at` / `updated_at` / `created_by` | 时间与创建人 | 沿用现有字段约定 |
| `organization` | 组织 ID | 用于多组织隔离 |

#### 4.1.2 NetworkLinkConfig(连线业务配置)

代表"一条连线的业务语义",独立于画布布局存储,以便后续按连线查询历史 / 状态。

| 字段 | 含义 | 备注 |
|---|---|---|
| `id` | 主键 | |
| `topology_id` | 所属画布 | 外键 → NetworkTopology |
| `link_id` | 画布内稳定 ID | 与 view_sets 边数据中引用的 ID 对齐,**画布内唯一** |
| `name` | 连线名(可空) | 默认 "Link <source> → <target>" |
| `source_node_id` | 源节点画布 ID | 与 view_sets 节点 ID 对齐 |
| `target_node_id` | 目标节点画布 ID | |
| `judgment_rule` | 状态判定规则 | 默认 `oper_status_down_only`,可选 `strict_snmp` |
| `created_at` / `updated_at` | | |

**约束**:`(topology_id, link_id)` 唯一;`source_node_id != target_node_id`;节点删除时级联删除。

#### 4.1.3 NetworkLinkConfigInterface(连线包含的接口)

一条连线由若干接口组成,两端各 0~N 个。

| 字段 | 含义 | 备注 |
|---|---|---|
| `id` | 主键 | |
| `link_config_id` | 所属连线 | 外键 → NetworkLinkConfig |
| `monitor_instance_id` | 该接口所属 MonitorInstance | |
| `interface_ref` | 接口标识 | 在该 instance 的 `network_interfaces` 维度里的唯一标识(`ifDescr` 或 `ifIndex`) |
| `endpoint` | 端点方向 | `source` / `target` |
| `metrics_selected` | 该接口上展示的指标 | JSON 数组,每项含 `metric_id` 和 `display_name` |
| `sort_order` | 显示排序 | 同一端点内升序 |

**约束**:同一 `(link_config_id, endpoint, monitor_instance_id, interface_ref)` 唯一;连线至少需要 1 个接口,否则视为配置不合法。

#### 4.1.4 节点侧(画布 view_sets 内)

节点在画布上的位置 / 设备绑定 / 指标配置全部存于 `NetworkTopology.view_sets` JSON,不单独建表。

`view_sets` 节点对象(高层结构):

```text
{
  "id": "node-xxx",
  "type": "network_device",
  "monitor_instance_id": "instance-yyy",
  "x": 200, "y": 120,
  "device_metrics": [ {metric_id, display_name, kind: "value|sparkline"} ],
  "show_interface_summary": true
}
```

### 4.2 业务规则

#### 4.2.1 状态判定规则

| 规则代码 | 语义 | 默认 |
|---|---|---|
| `oper_status_down_only` | 任一接口 `ifOperStatus == 2(Down)` → 中断;否则 → 活跃 | ✅ |
| `strict_snmp` | 任一接口 `ifOperStatus != 1(Up)` → 中断(Testing/Unknown 等都算中断) | 可选 |

未来可扩展 `custom_expression`(用户写 PromQL),**当前不做**。

#### 4.2.2 数据来源约束

- 节点设备 **必须** 来自 `MonitorInstance` 且 `instance.type = Network Device`;未监控的资产不可拖入
- 接口 **必须** 存在于该 instance 的 `network_interfaces` 指标维度集合内;否则视为配置错误,UI 提示用户
- 接口当前指标值由 `monitor` app 的 `metrics_instance/query` API 透出,本大屏不重新拉取原始数据

#### 4.2.3 连线生命周期

- 创建:用户连线时同步创建 `NetworkLinkConfig` + 至少 1 条 `NetworkLinkConfigInterface`
- 删除节点:级联删除其作为 source/target 的全部 `NetworkLinkConfig` 及其子表
- 删除画布:级联删除全部 `NetworkLinkConfig` 及其子表
- 编辑连线:只改 `NetworkLinkConfig` 和 `NetworkLinkConfigInterface`;画布 JSON 里只更新 `link_id` 引用

#### 4.2.4 数据一致性

- `NetworkLinkConfig.source_node_id` / `target_node_id` 必须能在对应画布的 `view_sets` 找到对应节点;否则视为脏数据,UI 高亮告警
- 接口失效检测时机:① 用户编辑连线时,若 instance 的 `network_interfaces` 维度集合不含该 `interface_ref` 则提示;② 大屏刷新时,聚合接口对每个 `interface_ref` 做存在性校验,失效的接口行变灰、状态列显示「无效」,但不影响其他接口聚合判定
- 接口变更不会自动迁移(需用户手动重新配置),避免误删

#### 4.2.5 权限与审计

- 沿用现有 `directory.network_topology` 权限 key
- 编辑操作(添加/删除节点、添加/删除/修改连线、修改指标选择)写入 audit_log,沿用现有 `audit_log.py` 模块

## 5. 交互流程

按用户视角的关键流程。每步描述「用户动作 → 系统反馈 → 边界情况」。

### 5.1 创建画布
- 用户在「网络拓扑大屏」目录页点击「新建」
- 跳转到大屏编辑器,生成空画布,左侧节点库加载完成
- **边界**:节点库为空时提示「暂无已监控的网络设备,请先在监控中心完成接入」

### 5.2 添加节点
- 用户在节点库侧栏搜索(设备名 / IP),从结果列表拖到画布任意位置
- 节点出现在拖入位置,显示设备名 + IP + 状态(unknown,等待首次刷新);同步写入 `view_sets`,数据库立即持久化
- **边界**:同设备重复拖入提示「该设备已存在」;已下线设备(`MonitorInstance.is_active=false`)可拖入但加灰色角标

### 5.3 配置节点展示
- 用户双击节点,弹出节点配置面板:基本信息 / 设备级指标 / sparkline / 显示开关
- 实时预览节点卡片;点确认后写入 `view_sets`
- **边界**:至少勾选 1 个指标,否则保存按钮 disabled;无指标时提示「该设备暂无可用指标」

### 5.4 创建连线
- 用户从源节点边缘的「可用接口」连接点拖到目标节点的「可用接口」连接点
- 画布显示连线草稿(虚线灰色);弹出连线配置面板(跳转 5.5)
- **边界**:同端点可拖入多个接口(连多条线);拖到无效位置显示禁止图标,不创建草稿

### 5.5 配置连线
- 用户在连线配置面板:选择源/目标端接口(各可多选)、每接口指标(创建连线时默认勾选监控中心接口常用指标:入口流量 / 出口流量 / 错包率,可改)、判定规则(默认 `oper_status_down_only`)、连线名(可选)
- 至少 1 个接口才能保存;保存后创建 `NetworkLinkConfig` + `NetworkLinkConfigInterface`;连线变实线带状态色(默认灰色)
- **边界**:无效接口(不在 `network_interfaces` 维度集合内)UI 标红提示「该接口在监控中心不存在」;规则变更不需重新拉取数据,下次刷新生效

### 5.6 实时观察(刷新)
- 用户被动等待(按配置的刷新周期)或主动点刷新按钮
- 节点卡片上的指标 / 接口摘要 / sparkline 全部更新;连线按判定规则重新着色(绿/红);刷新时间戳显示在画布右下角
- **边界**:节点数据拉取失败显示「数据异常」徽标,不影响其他节点;连线状态查询失败连线变灰,鼠标悬停提示「状态查询失败,请稍后重试」

### 5.7 故障定位
- 用户点击红色(中断)连线
- 弹出详情面板:状态判定结果 + 判定规则 + 触发中断的接口清单(oper_status / 当前指标值 / 最后变更时间)+ 全部接口列表
- **边界**:最后变更时间为 null(从未变更)显示「—」

### 5.8 删除节点 / 连线
- 删除节点:右键节点 → 删除,弹确认框「删除节点将同时删除其作为端点的 N 条连线,是否继续?」;确认后级联删除 `NetworkLinkConfig` 及其子表
- 删除连线:右键连线 → 删除,直接删除(无确认框)
- **边界**:删除不可撤销(沿用现有画布行为);画布可变为空画布

### 5.9 编辑刷新周期
- 用户在画布右上角「画布设置」选择刷新周期(10/30/60/300/600 秒)
- 即时生效,下次刷新按新周期;写入 `NetworkTopology.refresh_interval_seconds`

### 5.10 导入导出
- 用户沿用现有画布的导入导出菜单
- 导出 JSON 含 `view_sets` + 全部 `NetworkLinkConfig` + 接口子表
- 导入时校验节点引用的 monitor_instance 是否在当前系统存在(缺失时降级为「未知节点」并提示);用户对缺失节点可执行的操作:① 重新指定为当前环境的另一 MonitorInstance(需该 instance.type=Network Device);② 删除该节点(同时级联清理关联的 `NetworkLinkConfig`)

### 5.11 权限不足
- 无权限用户访问大屏:列表页看不到该画布;直接访问 URL 时跳转 403
- 仅 `view-View` 权限:编辑模式不可进入,工具栏新增/删除按钮 disabled

## 6. 验收标准

每条采用 **Given/When/Then** 格式,可直接转化为测试用例。

### 6.1 画布基础

**AC-01 创建画布**
- Given 用户有 `view-AddCatalogue` 权限
- When 在「网络拓扑大屏」目录页点击「新建」
- Then 跳转到大屏编辑器,画布为空,左侧节点库加载完成

**AC-02 画布命名**
- Given 大屏已创建
- When 用户修改名称并保存
- Then 名称在列表页、目录树、画布标题全部同步显示

**AC-03 节点库为空提示**
- Given 监控中心无已接入的网络设备
- When 用户新建画布
- Then 节点库侧栏显示空状态文案「暂无已监控的网络设备,请先在监控中心完成接入」

### 6.2 节点

**AC-04 拖入节点**
- Given 节点库含设备 X
- When 用户将 X 拖入画布位置 (200, 120)
- Then 节点出现在 (200, 120),显示设备名 + IP,数据库立即有 `NetworkTopology.view_sets` 含节点

**AC-05 重复设备阻止**
- Given 画布已有设备 X 节点
- When 用户再次拖入 X
- Then 弹出提示「该设备已存在」,画布不新增节点

**AC-06 节点配置保存**
- Given 用户双击节点
- When 勾选 3 个指标 + 1 个 sparkline,点确认
- Then `view_sets` 节点对象含 `device_metrics` 数组(3 项) + sparkline 字段;关闭面板后画布上节点卡片显示新指标

**AC-07 节点配置无指标拒绝**
- Given 节点配置面板打开
- When 用户未勾选任何指标就点确认
- Then 保存按钮 disabled,验证提示「至少选择 1 个指标」

**AC-08 节点指标为空提示**
- Given 设备无任何可展示的 metric
- When 用户双击该节点
- Then 配置面板显示「该设备暂无可用指标」,确认按钮 disabled

### 6.3 连线

**AC-09 创建连线**
- Given 画布有 2 个节点 A、B
- When 用户从 A 的「可用接口」拖到 B 的「可用接口」
- Then 画布显示连线草稿(虚线灰色),弹出连线配置面板

**AC-10 连线至少 1 接口**
- Given 连线配置面板打开
- When 用户未勾选任何接口就保存
- Then 保存按钮 disabled,验证提示「连线至少需要 1 个接口」

**AC-11 连线接口无效提示**
- Given 用户在连线配置面板勾选了「GigabitEthernet99」接口
- When 该接口不在 instance 的 `network_interfaces` 维度集合内
- Then 该接口项标红,显示「该接口在监控中心不存在」

**AC-12 连线保存成功**
- Given 用户完成连线配置(2 个接口,各 3 个指标)
- When 点确认
- Then 数据库新增 1 条 `NetworkLinkConfig` + 2 条 `NetworkLinkConfigInterface`;画布上连线变实线

### 6.4 状态判定

**AC-13 默认规则 Down 中断**
- Given 连线含 2 个接口,判定规则 `oper_status_down_only`
- When 其中 1 个接口的 `ifOperStatus=2(Down)`,另 1 个 `ifOperStatus=1(Up)`
- Then 连线显示为中断(红色)

**AC-14 默认规则全 up 活跃**
- Given 同 AC-13 配置
- When 2 个接口 `ifOperStatus` 都是 1(Up)
- Then 连线显示为活跃(绿色)

**AC-15 默认规则 Testing 仍活跃**
- Given 连线含 2 个接口,默认规则
- When 1 个接口 `ifOperStatus=3(Testing)`,另 1 个 `ifOperStatus=1(Up)`
- Then 连线显示为活跃(绿色,Testing 不计入 Down)

**AC-16 严格规则 Testing 中断**
- Given 连线含 2 个接口,判定规则 `strict_snmp`
- When 同 AC-15 场景
- Then 连线显示为中断(红色,Testing 算非 Up)

**AC-17 规则变更生效**
- Given 连线判定规则当前为默认
- When 用户改为 `strict_snmp` 并保存
- Then 下次刷新按新规则重新计算状态

### 6.5 实时刷新

**AC-18 默认刷新周期**
- Given 新建画布
- When 用户未修改刷新周期
- Then 默认 60 秒刷新一次,右下角显示「下次刷新: 60s」

**AC-19 刷新周期可选值**
- Given 画布设置打开
- When 用户选择 30 秒
- Then 立即生效,数据库 `refresh_interval_seconds=30`

**AC-20 手动刷新**
- Given 画布运行中
- When 用户点击刷新按钮
- Then 立即触发一次全量数据拉取,刷新时间戳更新为当前时间

**AC-21 节点拉取失败降级**
- Given 节点 A 的指标查询返回 5xx
- When 画布刷新
- Then A 节点卡片显示「数据异常」徽标,其他节点正常更新

**AC-22 连线状态查询失败**
- Given 连线状态聚合接口超时
- When 画布刷新
- Then 连线变灰,鼠标悬停提示「状态查询失败,请稍后重试」

### 6.6 故障定位

**AC-23 红色连线详情**
- Given 红色(中断)连线 L
- When 用户点击 L
- Then 弹出详情面板:展示判定规则、触发中断的接口清单(含 oper_status / 当前指标值 / 最后变更时间)

**AC-24 无中断历史**
- Given 连线 L 当前活跃
- When 用户悬停 L
- Then 详情面板只显示当前状态 + 全部接口当前值

**AC-25 最后变更时间为空**
- Given 接口从未变更过 oper_status
- When 详情面板展示该接口最后变更时间
- Then 显示「—」

### 6.7 删除与级联

**AC-26 删除节点确认**
- Given 节点 A 作为 2 条连线的端点
- When 用户右键 A → 删除
- Then 弹出确认「删除节点将同时删除其作为端点的 2 条连线,是否继续?」

**AC-27 级联删除**
- Given AC-26 确认对话框
- When 用户点确认
- Then A 节点从画布消失,2 条连线从画布消失,数据库删除 2 条 `NetworkLinkConfig` 及全部 `NetworkLinkConfigInterface`

**AC-28 删除连线无确认**
- Given 画布有连线 L
- When 用户右键 L → 删除
- Then 直接删除(无确认框)

### 6.8 导入导出

**AC-29 导出**
- Given 画布有 2 个节点 + 3 条连线
- When 用户点导出
- Then 下载 JSON 含 `view_sets` + 3 条 `NetworkLinkConfig` + 全部 `NetworkLinkConfigInterface`

**AC-30 跨环境导入降级**
- Given 导出的 JSON 含 `monitor_instance_id=instance-001`,目标环境无该实例
- When 用户导入该 JSON
- Then 节点降级为「未知节点」并提示用户「以下设备在当前环境不存在,请手动调整」

### 6.9 权限

**AC-31 列表可见性**
- Given 用户对目录 D1 有 view 权限、对 D2 无权限
- When 访问网络拓扑大屏列表页
- Then 只看到 D1 下的画布,D2 下的画布不显示

**AC-32 直接 URL 访问**
- Given 用户对画布 X 无权限
- When 用户直接访问 `/ops-analysis/view/networkTopology/<X>`
- Then 跳转 403 页面

**AC-33 操作权限**
- Given 用户仅有 `view-View` 权限
- When 打开画布
- Then 编辑模式不可进入,节点/连线不可拖动,工具栏新增/删除按钮 disabled

### 6.10 审计

**AC-34 操作留痕**
- Given 用户编辑画布(添加/删除/修改节点或连线)
- When 操作完成
- Then audit_log 写入一条记录(操作人、时间、动作、对象)

### 6.11 性能(基础要求)

**AC-35 画布打开延迟**
- Given 画布含 10 个节点 + 15 条连线,生产环境等价规模数据
- When 用户打开画布
- Then 首屏渲染 ≤ 3 秒(P95)

**AC-36 刷新耗时**
- Given 同 AC-35
- When 触发一次完整刷新
- Then 刷新耗时 ≤ 5 秒(P95),刷新期间不阻塞用户交互

## 7. 不在范围(显式)

明确列出**本期不做**的项,避免后续范围蔓延。

### 7.1 数据源 / 资产

| 不做项 | 原因 | 未来引入方式 |
|---|---|---|
| CMDB 资产联动 | 用户已明确"完全与 CMDB 无关" | 后续若需要,节点可扩展第二种来源类型 |

| 未监控资产的支持 | 接口状态依赖监控中心 SNMP 采集 | — |

|---|---|---|

|---|---|---|

## 8. 附录

### 8.1 关键决策回顾

| 决策点 | 选择 | 备选 |
|---|---|---|
| 整体形态 | 新建独立画布类型 | 嵌入大屏 widget / 复用现有 topology |
| 连线指标粒度 | 状态 + N 个指标(每接口一行) | 单指标 sparkline / 仅汇总 |
| 状态判定 | 仅 oper_status==2(Down) 中断 | 严格 SNMP / SNMP + 告警 / 自定义表达式 |
| 节点展示 | 标识 + 多指标 + 接口摘要 + sparkline | 仅标识 / + 指标 / + 接口摘要 |
| 实时性 | 复用 setInterval + reloadVersion | 手动 / 智能退避 / SSE |
| 数据源 | 仅 MonitorInstance | 仅 CMDB / 双源 |
| 存储 | view_sets JSON + NetworkLinkConfig 旁路表 | JSON-only / 全部结构化 |
| 节点拖拽 | 左侧节点库侧栏(复用 Topology 模式) | 双击画布弹搜索 / 工具栏弹窗 |
| 数据拉取 | 节点走 DataSource,连线状态走专用接口 | 全走 DataSource / 全走专用接口 |
