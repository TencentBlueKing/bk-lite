# 模型字段支持标签类型

日期：2026-03-04

## 方案概览

本方案严格基于 PRD 与 Requirement，目标是在 CMDB 中以最小改动新增 `tag` 字段类型能力，覆盖模型定义、实例维护、列表展示、搜索、导入导出、批量更新全链路。

总体策略：复用现有字段体系与交互框架，在不引入新服务、不扩展非目标能力的前提下，分阶段打通“类型接入 → 统一校验 → 交互适配 → 导入导出一致性”。

## 范围与约束

- 单模型最多 1 个标签字段。
- 标签字段 `attr_id` 固定为 `tag`。
- 标签字段内置约束固定：可编辑=是、唯一=否、必填=否。
- 标签数据语义：`key:value`，支持同 key 多 value。
- `key` 不可为空；`value` 不可包含空格、冒号、换行。
- 单实例最多 20 个标签对。
- 导入、导出、批量更新复用同一标签校验规则。

## 已确认决策

- 模式配置层级：按字段配置（每个 `tag` 字段独立）。
- 搜索组合逻辑：同字段多个标签条件全部 OR。
- 自由模式回写：新输入合法标签立即回写到对应模型字段配置的候选中。

## 分阶段计划

### 阶段一：字段类型与模型约束接入

**里程碑目标**
- 完成 `tag` 作为标准字段类型的前后端接入。
- 建立模型侧硬约束：单模型唯一、`attr_id=tag` 固定、删除二次确认。

**交付内容**
- 字段类型清单增加 `tag`。
- 模型字段配置在选择 `tag` 时自动填充并锁定 `attr_id=tag`。
- 创建/更新模型字段时执行“单模型最多一个标签字段”校验。
- 删除标签字段走二次确认流程。

### 阶段二：统一校验与实例维护打通

**里程碑目标**
- 标签规则在所有写入入口一致生效。

**交付内容**
- 建立统一标签解析/校验能力，覆盖：格式、字符限制、数量上限、重复处理。
- 接入实例新增、修改、详情页修改、复制、批量更新、导入。
- 严谨模式仅允许候选；自由模式允许自定义并自动回写候选。

### 阶段三：展示与搜索能力接入

**里程碑目标**
- 完成用户可感知的标签录入、展示、搜索闭环。

**交付内容**
- 实例维护使用级联选择：第一层 key、第二层 value，回填 `key:value`。
- 列表标签列按逗号分隔展示，超长截断并支持 tooltip。
- 搜索字段为标签时展示二层树筛选，支持多选与回显。
- 标签搜索固定精确匹配（Accurate 固定开启，不支持模糊）。

### 阶段四：导入导出一致性与上线准备

**里程碑目标**
- 导入/导出/批量更新规则一致并可稳定上线。

**交付内容**
- Excel 标签格式统一 `key:value`。
- 多标签分隔支持 `,` / `，` / 换行。
- 非法数据返回可定位错误（至少行号+字段名）。
- 完成回归验证与灰度上线准备。

## 阶段验收清单

- 模型层
  - 选择类型为 `tag` 时，`attr_id` 自动填充为 `tag` 且不可编辑。
  - 同一模型无法新增第二个标签字段。
  - 标签字段编辑仅允许改名称与分组，不允许改固定约束。
- 实例层
  - 新增/修改/复制/详情页修改均支持标签维护。
  - 标签非法输入（空 key、value 含空格/冒号/换行、超过 20 对）可被拦截并提示。
  - 自由模式新增标签可立即在候选中可选。
- 搜索与展示
  - 标签搜索组件为二层 key/value，多选可回显。
  - 标签搜索仅精确匹配。
  - 列表标签列格式与截断展示符合要求。
- 数据交换
  - 导入、导出、批量更新标签规则一致。
  - 导入错误能定位到具体行与字段。

## 风险与应对

- 风险：多入口规则不一致导致数据污染。  
  应对：统一校验能力下沉为公共逻辑，所有写入口强制复用。

- 风险：自由模式自动回写带来候选膨胀。  
  应对：仅回写通过校验的标准标签，并做去重。

- 风险：标签搜索状态复杂导致跨页选择异常。  
  应对：复用现有查询会话状态模型，限定最小新增状态字段。

- 风险：导入格式兼容导致错误提示不清晰。  
  应对：解析与校验分层，统一输出“行号 + 字段名 + 原因”。

## 非目标

- 三级及以上标签层级。
- 标签 value 模糊检索。
- 标签推荐、治理统计等运营能力。
- 非标签字段的联动改造。
