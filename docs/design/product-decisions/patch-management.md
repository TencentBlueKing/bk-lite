# 补丁管理产品决策记忆

- 最近更新：2026-06-25
- 当前规格：`docs/superpowers/specs/2026-06-17-patch-management-design.md`

## 产品定位

- 当前版本是 Windows + Linux 服务器补丁基线治理的**可商用验证版 MVP**。
- 产品主轴是“主机绑定基线 → 自动评估 → 风险治理 → 重启验证 → 恢复合规”。
- 扫描、安装、重启和验证任务是后台执行记录，不是用户日常工作的一级入口。

## 已确认范围

- 保留 Microsoft Update Catalog、WSUS、Linux yum/dnf/apt repo 和手工上传。
- 保留 PatchTarget 独立 Ansible 目标及 Node Manager 已纳管节点。
- 主机可以不绑定基线；未配置数量在首页展示并引导绑定。
- 每台主机同时最多绑定一个可复用基线。
- 基线为固定必装清单，一个基线只适用一种 OS 类型及版本范围。
- 提供全局周期评估、立即评估、三视角风险治理和执行记录。
- 支持多主机、多补丁的立即执行或一次性执行窗口。

## 已确认设计决策

### 1. 信息架构

- 一级菜单：`首页 / 风险治理 / 补丁库 / 基线管理 / 目标管理 / 设置`。
- 风险治理二级页面：`待治理风险 / 执行记录`。
- 设置二级页面：`补丁源 / 扫描设置 / Windows 版本管理`。
- MVP 不提供独立通知设置。

### 2. 首页和合规口径

- 首页重点展示纳管主机、已评估合规率、评估覆盖率、不合规、未配置基线、待治理风险和修复异常。
- 主机状态为合规、不合规、待评估、评估失败、未配置。
- 合规率只计算合规与不合规主机；未配置、待评估和评估失败单独展示。

### 3. 基线和主机绑定

- 基线内所有补丁要求均为必装项，不做推荐、可选、禁止或单机豁免。
- 每次基线变更产生不可变版本快照，历史任务和评估引用原版本。
- 新增补丁、修改基线或切换主机基线后，管理员选择立即扫描或按周期扫描。
- 等待重新评估时主机进入待评估，旧结果过期并退出当前合规率。

### 4. 基线要求不是固定安装包

- Windows 基线要求是指定 KB；指定 KB 或可信源确认的有效替代 KB 均可满足。
- Linux 基线要求是包名与最低版本；更高版本可满足。
- 数据层区分基线要求、当前满足证据和实际安装方案。
- 执行前重新计算替代补丁、版本和依赖；重大方案变化要求重新确认。

### 5. 风险项和治理状态

- 风险项按“主机 × 基线版本 × 基线要求”生成。
- 数据层分开保存合规状态和治理状态。
- 页面组合显示待修复、已计划、修复中、待重启、修复失败、已修复、已失效。
- 聚合层展示状态数量分布，状态归属于风险项。
- 同一风险项不能同时进入多个有效任务；同一补丁的不同主机可拆入不同执行窗口。

### 6. 补丁库就绪流水线

- 补丁状态为处理中、可用、需处理、不可用。
- 只有可用补丁能加入基线。
- 官方源自动完成元数据、包下载、完整性、适用范围和执行能力匹配。
- 管理员只处理缺包、元数据异常、下载失败和不支持原因。
- 手工补丁只补必要信息，不开放任意命令或脚本。

### 7. 提交前范围确认和执行时预检

- 提交前范围确认只展示治理范围、平台已知阻断项、执行窗口、重启策略和影响确认，不连接主机。
- 实时连通性、权限、磁盘、更新进程、包管理器、待重启和实际安装方案在执行时预检。
- 临时占用与环境异常必须区分，后者不进入等待。

### 8. 立即执行与执行窗口

- 立即执行不排队。遇到 BK-Lite 前序任务或系统更新进程占用时，对应主机未执行并支持人工重试。
- 执行窗口表示允许启动安装的时间范围。
- 窗口任务按主机独立 FIFO 串行，重叠窗口提示但不阻止创建。
- 临时占用可在窗口内等待；窗口结束后不再启动新安装。
- 已启动安装继续完成，并标记是否超出窗口。
- 队列区分等待前序任务、等待系统更新进程和环境异常。

### 9. 重启

- 重启策略只保留不自动重启、窗口内自动重启。
- 立即执行只能选择不自动重启。
- 系统使用统一重启预留时长；窗口剩余时间不足时转为待重启。
- 待重启风险通过单独的重启窗口任务处理，完成后自动验证。

### 10. 多补丁、取消和变更

- 一次任务可包含多台主机和多个补丁，同一主机安装完成后统一判断重启。
- 单个补丁失败不阻止无依赖的其他补丁。
- 计划任务保存风险项快照，基线后续新增补丁不会自动加入。
- 取消只影响尚未开始的主机；正在预检、安装、重启或验证的主机继续完成。
- 基线变更导致风险失效时，任务项执行时跳过；全部失效则任务自动取消。

### 11. Windows 与 Linux 扩展

- Windows 使用通用执行引擎和可配置版本目录。
- 内置 Windows Server 2008、2008 R2、2012、2012 R2、2016、2019、2022、2025。
- Windows 版本管理只维护识别信息，不展示生命周期，也不开放执行脚本。
- Linux 使用后端 Debian 系和 RHEL 系插件；页面不提供插件状态或代码管理入口。
- Windows 未识别为 `unmapped`；Linux 未匹配插件为 `unsupported`。

## 明确后置

- Mac 和第三方应用补丁。
- 动态规则基线和单机豁免。
- 审批、测试批准、自动快照和自动回滚。
- 周期自动修复策略和用户可配置分批策略。
- 独立报表中心和独立通知设置。
- 补丁卸载、回滚和专用 Patch Agent。
- 页面自定义 Linux 插件。

## 已替代决策

- 2026-06-25：扫描任务/安装任务一级菜单被基线合规与风险治理模式替代。
- 2026-06-25：扫描通知和通知日志退出 MVP。
- 2026-06-25：指定开始时间改为立即执行或一次性执行窗口。
- 2026-06-25：指定时间重启改为不自动重启或窗口内自动重启。
- 2026-06-25：立即执行 FIFO 排队方案被替代；立即执行遇到占用直接未执行，只有窗口任务排队。
- 2026-06-25：创建任务前远程预检被替代；弹窗只做范围确认，实时预检作为执行阶段。
- 2026-06-25：基线直接引用固定安装包的理解被替代；基线保存最低要求，实际安装方案可使用替代补丁或更高版本。
- 2026-06-18：固定 Windows/Linux 支持矩阵被替代。Windows 改为可配置版本目录，Linux 改为后端插件能力。
- 2026-06-17：极简 MVP 建议被替代。当前版本仍是可商用验证闭环。
