# 补丁管理产品决策记忆

- 最近更新：2026-06-22
- 当前规格：`docs/superpowers/specs/2026-06-17-patch-management-design.md`

## 产品定位

- 当前版本定位为 Windows + Linux 服务器补丁管理的**可商用验证版 MVP**，不是极简演示。
- 核心价值是完成补丁源接入、补丁建档、扫描评估、安装、重启、通知、导出和审计闭环。

## 已确认范围

- 保留 Microsoft Update Catalog、WSUS、Linux yum/dnf/apt repo 接入。
- 保留手动和周期扫描、双视角结果、通知日志、Excel 导出。
- 保留立即/指定时间安装和不重启/立即/指定时间重启。
- 同时支持 PatchTarget 独立 Ansible 目标和 Node Manager 已纳管节点。
- 支持 Windows/Linux OS 安全补丁和管理员手工导入的通用补丁。

## 已确认设计决策

1. **Windows 使用通用执行引擎和可配置版本目录。**
   - 内置 Windows Server 2008、2008 R2、2012、2012 R2、2016、2019、2022、2025。
   - 管理员可增加版本别名、构建号范围、架构和启停配置。
   - 配置页面不开放扫描、安装或重启脚本。
   - 未匹配配置的 Windows Server 状态为 `unmapped`，补充映射后可重扫。

2. **Linux 使用后端插件扩展。**
   - MVP 内置 Debian 系插件和 RHEL 系插件。
   - 插件负责系统识别、源解析、扫描、适用性判断、安装、验证和重启检测。
   - 页面只读展示插件能力，不支持在线编写或上传插件。
   - 未匹配插件为 `unsupported`；匹配后执行异常为 `failed`。

3. **不展示微软生命周期标签，也不以生命周期阻止执行。**
   - 能否扫描和安装由 BK-Lite 引擎、版本映射、补丁适用性及包可用性决定。
   - 微软生命周期属于需要按当时官方信息核验的外部事实。

4. **安装前必须有补丁与目标预览矩阵。**
   - 至少区分可执行、已安装跳过、不适用跳过、缺包阻断、Windows 未映射、Linux 不支持、目标不可达和已有待重启。

5. **不提供用户可配置的分批和维护窗口，但保留系统并发上限。**

6. **自动源失败必须有手工降级路径。**
   - Catalog、WSUS 或 Linux repo 同步失败后允许手工上传并补齐元数据。

7. **取消语义按下发边界处理。**
   - 未开始任务可取消；执行中只取消未下发目标；已下发目标继续等待回传。

## 明确后置

- Mac 和第三方应用补丁。
- 测试批准流程。
- 长期自动部署策略管理。
- 用户可配置的分批执行和维护窗口。
- 独立报表中心。
- 补丁卸载或回滚。
- 专用 Patch Agent。
- 页面自定义 Linux 插件。

## 仍待确认

无。

## 已替代决策

- 2026-06-18：固定 Windows/Linux 支持矩阵被替代。Windows 改为可配置版本目录，Linux 改为后端插件能力。
- 2026-06-17：极简 MVP 建议被替代。当前 MVP 保留完整源接入、周期扫描、通知、双目标来源和计划重启。
