# 告警通知可观测性产品决策记忆

- 最近更新：2026-07-10
- 当前规格：`specs/changes/legacy-2026-07-10-alert-notification-tooltip/spec.md`

## 产品定位

告警通知明细服务于一线告警处置闭环，让处理人员无需查询后台日志即可判断通知对象、时间、渠道、结果和失败原因。

## 已确认范围

- 告警列表和详情页的“通知情况”使用同一 Tooltip 组件。
- Tooltip 显示通知总次数及最近 5 条通知。
- 每条显示时间、渠道、接收人和结果；失败时显示脱敏原因。
- 失败原因仅记录功能上线后的新通知，历史数据不回填。

## 已确认设计决策

- 复用现有 `NotifyResult`，新增可空 `failure_reason` 字段，不新增通知记录表。
- 采用现有 Alert API 批量内嵌通知摘要与最近 5 条明细，不在悬停时额外请求。
- 统一识别 `result`、`errcode`、`code` 和调用异常，单次失败不能阻断后续通知。
- 失败原因在落库前脱敏并截断，不保存通知正文或完整渠道响应。

## 明确后置

- 不提供完整通知历史抽屉或独立通知审计页面。
- 不提供通知重试入口、失败原因搜索或通知统计报表。
- 不回填历史失败原因。
