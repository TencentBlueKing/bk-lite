# 监控告警阈值单位产品决策记忆

- 最近更新：2026-07-14
- 当前规格：`docs/superpowers/specs/2026-07-13-monitor-threshold-unit-decoupling-design.md`

## 产品定位

监控策略单位配置服务于真实阈值判断闭环。用户应能明确区分公式结果的量纲与阈值输入量级，页面展示、预览和后台扫描必须使用同一比较口径。

## 已确认范围

- 公式结果单位与告警阈值单位独立配置。
- 公式结果单位默认 `%`，可选择全部有效单位。
- 阈值单位只允许结果单位所属的同一单位体系，结果单位为 `%` 时只允许 `%`。
- 阈值支持负数、零和小数，覆盖减法、净增量和偏差类公式。
- 同体系单位在比较前自动换算，历史策略保持原有比较语义。

## 已确认设计决策

- 使用独立 `threshold_unit` 字段解释 `threshold[].value`，不再让 `calculation_unit` 同时承担结果单位和阈值单位。
- `metric_unit` 表示原始采集单位，`calculation_unit` 表示最终结果单位，`threshold_unit` 表示阈值输入单位。
- 数据库存储用户输入的原始阈值数值；预览和扫描在运行时把阈值转换到结果单位后比较，不改写策略配置。
- 不提供容量、时长等结果到 `%` 的隐式转换。没有分母或基准时，跨体系百分比没有可靠数学语义，必须阻止保存。
- 历史空 `threshold_unit` 按 `calculation_unit` 回退，避免升级改变告警结果。
- 阈值型图表（策略预览、告警详情）以 `threshold_unit` 为展示口径：服务端将曲线数据副本换算到阈值单位，Y 轴、曲线和阈值线统一；扫描和持久化仍使用 `calculation_unit`。

## 明确后置

- 自动分析公式并推导量纲。
- 为百分比配置独立的分母或基准指标。
- 每个告警等级使用不同阈值单位。

## 仍待确认

无。

## 已替代决策

- 2026-07-13：替代“阈值单位完全跟随公式结果单位、公式模式隐藏阈值单位选择器”的旧设计；结果单位与阈值单位改为独立状态和字段。
- 2026-07-13：原需求中的“指标类 + %”收敛为“结果单位同体系；百分比体系仅允许百分比”，避免无法换算的跨量纲误告警。
