# 知识库检查审核页面视觉还原设计

## 目标

将 Storybook `OpsPilot/WikiDecisionRecordPrototype` 的信息架构和视觉布局落入正式的“知识库 → 检查审核”页面。正式页面继续使用现有真实接口、国际化、权限与决策提交逻辑，不复制 Story 中的模拟数据或硬编码业务状态。

## 范围

- 重构 `WikiDecisionCenter` 的页面结构与样式。
- 保留 `CheckTab` 作为正式入口和数据刷新边界。
- 覆盖待决策、决策记录、选中项、加载、空状态、过期决策和规则撤销状态。
- 保留现有“保留当前知识”“使用新知识”“编辑后采用”“保持页面独立”“确认页面合并”等动作。
- 不修改后端接口、数据模型、知识构建流程或 Storybook 原型本身。

## 页面结构

正式页面采用单一工作台容器，而不是多个独立卡片：

1. 顶部标题区显示当前视图、数量、说明和待决策/决策记录切换。
2. 主体左栏显示决策事项列表，保留类型、时间、标题、摘要和处理状态。
3. 主体右栏显示选中事项的决策类型、状态、标题、摘要和时间。
4. 摘要区展示需要决策的原因、触发来源、影响范围与可恢复性。
5. 差异区并列展示当前知识和候选知识，以语义化增删提示突出关键变化。
6. 底部固定操作区解释动作后果，并承载当前决策可用的操作按钮。

窄屏下左右栏改为纵向布局；列表具有合理最大高度，详情自然向下排列，操作区允许换行，避免整页横向滚动。

## 组件与数据流

`CheckTab` 继续获取决策列表并向 `WikiDecisionCenter` 传入状态和回调。`WikiDecisionCenter` 只负责视图切换、选中状态和展示组合：

- 切换视图时保留各自选中项；当前选中项消失时回退到第一项。
- 所有写操作继续调用父级既有回调，由父级负责 API、loading、成功提示和刷新。
- 编辑后采用继续使用现有编辑弹窗，不在工作台内复制表单。
- 撤销自动复用继续使用确认交互，明确说明不会回滚已生效知识。

## 视觉规则

- 使用 `web/DESIGN.md` 和全局 `--color-*` token，主色只表达主操作、选择和焦点。
- 主要分层依赖背景与细边框，不给带边框面板叠加大范围阴影。
- 卡片圆角不超过 12px，按钮和标签沿用 Ant Design。
- 当前知识、候选知识和决策状态使用成组的背景、文字、图标语义，不只依赖颜色。
- 正文最小 12px，标题保持产品界面的固定字号层级。

## 异常与边界状态

- 加载：在工作台内容区展示 Skeleton，避免页面结构跳动。
- 空状态：分别说明“暂无待决策事项”和“暂无已处理事项”，仍允许切换视图。
- 过期：保留现有刷新入口，禁止继续提交旧上下文。
- 长文本：标题、来源与摘要允许换行或省略，并可通过 Tooltip 查看完整内容。
- 写操作：按钮呈现 loading/disabled，防止重复提交。

## 测试与验收

- 先为正式组件补行为测试，覆盖视图切换、选择事项、空状态和动作回调。
- 运行涉及文件的定向 ESLint 与 TypeScript 检查。
- 在 Storybook 与正式页面分别进行桌面和窄屏视觉检查。
- 验收重点是正式页面与原型的信息层级、尺寸关系和操作位置一致，同时真实业务行为无回归。
