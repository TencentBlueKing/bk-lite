# CMDB 资产模型树展开习惯设计

## 背景

CMDB 资产页面左侧模型树当前在分类数据加载后默认展开全部分类。用户手动收起分类后，刷新页面或下次进入页面时状态会丢失；模型分类还可能随后台配置新增、删除或调整，因此不能缓存整棵树或假设历史节点始终存在。

## 目标

- 在模型树搜索框右侧提供“全部展开”和“全部收起”两个图标按钮。
- 记录当前登录用户手动展开、收起分类的习惯，并在同一浏览器再次进入页面时恢复。
- 后台删除历史分类、增加新分类或返回空分类时，恢复过程保持安全且不报错。
- 搜索产生的自动展开是临时状态，不覆盖用户保存的展开习惯。

## 非目标

- 不把展开状态同步到服务端或跨浏览器同步。
- 不缓存模型树完整数据、模型名称、实例数量或模型叶子节点。
- 不改变模型选择、实例列表加载、URL 参数和搜索匹配规则。
- 不为历史缓存格式提供无版本迁移；缓存键升级时按新默认状态重新开始。

## 方案选择

采用“保存已收起分类 ID”的方案，而不是保存已展开节点或完整树快照。

- 当前页面默认全部展开，保存收起集合能延续现有默认行为。
- 新增分类不在收起集合中，因此默认展开并立即可见。
- 已删除分类即使残留在缓存中，也会在恢复时被当前分类集合过滤。
- 模型节点是叶子节点，不参与展开状态持久化。

缓存键使用带版本号和用户 ID 的命名空间：

```text
bk-lite:cmdb:asset-model-tree:v1:<userId>
```

缓存值仅包含字符串分类 ID：

```json
{"collapsedClassificationIds":["cloud","middleware"]}
```

## 交互设计

搜索输入框与两个图标按钮位于同一行。按钮使用 Ant Design 小型图标按钮：

- “全部展开”：展开当前后台返回的全部分类，并把缓存中的收起集合更新为空。
- “全部收起”：收起当前后台返回的全部分类，并把这些分类 ID 写入缓存。
- 每个图标按钮提供中英文 Tooltip、可访问名称 `aria-label`，图标本身不承载文本语义。
- 没有分类数据时按钮禁用，避免产生无意义操作。

用户点击单个分类的展开箭头时，根据 Ant Design Tree 展开事件中本次切换的分类键，将对应分类 ID 从收起集合移除或加入，并立即写入当前用户缓存。这样搜索过滤掉的其他分类不会被误判为收起。只记录以 `group:` 为前缀且当前真实存在的分类节点，忽略模型叶子节点和未知键。

## 状态与数据流

1. 页面获得当前用户 ID 和后台模型分类后，构造当前有效分类 ID 集合。
2. 从对应用户的版本化 localStorage 键读取已收起分类 ID。
3. 对缓存内容做结构校验，只保留字符串值，并与当前有效分类 ID 取交集。
4. 当前树的 `expandedKeys` 等于“全部当前分类键减去有效收起分类键”。
5. 用户单节点操作时只增删本次切换的收起分类 ID；点击批量按钮时一次替换完整收起集合。两类操作都同步更新受控 `expandedKeys` 和缓存偏好。
6. 后台新增分类时，该分类不在收起集合中，默认展开；后台删除分类时，过期 ID 被忽略。

缓存逻辑放在独立纯函数工具中，页面组件只负责用户 ID、树数据和 Ant Design Tree 事件的接线。这样缓存解析、分类对账和展开键计算可以不依赖 React 或浏览器 DOM 单独测试。

## 搜索行为

- 提交非空搜索时，匹配结果所需节点临时全部展开，但不写 localStorage。
- 清空并提交搜索时，重新按照内存中的用户偏好计算展开状态，而不是默认全部展开。
- 搜索期间用户主动点击分类箭头或批量按钮属于明确操作：更新用户偏好，并立即控制当前可见树；清空搜索后继续使用更新后的偏好。
- 输入框文字变化但尚未提交搜索时，保持现有搜索触发方式，不扩大本次需求范围。

## 容错

- localStorage 不可用、读取抛错、值不是合法 JSON 或结构不合法时，回退为“全部展开”。
- 写入 localStorage 失败时仍更新当前页面状态，不向用户弹出错误消息，也不阻断树操作。
- 缓存包含后台已不存在的分类时静默丢弃，不索引不存在节点，不抛异常。
- 用户 ID 尚未就绪时不读取或写入共享缓存；待用户 ID 可用且分类数据已加载后再恢复。
- 分类列表为空时展开键和收起集合均为空。

## 测试策略

按照 TDD 先写失败测试，再实现最小代码。至少覆盖：

- 无缓存时所有当前分类默认展开。
- 合法缓存可以恢复部分收起状态。
- 缓存中的已删除分类被忽略且不报错。
- 后台新增分类默认展开。
- 非法 JSON、错误数据结构和非字符串分类 ID 安全回退。
- 全部展开生成空收起集合，全部收起只包含当前分类。
- 单节点展开事件只持久化当前有效分类的收起状态。
- 搜索自动展开不写缓存，清空搜索恢复已保存习惯。
- 不同用户使用不同缓存键。

实现后运行新增定向回归测试、触及文件 ESLint，并执行 `pnpm lint` 与 `pnpm type-check`。若全量门禁被任务外既有问题阻断，需要记录具体错误并确认本次触及文件无新增问题。

## 验收标准

- 用户可以通过两个图标按钮一键展开或收起全部模型分类。
- 用户刷新或重新进入 CMDB 资产页面后，单节点和批量操作形成的习惯按用户恢复。
- 同一浏览器切换账号时，各账号的模型树习惯互不影响。
- 删除缓存中曾记录的分类、增加新分类、返回空分类或破坏缓存内容均不会导致页面报错。
- 搜索后清空搜索，用户原有展开习惯仍然存在。
