# 聚合告警标题展示规则设计

## 目标

聚合指纹继续使用按策略 `group_by` 顺序生成的非空 `key=value` 组合，保证聚合身份稳定；Alert 标题仅展示这些维度的 value，避免把内部字段名暴露给用户。

## 已确认规则

1. 从内部原始指纹中移除策略 ID，并按 `|` 取得非空维度段。
2. 每段仅取第一个 `=` 后面的完整 value；value 内后续出现的 `=` 不应被截断。
3. 最后一个非空 value 作为标题主体。
4. 其余 value 保持原顺序，以中文逗号 `，` 分隔，并放入中文括号 `（）`。
5. 只有一个 value 时不展示空括号。
6. 致命级别使用后缀 `发生严重问题`；普通级别使用后缀 `检测到异常`。
7. MD5 兜底指纹不改写标题；没有可展示 value 时保留聚合查询产生的原始事件标题。

## 示例

- `resource_name=1111` → `1111 发生严重问题`
- `service=支付|location=北京|resource_name=1111` → `1111（支付，北京）发生严重问题`
- `service=支付|resource_name=1111|item=memory` → `memory（支付，1111）检测到异常`

## 实现范围

仅调整 `AggregationProcessor._normalize_fingerprint` 的可见标题格式化。内部 MD5 指纹、聚合维度顺序、Alert 内容和严重级别判断保持不变；已有 OR 聚合维度判断改动继续保留。

## 验收

- 单维和多维标题符合上述格式。
- 标题中不出现维度 key、`=` 或 `|` 分隔符。
- 内部指纹仍由完整的有序 `key=value` 原文计算。
- 按用户要求，本次不运行自动化测试，由用户替换到线上后验收实际 Alert。
