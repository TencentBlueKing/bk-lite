# Ops Analysis 拓扑图 UI/UX 与核心能力差距评分表设计

## 背景

运营分析模块的拓扑图已经具备基础编辑、数据绑定、大屏展示和导入导出能力，但从产品经理视角看，仍需要一套统一的评估框架来判断它与业内最佳实践之间的差距，并为 roadmap 排优先级提供依据。

本设计文档用于定义这套评分框架，并给出当前版本的首版差距评分。

## 目标

- 形成一张可在产品评审会直接使用的拓扑图差距评分表。
- 同时覆盖 UI/UX 与核心能力两类评价维度。
- 评分结果既能支持管理层判断，也能直接沉淀为需求池输入。
- 明确差距分、影响级别和优先级之间的映射关系。

## 非目标

- 不展开到研发实现方案或排期拆解。
- 不与某一个单一竞品逐项做功能复刻式对比。
- 不覆盖仪表盘、架构图等其他画布类型的完整评分。

## 评估基准

- 对标方式：业内最佳实践综合基准。
- 使用场景：综合产品视角，兼顾体验成熟度与能力成熟度。
- 输出颗粒度：详细版，可直接支撑后续需求拆分。

## 评分结构

采用 100 分制，分为两大主维度。

### 一、UI/UX（40 分）

1. 信息架构与入口清晰度（5）
2. 画布可读性与视觉层次（6）
3. 编辑器易用性（8）
4. 交互反馈与状态可感知性（6）
5. 演示 / 大屏体验（5）
6. 学习成本与上手效率（5）
7. 空态 / 错误态 / 引导完善度（5）

### 二、核心能力（60 分）

1. 节点建模与图元丰富度（8）
2. 连线表达与关系语义（8）
3. 数据绑定与指标展示能力（10）
4. 过滤 / 联动 / 动态刷新能力（8）
5. 布局与大图导航能力（8）
6. 编辑治理能力（8）
7. 场景适配能力（6）
8. 扩展性与集成能力（4）

## 评分口径

单项采用 5 分制：

- 5 分：接近业内优秀水平，可直接作为亮点能力。
- 4 分：能力完整，仅存在少量体验或效率短板。
- 3 分：基本可用，但存在明显断点。
- 2 分：能力存在，但不成体系。
- 1 分：能力明显缺失，或仅有非常初级支撑。

## 差距定义与优先级映射

### 差距分定义

差距分 = 最佳实践目标分 - 当前得分

- 差距分 0-1：优化项
- 差距分 2：补强项
- 差距分 3：重点建设项
- 差距分 4：战略补课项

### 优先级建议

- 差距分 >= 3 且影响高：P0
- 差距分 = 2 或影响中高：P1
- 差距分 <= 1：P2

## 正式输出表结构

评分表建议统一包含以下字段：

- 评分维度
- 当前表现
- 最佳实践参考
- 当前得分
- 差距分
- 影响级别（高 / 中 / 低）
- 建议优先级（P0 / P1 / P2）
- 典型改进方向

## 核心结论

1. 当前拓扑图已经具备“可搭建、可展示、可保存”的基础产品能力。
2. 当前主要强项集中在数据绑定、大屏展示、基础编辑和导入导出。
3. 当前主要差距集中在新手上手、异常引导、自动布局、大图导航和关系语义表达。
4. 从 roadmap 视角看，最优先的建设方向应为：
   - P0：新手引导与模板化起步
   - P0：空态 / 错误态 / 数据异常处理产品化
   - P1：自动布局 + 搜索定位 + 路径聚焦
   - P1：关系语义增强 + 行业场景模板

## 评分依据（当前实现观察）

- 已具备缩放、适配视图、全屏、固定分辨率与撤销重做等能力。
- 已具备基础节点类型、连线样式、节点复制、层级调整等编辑能力。
- 已具备统一筛选、命名空间、自动刷新、数据绑定等运行态能力。
- 已支持拓扑 YAML 导入导出，说明对象迁移和集成能力已有基础。
- 仍缺少自动布局、搜索定位、路径聚焦、模板化起步、版本协作等成熟拓扑产品常见能力。

## 后续使用建议

- 在产品评审会上，先汇报总分与 P0 / P1 项，再进入单项分析。
- 在 roadmap 规划中，以差距分和影响级别共同决定优先级，而不是只看功能缺失。
- 若后续要转为需求池，建议按“体验补课”“能力补课”“场景化增强”三类继续拆分。
