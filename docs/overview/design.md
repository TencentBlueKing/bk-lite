# BK-Lite 设计理念

> BK-Lite 是腾讯蓝鲸生态的轻量级 IT 运维产品，秉承「恰如其分，渐进式设计」的核心原则。

---

## 1. 核心设计原则

### 1.1 轻量优先

- **最小化资源占用**：选用 VictoriaMetrics/VictoriaLogs 替代 ELK，存储和内存消耗降低 70% 以上
- **快速启动**：所有组件均可在分钟级完成部署并投入使用
- **依赖精简**：避免引入重量级中间件，优先选用轻量替代方案

### 1.2 AI 原生

- **OpsPilot 内置**：AI 运维助手作为一等公民与各功能模块深度集成
- **智能异常检测**：通过算法服务（`algorithms/`）提供时序异常、日志分类等智能能力
- **自然语言交互**：用户可通过对话方式查询数据、分析问题

### 1.3 渐进式体验

- **按需启用**：各功能模块独立可用，用户可从最小功能集开始，按需扩展
- **零门槛上手**：内置大量开箱即用的默认配置（采集插件、仪表盘模板、告警策略）
- **平滑扩展**：从单机部署到 Kubernetes 集群，架构平滑演进

### 1.4 模块化架构

- **松耦合**：各功能模块通过 NATS 消息总线通信，相互独立、可替换
- **统一基座**：共享系统管理（用户/权限/组织）和 CMDB（资产配置）底座
- **开放集成**：所有模块提供 RESTful API，支持第三方系统集成

## 2. 技术选型原则

| 场景 | 选型 | 理由 |
|------|------|------|
| 后端框架 | Django 4.2 + DRF | 成熟稳定，ORM 完善，生态丰富 |
| 日志存储 | VictoriaLogs | 低资源占用，LogsQL 强大，运维简单 |
| 指标存储 | VictoriaMetrics | 高性能时序数据库，兼容 Prometheus |
| 消息总线 | NATS | 轻量、高吞吐、支持 TLS |
| 任务队列 | Celery + Redis | 成熟可靠，与 Django 深度集成 |
| 前端框架 | Next.js + React | 全栈能力，Server Components，性能优异 |
| 图数据库 | FalkorDB / Neo4j | CMDB 关系图谱，图查询灵活高效 |
| 对象存储 | MinIO | 自部署 S3 兼容存储，无云厂商依赖 |

## 3. 产品模块设计

### 3.1 核心支撑模块

这些模块为其他功能提供基础能力，所有用户默认使用：

- **系统管理**：用户认证、组织架构、角色权限、SSO 接入
- **CMDB**：配置管理数据库，维护所有 IT 资产的配置信息与关联关系
- **节点管理**：统一管理被监控主机的 Agent 生命周期

### 3.2 可观测性模块

提供完整的 IT 可观测性（Metrics + Logs + Traces）能力：

- **监控管理**：指标采集、阈值告警、监控仪表盘
- **日志管理**：多源日志采集、实时检索、智能分析、日志告警
- **告警中心**：统一汇聚监控告警与日志告警，统一分发通知

### 3.3 智能运维模块

- **OpsPilot**：AI 运维助手，基于 LLM 提供对话式运维能力
- **运营分析**：面向管理层的运营仪表盘与架构图

---

## 4. 数据安全设计

- **最小权限原则**：用户仅能访问其组织权限范围内的数据
- **数据隔离**：多租户场景下，日志分组与监控分组按组织严格隔离
- **传输加密**：生产环境强制 TLS 加密
- **敏感数据保护**：密钥通过环境变量注入，不写入代码和配置文件
