# BK-Lite 日志管理白皮书

> 版本：V3.8 | 发布时间：2025年9月

---

## 1. 产品概述

BK-Lite 日志管理是蓝鲸轻量版（BK-Lite）的核心功能模块之一，提供企业级日志采集、传输、存储、检索与分析的一体化解决方案。产品面向 IT 运维、安全审计及研发团队，以低资源消耗、轻量部署为核心设计理念，帮助用户快速构建日志可观测性平台。

### 1.1 核心价值

- **轻量高效**：基于 VictoriaLogs 存储引擎，相较传统 ELK 方案资源消耗降低 70% 以上
- **多源采集**：内置 6 类采集探针，覆盖操作系统、容器、中间件、网络设备等主流日志来源
- **智能分析**：支持 LogsQL 查询语言，提供字段提取、聚合统计、趋势分析等能力
- **实时告警**：支持关键字告警与聚合告警两种策略，异常发生时即时推送通知
- **权限管控**：按组织架构进行日志分组权限隔离，满足多租户场景需求

## 3. 核心功能

### 3.1 日志采集（集成）

#### 3.1.1 支持的采集探针

**Vector（通用采集器）**

Vector 是高性能、低资源占用的日志采集与处理器，支持以下采集类型：

| 采集类型 | 说明 |
|----------|------|
| File | 本地文件日志采集，支持多行合并（Java堆栈、续行符等） |
| Docker | Docker 容器日志采集 |
| Syslog | 系统 Syslog 日志采集（TCP/UDP） |
| Exec | 命令执行结果日志采集 |
| Kafka | Kafka 主题消息采集 |
| Redis | Redis 数据流采集 |
| MongoDB | MongoDB 操作日志采集 |
| MySQL | MySQL 慢查询/错误日志采集 |
| Nginx | Nginx 访问/错误日志采集 |
| Apache | Apache 访问/错误日志采集 |
| PostgreSQL | PostgreSQL 日志采集 |
| RabbitMQ | RabbitMQ 消息队列日志采集 |
| Elasticsearch | Elasticsearch 服务日志采集 |

**Filebeat（应用日志采集器）**

专为结构化应用日志设计，内置常见中间件模块：Apache、Nginx、MySQL、Redis、Kafka、MongoDB、PostgreSQL、RabbitMQ、Elasticsearch。

**Packetbeat（网络流量分析）**

实时分析网络数据包，支持协议：
- HTTP 流量解析
- 网络流（Flows）统计
- ICMP 协议分析
- DHCP 协议解析

**Auditbeat（安全审计）**

- 文件完整性监控（File Integrity Monitoring）
- 系统进程审计
- 用户行为记录

**Winlogbeat（Windows 日志）**

- Windows 系统事件日志（Security、System、Application）
- PowerShell 执行日志
- Sysmon 日志

**Snmptrapd（网络设备）**

- SNMP Trap 采集，适配网络交换机、路由器等设备

#### 3.1.2 Kubernetes 原生支持

提供 Kubernetes 专属采集方案，以 DaemonSet 方式部署：

- **Vector DaemonSet**：自动采集集群所有节点的容器日志，挂载 `/var/log` 与 Docker 容器目录
- **自动元数据注入**：自动注入 Node 名称、Pod 名称、Namespace 等 K8s 元数据

#### 3.1.3 多行日志合并

针对 Docker 与文件采集，支持智能多行合并：

| 合并模式 | 适用场景 |
|----------|----------|
| `continue_through` | Java 异常堆栈 |
| `continue_past` | 续行符结尾的多行日志 |
| `halt_before` | 以特定关键字开头的日志块 |
| `halt_with` | 以特定关键字结尾的日志块 |

### 3.2 日志存储

#### 3.2.1 VictoriaLogs 存储引擎

BK-Lite 日志管理采用 **VictoriaLogs** 作为核心存储引擎，相比传统 Elasticsearch 具有以下优势：

- **资源占用低**：相同数据量下，内存和磁盘占用约为 Elasticsearch 的 1/5～1/3
- **写入性能高**：支持每秒百万级日志写入
- **查询效率高**：原生支持 LogsQL 查询语言，针对日志场景深度优化
- **运维简单**：单二进制部署，无需 JVM，启动时间秒级

#### 3.2.2 日志分组与权限

- 支持按组织架构定义**日志分组**，每个分组对应一组日志过滤规则
- 内置 `【全日志数据】` 默认规则，归属 `default` 分组
- 用户检索时，仅可查看其所属组织授权的日志分组数据

### 3.3 日志检索

#### 3.3.1 查询语言（LogsQL）

BK-Lite 日志检索基于 **LogsQL** 查询语言，提供强大的日志过滤与分析能力：

**基本查询示例**

```logsql
# 关键词检索
error

# 短语检索
"connection refused"

# 时间范围 + 关键词
_time:5m error

# 排除特定内容
_time:1h error -"test environment"

# 字段过滤
app:nginx AND level:error

# 字段值包含
host:~"prod-.*"
```

**聚合分析示例**

```logsql
# 按应用统计错误数
_time:1h error | stats count() by (app)

# Top 10 错误消息
_time:1h level:error | stats count() by (_msg) | sort by (count) desc | limit 10

# 时间趋势分析
_time:24h error | stats count() by (_time:1h)
```

#### 3.3.2 检索功能特性

- **实时流式检索**：支持 Tail 模式，实时追踪日志流（SSE 长连接）
- **字段提取与展示**：自动解析 JSON 日志字段，支持自定义字段展示列
- **字段值补全**：输入查询语句时，自动提示字段名与字段值
- **直方图分布**：展示检索结果在时间轴上的分布情况
- **检索条件保存**：支持保存常用检索条件，按组织隔离可见范围
- **快捷字段操作**：点击字段值可快速添加为过滤条件或排除条件

#### 3.3.3 展示模式

| 模式 | 说明 |
|------|------|
| 列表模式 | 表格化展示，支持自定义表头字段 |
| 终端模式 | 类 tail -f 样式，适合实时监控 |

### 3.4 日志分析（仪表盘）

#### 3.4.1 内置仪表盘

BK-Lite 日志分析提供可视化仪表盘，内置以下组件类型：

| 组件类型 | 说明 |
|----------|------|
| 日志消息组件 | 展示原始日志消息，支持条件过滤与字段选择 |
| 表格组件 | 聚合统计结果表格，支持多种计算函数 |
| 单值组件 | 关键指标单值展示（COUNT/AVG/MAX/MIN 等） |
| 折线图组件 | 时序趋势分析图 |

内置 **Packetbeat 基础仪表盘**，开箱即用展示网络流量统计。

#### 3.4.2 查询条件

仪表盘查询条件支持：
- 日志分组（权限过滤）
- 时间筛选器（相对时间/绝对时间范围）

### 3.5 告警与事件

#### 3.5.1 告警策略类型

**关键字告警**

- 基于 LogsQL 查询语句，周期性检测日志中是否出现指定关键词
- 检测周期：可配置（默认 5 分钟）
- 触发规则：单策略所有匹配事件合并为 1 条告警

**聚合告警**

- 基于 LogsQL 查询 + 聚合字段，对日志中的指标进行聚合统计
- 支持设置聚合函数（COUNT/AVG/MAX/MIN 等）、条件（>/</ =）与阈值
- 每个聚合字段值独立生成告警（如按 `host` 聚合，每台主机独立告警）

#### 3.5.2 告警通知

支持多种通知渠道：
- 邮件
- 企业微信
- 企业微信群助手（Webhook）
- 钉钉

#### 3.5.3 事件管理

- 告警事件列表，支持按日志分组的树形展示
- 关键字告警详情：基础信息 + 原始日志（可跳转到日志搜索）+ 事件列表
- 聚合告警详情：聚合字段值 + 表格化日志（可跳转）+ 事件列表

## 4. 数据流转说明

```
日志产生
   │
   ▼
采集探针（Vector / Filebeat / Packetbeat 等）
   │  采集、预处理（解析、过滤、字段提取）
   ▼
NATS 消息总线
   │  可靠异步传输
   ▼
BK-Lite 日志接收服务
   │  格式标准化、分组规则匹配
   ▼
VictoriaLogs
   │  持久化存储
   ▼
日志检索 / 分析仪表盘 / 告警策略
   │  用户通过 Web 控制台查询与管理
   ▼
告警通知（企业微信 / 邮件 / 钉钉）
```

```
┌──────────────────────────────────┐

         ↑ NATS
```

## 7. 与监控模块的联动

BK-Lite 日志管理与监控管理模块深度集成：

- 日志告警与监控告警统一汇聚到告警中心
- 日志事件可关联到 CMDB 资产（主机/服务）
- 支持通过告警跳转到对应日志检索界面，快速定位根因
