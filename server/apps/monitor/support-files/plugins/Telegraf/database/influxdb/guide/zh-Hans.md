# InfluxDB 监控接入指南

## 前置要求

- 目标实例已启动 InfluxDB v1 服务，默认可访问 `8086` 端口。
- InfluxDB v1 的 `/debug/vars` 接口已开放（Telegraf `inputs.influxdb` 通过该接口采集运行指标）。
- 采集节点到 InfluxDB 地址网络可达（含安全组 / 防火墙放通）。
- 若接口启用了 Basic Auth，请提前备好用户名和密码；未启用则可留空。
- 页面「服务器地址」需填写完整 URL，以 `http://` 或 `https://` 开头，并以 `/debug/vars` 结尾，例如 `http://10.0.0.10:8086/debug/vars`。

## 推荐账号权限

- InfluxDB v1 的 `/debug/vars` 是只读的运行时统计接口，不涉及库表数据。
- 若实例未开启 Basic Auth，无需任何账号即可采集。
- 若已开启 Basic Auth，使用任意能通过认证访问 `/debug/vars` 的只读账号即可，避免使用高权限管理员账号。

## 接入步骤

1. 在采集节点上确认可访问目标 InfluxDB 的 `/debug/vars` 接口。
2. 在监控对象接入页填写服务器地址、用户名/密码（如启用认证）、采集间隔（默认 `60s`）和超时时间（默认 `30s`）；HTTPS 场景按需填写证书路径或开启「跳过证书校验」。
3. 在「监控对象」表格中填写节点、服务器地址、实例名称和所属组。
4. 点击「确认」保存配置，等待至少一个采集周期。
5. 到资产或指标页确认实例已上报数据。

## 接入前校验

无认证（若环境允许）可执行：

```bash
curl -sS "http://<host>:8086/debug/vars"
```

带 Basic Auth：

```bash
curl -sS -u "<username>:<password>" "http://<host>:8086/debug/vars"
```

满足以下条件可认为端点基本可用：

- HTTP 状态码为 `200`
- 返回体为 JSON，包含 `memstats`、`cmdline` 等运行时统计字段

## 页面字段说明

| 页面字段 | 是否必填 | 说明 |
| --- | --- | --- |
| 服务器地址 | 是 | 完整 URL，以 `/debug/vars` 结尾，例如 `http://localhost:8086/debug/vars` |
| 用户名 | 否 | 接口启用 Basic Auth 时填写，否则留空 |
| 密码 | 否 | 与用户名对应，接口启用 Basic Auth 时填写 |
| 间隔 | 是 | 采集周期，单位秒，默认 `60` |
| 超时时间 | 是 | 单次请求 `/debug/vars` 的超时时间，单位秒，默认 `30` |
| CA证书路径 | 否 | HTTPS 场景使用的 CA 证书路径 |
| 客户端证书路径 | 否 | HTTPS 双向认证时的客户端证书路径 |
| 客户端密钥路径 | 否 | HTTPS 双向认证时的客户端密钥路径 |
| 跳过证书校验 | 否 | HTTPS 场景是否跳过服务端证书校验 |
| 节点 | 是 | 负责采集的探针节点 |
| 实例名称 | 是 | 平台内展示的实例名 |
| 组 | 否 | 组织分组，便于权限与资产归属 |

## 常见问题

### 1. 保存后长时间无数据

- 核对服务器地址协议、端口是否正确，且是否以 `/debug/vars` 结尾。
- 在采集节点上重新执行上述 `curl`，确认不是仅控制台可达。
- 确认 InfluxDB v1 的 `/debug/vars` 接口已开放（部分版本或配置可能关闭该接口）。
- 等待至少一个采集间隔后再查看，并检查节点上 Telegraf / 采集任务是否正常运行。

### 2. 认证失败（401 / 403）

- 确认接口是否确实启用了 Basic Auth：未启用时填写账号密码反而可能导致异常。
- 确认用户名密码无首尾空格与特殊字符转义问题。
- 确认所用账号可通过认证访问 `/debug/vars`。

### 3. 仅部分指标缺失或权限不足

- InfluxDB 运行指标均来自 `/debug/vars` 返回体，若部分指标缺失多为该接口返回内容不完整或版本差异所致。
- 先用 `curl` 直接查看 `/debug/vars` 返回，确认目标字段是否存在。
- HTTPS 场景下若返回异常，确认证书路径正确，或在测试环境临时开启「跳过证书校验」定位问题。
