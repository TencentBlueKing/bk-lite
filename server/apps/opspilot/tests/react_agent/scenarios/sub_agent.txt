场景名称: 子 Agent 独立上下文、结果摘要与并行执行测试

对应计划: Phase 3 — #13 子 Agent 独立上下文, #14 子 Agent 结果摘要 (toModelOutput), #15 并行子 Agent

功能描述:
  #13: Supervisor 下的子 Agent 使用独立 context window（context_isolation=True 时），
    不继承主 Agent 全部历史，执行完毕后中间消息不回灌主 Agent。
  #14: toModelOutput 机制——子 Agent 执行完毕后通过 with_structured_output(output_schema)
    提取结构化 JSON 摘要返回给 Supervisor，而非全部消息。graceful fallback on error。
  #15: Supervisor 返回逗号分隔的 agent 名称时触发 PARALLEL 路由，
    parallel_executor_node 使用 asyncio.gather 并行执行多个子 Agent，合并结果。

当前状态: 已实现

涉及代码:
  - server/apps/opspilot/metis/llm/agent/supervisor_multi_agent.py:
    - AgentConfig（context_isolation, output_schema 字段）
    - _structured_extract（toModelOutput 结构化提取）
    - parallel_executor_node（asyncio.gather 并行执行）
    - sub_agent_progress 事件（started/completed/error/parallel_started/parallel_completed）
  - server/apps/opspilot/metis/llm/chain/node.py: build_react_nodes L967（agent_name 参数，注入 step_progress 事件）

测试用例:

  TestSubAgentIsolation:
    1. context_isolation=True 时子 Agent 不继承主 Agent 全部 history: 子 Agent 收到的 messages 只含任务描述
    2. 子 Agent 执行不污染主 Agent 上下文: 子 Agent 的中间 tool_call 消息不回灌主 Agent
    3. context_isolation=False 时子 Agent 继承完整上下文

  TestSubAgentToModelOutput:
    4. output_schema 配置时返回结构化 JSON: with_structured_output 提取结果
    5. 结构化提取失败时 graceful fallback: 返回原始文本而非报错
    6. 无 output_schema 时返回原始 AIMessage content

  TestParallelSubAgents:
    7. 逗号分隔 agent 名称触发并行: Supervisor 返回 "agent_a,agent_b" → PARALLEL 路由
    8. 并行执行 wall-clock 时间小于串行: asyncio.gather 并发
    9. 某子 Agent 失败不影响其他: 失败 agent 返回错误信息，其他正常
    10. 并行结果合并: 所有子 Agent 结果拼接返回给 Supervisor

  TestSubAgentProgressEvents:
    11. 子 Agent 启动时发射 sub_agent_progress started 事件
    12. 子 Agent 完成时发射 sub_agent_progress completed 事件
    13. 子 Agent 失败时发射 sub_agent_progress error 事件
