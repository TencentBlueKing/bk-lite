### 说明
采集 Nginx 的基础配置清单（监听端口、版本、可执行/配置/日志路径、server_name、include、系统 OpenSSL 版本等），标准化同步至 CMDB。本插件为 **JOB（SSH 脚本）** 类型，登录目标读取进程与配置后退出，不修改任何配置。

### 执行方式
本插件为 **JOB（脚本）** 类型，按目标 IP 自动选择执行方式：

| 目标情况 | 执行方式 | 是否需要 SSH 凭据 |
| :--- | :--- | :--- |
| 目标主机**已安装 Agent**（在节点管理的节点列表中） | **本地执行**：Agent 直接在该主机上运行采集脚本 | 不需要 |
| 目标主机**未安装 Agent**（不在节点列表中） | **SSH 远程回退**：由接入点节点 SSH 连入目标执行脚本 | 需要 |

> 一句话：装了 Agent 的机器零凭据即可采集；未装的依赖你填写的 SSH 账号远程采集。

### 版本兼容性
- 支持官方 Nginx 1.10+ 版本（包括：1.28.x、1.29.x、1.26.x、1.24.x 等）。

### 前置要求
1. **网络与连通**
   - SSH 目标：接入点到目标的 SSH 端口（默认 `22`，可自定义）连通。
   - 本地执行目标：该主机的 Agent/Executor 在节点管理中正常在线即可。
2. **采集账号与权限**
   - 能读取 Nginx 主配置文件（`nginx.conf`）。
   - 能执行 `ps` 查看 Nginx 进程、`readlink` 解析可执行文件路径。
   - 能执行 `nginx -v` 获取版本。
3. **目标依赖**
   - 目标上 Nginx 已启动（脚本依赖运行中的进程定位可执行路径与配置）。

### 操作步骤
#### 步骤 0：操作入口
1. 进入“CMDB → 管理 → 自动发现 → 采集 → 专业采集”。
2. 选择插件 **Nginx**，点击“新增任务”。

> 任务实际执行发生在你选择的“接入点”上；下面的自测命令应在接入点机器上执行。

#### 步骤 1：网络连通性自测（接入点执行，仅 SSH 目标）
- Linux：`nc -vz <host_ip> 22`
- Windows PowerShell：`Test-NetConnection <host_ip> -Port 22`

判断标准：端口连通即可。

#### 步骤 2：填写任务
- **采集目标**：填写目标 IP。
- **凭据**：为“未装 Agent”的目标准备 SSH 凭据。
- 设置超时（默认 60s）与采集周期，保存并执行。

#### 步骤 3：验证结果
- 在任务详情查看 `新增 / 更新 / 删除` 摘要与原始数据；在 CMDB 的 Nginx 模型下应能查询到对应实例。

### 凭据字段说明
- `username`：SSH 登录用户名（仅 SSH 目标需要）。
- `password`：上述账号的密码。落库自动加密，下发时以环境变量注入，不写入明文配置文件。
- `port`：SSH 端口，默认 `22`。

### 采集内容
**Nginx（nginx）**

| Key 名称 | 含义 |
| :--- | :--- |
| inst_name | 实例展示名（`{ip}-nginx-{port}`） |
| ip_addr | 主机 IP |
| port | 监听端口 |
| version | Nginx 版本（`nginx -v`） |
| bin_path | 可执行文件路径 |
| conf_path | 主配置文件路径 |
| log_path | 错误日志路径（`error_log` 指令） |
| server_name | server_name 配置 |
| include | include 引入的配置 |
| ssl_version | 系统 OpenSSL 版本 |

> 补充说明：当目标未配置对应指令（如 `error_log`、`server_name`、`include`）或脚本权限不足时，相关字段可能为空。
