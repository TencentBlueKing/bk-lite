### 说明
采集目标主机操作系统层的**文本配置文件内容**，并以配置文件形式归档到目标实例（默认 host）下。采集为**只读**，不修改目标任何文件。

> 说明：`config_file` 不是独立的 CMDB 模型，采集结果以配置文件形式挂到你选择的目标实例下，便于配置盘点与变更追溯。

### 执行方式
本插件为 **JOB（脚本）** 类型，按目标 IP 自动选择执行方式，支持 Linux 与 Windows：

| 目标情况 | 执行方式 | 是否需要 SSH 凭据 |
| :--- | :--- | :--- |
| 目标主机**已安装 Agent**（在节点管理的节点列表中） | **本地执行**：Agent 直接在该主机上运行采集脚本 | 不需要 |
| 目标主机**未安装 Agent**（不在节点列表中） | **SSH 远程回退**：由接入点节点 SSH 连入目标执行脚本 | 需要 |

> 一句话：装了 Agent 的机器零凭据即可采集；未装的依赖你填写的 SSH 账号远程采集。

### 前置要求
1. **网络与连通**
   - SSH 目标：接入点到目标的 SSH 端口（默认 `22`，可自定义）连通。
   - 本地执行目标：该主机的 Agent/Executor 在节点管理中正常在线即可。
2. **采集账号与权限（仅 SSH 远程时需要）**
   - SSH 账号对该配置文件具有**读权限即可，无需 root**。
3. **文件类型**
   - 文件须为文本类型（如 `text/*`、json、xml、yaml）；二进制文件会被拒绝采集。

### 操作步骤
#### 步骤 0：操作入口
1. 进入“CMDB → 管理 → 自动发现 → 采集 → 专业采集”。
2. 选择插件 **配置文件**，点击“新增任务”。

> 任务实际执行发生在你选择的“接入点”上；下面的自测命令应在接入点机器上执行。

#### 步骤 1：网络连通性自测（接入点执行，仅 SSH 目标）
- Linux：`nc -vz <host_ip> 22`
- Windows PowerShell：`Test-NetConnection <host_ip> -Port 22`

判断标准：端口连通即可。

#### 步骤 2：选择目标并填写文件路径
- **采集目标**：选择要挂载结果的目标实例（默认 host）。
- **文件路径**：填写要采集的配置文件绝对路径（`config_file_path`）。
- 为“未装 Agent”的目标准备 SSH 凭据，设置采集周期，保存并执行。

#### 步骤 3：验证结果
- 在任务详情查看执行结果与状态；采集成功后，结果会以配置文件形式归档到目标实例下。

### 凭据/参数字段说明
- `config_file_path`：必填，要采集的配置文件绝对路径。
- `target_model_id`：结果挂载的目标模型，默认 `host`。
- `username`：SSH 登录用户名（仅 SSH 目标需要），对目标文件有读权限即可，无需 root。
- `password`：上述账号的密码。落库自动加密，下发时以环境变量注入，不写入明文配置文件。
- `port`：SSH 端口，默认 `22`。

### 采集内容
| Key 名称 | 含义 |
| :--- | :--- |
| 文件内容 | 配置文件内容（base64 编码） |
| 文件大小 | 文件字节大小 |
| 内容哈希 | 文件内容哈希（SHA256） |
| 版本 | 采集时间戳 |
| 状态 | 采集状态：`success` / `file_not_found` / `permission_denied` / `not_text` |

> 补充说明：`config_file` 不是独立 CMDB 模型，结果以配置文件形式归档到目标实例。状态为 `file_not_found` 表示路径不存在，`permission_denied` 表示账号对文件无读权限，`not_text` 表示文件非文本类型（如二进制）被拒绝采集。
